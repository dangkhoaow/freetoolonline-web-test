Tombol Tidak Terdeteksi Di Tes Keyboard: 4 Perbaikan Yang Benar-Benar Bekerja

Ditinjau terakhir 2026-05-03. Jika tes keyboard browser terbuka tetapi beberapa tombol tidak pernah menyala saat Anda menekan, keyboard jarang rusak - salah satu dari empat penyebab di bawah jauh lebih mungkin. Telusuri daftar dari atas ke bawah; setiap perbaikan memakan waktu kurang dari satu menit.

Jawaban 30 detik. Tombol yang ditekan tetapi tidak pernah menyala di tes keyboard browser hampir selalu berarti halaman kehilangan fokus, OS mengirim scancode yang berbeda dari yang diharapkan tes, filter aksesibilitas mencegat peristiwa, atau tombol terkunci di lapisan hardware - bukan hardware rusak. Langkah 1: klik di mana saja di halaman tes keyboard agar memiliki fokus tab. Langkah 2: konfirmasi layout keyboard OS Anda cocok dengan label fisik (Privasi dan Keamanan → Keyboard / Wilayah). Langkah 3: matikan Sticky Keys, Filter Keys, dan driver keyboard virtual yang mungkin menyerap peristiwa. Langkah 4: periksa toggle baris Fn, LED Insert/ScrollLock, atau masalah koneksi USB. Sebagian besar laporan "tombol tidak terdeteksi" terselesaikan dalam langkah 1 atau langkah 3.

Perbaikan 1 - Halaman kehilangan fokus keyboard

Browser merutekan peristiwa tombol ke elemen mana pun yang memiliki fokus. Tes keyboard menggambar layout di layar, tetapi kecuali Anda klik di halaman terlebih dahulu, ketukan tombol Anda pergi ke bar alamat, popup ekstensi, atau tab berbeda. Gejala: setiap tombol terlihat bekerja (OS menangkapnya), tetapi tidak ada yang menyala di tes.

Klik di tes: klik area kosong di halaman tes keyboard sekali. Halaman sekarang memiliki perutean ketukan; tekan tombol dan akan menyala segera.

Tutup popup dan overlay: prompt autofill, ekstensi pengelola password, dan alat screenshot (Snagit, Greenshot) terkadang menangkap hotkey global. Tutup setiap popup, lalu tes ulang.

Pindah tab dan kembali: jika klien rapat (Zoom, Teams) adalah jendela foreground sebelumnya, mungkin masih memegang ketukan global. Pindah tab dan kembali ke tes keyboard untuk memaksa browser mengklaim ulang fokus.

Jika klik kembali ke halaman membuat tombol menyala, fokus adalah penyebabnya. Lanjut hanya jika penyalaan masih gagal.

Perbaikan 2 - Layout atau bahasa keyboard OS tidak cocok

Tes keyboard menggunakan JavaScript KeyboardEvent.code untuk mengidentifikasi tombol fisik (misal KeyA, Quote). Ketika layout keyboard OS tidak cocok dengan label yang dicetak pada tombol, tekanan terdaftar, tetapi mendarat pada posisi layar yang berbeda - yang terlihat seperti "tombol tidak melakukan apa-apa" karena penyalaan muncul di sel berbeda.

macOS: Pengaturan Sistem → Keyboard → Sumber Input. Konfirmasi layout aktif cocok dengan keyboard fisik Anda (US, UK, Telex Vietnam, dst). Jika baru-baru ini ganti bahasa, ganti kembali; tes akan menyalakan sel yang benar.

Windows 11 / 10: Pengaturan → Waktu dan bahasa → Bahasa dan wilayah. Periksa metode input aktif per bahasa. Tekan Win + Space untuk siklus layout dan tes ulang.

Linux (X11 / Wayland): jalankan setxkbmap -query untuk melihat layout aktif; setxkbmap us reset ke layout US untuk testing.

Tes cepat: tekan tombol Q di kiri atas dan konfirmasi "Q" menyala di layout tes. Jika menyalakan "A" sebagai gantinya, OS pikir keyboard adalah AZERTY (Prancis). Ganti layout OS, jangan beri label ulang tombol.

Perbaikan 3 - Filter aksesibilitas mencegat peristiwa

OS modern dikirim dengan helper aksesibilitas (Sticky Keys, Filter Keys, Slow Keys, Mouse Keys) yang menyerap atau menunda ketukan sebelum browser melihatnya. Sticky Keys mengubah input chord (Ctrl + C) menjadi input urutan (Ctrl, lalu C); Filter Keys mengabaikan tekanan singkat untuk membantu pengguna dengan tremor tangan; keduanya juga mempengaruhi tes keyboard tekan cepat.

Windows: Pengaturan → Aksesibilitas → Keyboard. Matikan Sticky Keys, Filter Keys, dan Toggle Keys. Tes ulang.

macOS: Pengaturan Sistem → Aksesibilitas → Keyboard. Matikan Sticky Keys dan Slow Keys. Banyak laptop secara tidak sengaja mengaktifkan Slow Keys ketika pintasan bantuan pengguna (5x Shift) terpicu.

Driver keyboard virtual: beberapa keyboard dikirim dengan software remapping mereka sendiri (Logitech G HUB, Razer Synapse, ASUS Armoury Crate, Karabiner-Elements di macOS). Ini bisa mencegat peristiwa untuk makro. Keluar dari aplikasi helper dan tes ulang; jika tombol sekarang menyala, helper memiliki remap yang menunjuk ke tempat lain.

Ekstensi browser: Vimium, Surfingkeys, dan Tridactyl mengikat ulang tombol di tingkat halaman. Nonaktifkan ekstensi browser atau gunakan profil tamu.

Perbaikan 4 - Kunci atau koneksi tingkat hardware

Lapisan terakhir adalah keyboard itu sendiri. Beberapa alasan hardware umum membuat tombol tidak pernah mencapai OS:

Toggle baris Fn: laptop bisnis (ThinkPad, Latitude, EliteBook) dikirim dengan Fn-Lock yang mengganti baris F1-F12 antara tombol media (default) dan tombol fungsi. Tekan Fn + Esc (kebanyakan ThinkPad) atau Fn + Shift untuk toggle. Jika tombol F tidak pernah menyala, ini biasanya alasannya.

Tombol kunci (Caps / Num / Scroll): LED di keyboard memberi tahu apakah toggle kunci aktif. Tes: nyalakan Num Lock ON dan tekan numpad - digit akan menyala. Jika tidak, numpad keyboard terhubung ke kontroler USB terpisah yang mungkin tidak terdaftar.

Koneksi USB / Bluetooth: keyboard eksternal di kabel goyah dapat menjatuhkan ketukan individu. Pasang ulang kabel di port berbeda; untuk Bluetooth, pasangkan ulang. Keyboard nirkabel dengan baterai hampir kosong sering menampilkan gejala persis ini.

BIOS HID nonaktif: BIOS bisnis Lenovo, Dell, dan HP mengekspos toggle "Dukungan keyboard USB" yang mengontrol apakah keyboard bekerja di lingkungan pra-OS dan apakah semua tombol diteruskan ke OS. Reboot, masuk setup (F1 / F2 / F12), cari Perangkat → Pengaturan USB atau Keamanan → Akses Port I/O → USB, konfirmasi diaktifkan.

Masih ada tombol hilang? - alur diagnostik

  1. Coba situs kedua. Buka en.key-test.ru dalam sesi browser yang sama. Jika situs itu juga melewatkan tombol yang sama, penyebabnya di hulu halaman tes (fokus, layout, aksesibilitas, atau hardware).
  2. Coba browser kedua. Jika Chrome melewatkan tombol tetapi Firefox menangkapnya, penyebabnya di tingkat browser (ekstensi, profil, atau listener peristiwa basi). Gunakan profil tamu untuk konfirmasi.
  3. Coba akun pengguna kedua. Beralih ke pengguna OS yang baru dibuat dan buka tes keyboard. Jika berhasil di sana, penyebabnya di profil pengguna Anda (remap, flag aksesibilitas, atau driver keyboard virtual terdaftar untuk pengguna Anda).
  4. Coba mesin kedua. Jika ponsel atau tablet menunjukkan tes keyboard bekerja segera dengan keyboard Bluetooth yang dipasangkan, mesin yang gagal memiliki masalah hardware atau driver. Buka Device Manager (Windows) atau System Information (macOS) dan cari keyboard di bawah Human Interface Devices.

Setiap pass menelusuri ketukan dari tempat browser menangkapnya kembali ke saklar di bawah jari Anda - fokus terlebih dahulu, lalu layout, lalu filter aksesibilitas, lalu keyboard itu sendiri - dan sebagian besar laporan tombol hilang terselesaikan dalam langkah satu (fokus) atau langkah tiga (filter aksesibilitas yang Anda lupa aktif). Jika tombol masih tidak menyala setelah langkah empat, saklar itu sendiri yang dicurigai; ganti ke keyboard yang dikenal baik sebelum mengasumsikan mesin rusak.

Bagaimana dengan tombol modifier (Shift, Ctrl, Alt, Cmd, Win)?

Tombol modifier adalah tombol "hilang" paling umum dalam tes keyboard cepat karena mereka hanya menyala saat Anda juga menekan non-modifier dalam chord. Halaman tes keyboard menangani setiap modifier sebagai tekanan mandiri: tahan Shift (kiri atau kanan) dan sel terkait di tes akan menyala - tidak perlu tombol lain. Jika hanya satu sisi menyala, saklar sisi lain mungkin memiliki kontak macet - tekan kedua sisi dan cari perilaku asimetris.

FAQ

Mengapa tombol yang sama kadang menyala dan kadang tidak?

Deteksi tidak konsisten biasanya masalah hardware - kotoran atau aus di bawah keycap. Pop keycap, tiup debu dengan udara terkompresi, dan tes ulang. Jika masalah berlanjut di pengguna OS lain (Diagnostik Fix 3), hardware keyboard memerlukan layanan garansi.

Apakah Caps Lock mempengaruhi hasil tes keyboard?

Tidak. Tes membaca KeyboardEvent.code, yang sama terlepas dari status shift / caps. Jika A dan a keduanya menyalakan sel A, tes bekerja dengan benar.

Mengapa tombol multimedia (Play, Pause, Volume) tidak menyala?

Sebagian besar tombol multimedia tingkat OS diserap oleh OS sebelum browser melihatnya. Tes keyboard tidak dapat menyalakannya secara andal. Ketiadaan mereka tidak berarti tombol rusak - tekan satu dan periksa volume OS merespons.

Saya melihat tombol yang benar menyala di tes, tetapi karakter yang salah diketik di kolom teks. Apakah keyboard rusak?

Tidak - hardware keyboard baik-baik saja. Layout keyboard OS Anda diatur ke bahasa berbeda (Perbaikan 2). Tes mengkonfirmasi tombol fisik bekerja; layout memutuskan karakter apa yang dihasilkan.
