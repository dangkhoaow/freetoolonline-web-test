Memulihkan File Zip Rusak - Opsi

Ditinjau terakhir 2026-04-27. Alat browser dapat mengekstrak bagian yang dapat dipulihkan dari zip yang rusak; perbaikan penuh biasanya memerlukan alat desktop.

Jawaban 30 detik. Zip yang mengatakan "rusak" dapat berarti empat hal berbeda, dengan empat perbaikan berbeda. Unduh ulang file - sebagian besar zip "rusak" adalah unduhan yang terputus. Jika itu gagal, coba ekstraksi parsial yang menarik apa yang dapat dibaca. Jika itu gagal, jalankan alat perbaikan desktop. Jika itu gagal, data kemungkinan sudah hilang dan langkah yang tepat adalah meminta pengirim untuk mengirim ulang.

Langkah 1 - unduh ulang

Sebagian besar laporan "zip rusak" berasal dari unduhan yang terpotong. Browser menghentikan transfer terlalu cepat, atau CDN menyajikan file parsial. Perbaikan yang bekerja dalam hitungan detik sebelum alat perbaikan apa pun: hapus cache browser dan unduh ulang; tarik file dengan alat yang berbeda (curl di desktop, manajer unduhan di mobile); atau minta tautan segar jika yang asli berasal dari aplikasi chat di mana URL lampiran kedaluwarsa.

Diagnostiknya: jalankan checksum pada zip yang diunduh dan bandingkan dengan pengirim. Jika berbeda, file adalah masalahnya, bukan zip itu sendiri.

Langkah 2 - ekstraksi parsial

Direktori pusat zip berada di akhir file. Jika kerusakan ada di akhir, setiap file di dalam tidak dapat dijangkau - meskipun byte masih ada. Jika kerusakan ada di tengah, file sebelum area yang rusak masih dapat diekstrak.

Coba unzip file di browser. Jika ia mendaftar konten tetapi error saat ekstraksi, direktori pusat utuh dan satu atau lebih entri file rusak. Beberapa alat unzip (7-Zip di Windows, The Unarchiver di macOS) dapat mengekstrak semua sebelum kerusakan dan melewati sisanya. Layak dicoba ketika hanya sebagian file penting.

Langkah 3 - alat perbaikan desktop

Ketika ekstraksi parsial gagal, alat perbaikan dapat membangun ulang direktori pusat dengan memindai header file di badan. Dua opsi gratis:

  • 7-Zip di Windows. Klik kanan pada zip rusak, pilih 7-Zip > Open. Jika dapat memindai badan, ia mendaftar apa yang ditemukan - bahkan tanpa direktori pusat.
  • zip di Linux/macOS. Perintah zip -FF broken.zip --out fixed.zip mencoba memperbaiki zip dengan struktur yang rusak. Bekerja untuk banyak kasus di mana file berakhir di tengah-entri.

Perbaikan berbasis browser jarang karena algoritma memindai file besar dan berjalan lebih cepat di disk lokal. Jika Anda tidak memiliki desktop yang tersedia, jalur yang tepat biasanya adalah meminta pengirim untuk mengirim ulang.

Langkah 4 - kapan menyerah

Jika zip dibuat dengan enkripsi dan kerusakan ada di mana saja dekat header file, konten kemungkinan tidak dapat dipulihkan. Sama jika media penyimpanan (drive USB yang gagal, kartu SD yang rusak) adalah sumbernya - kerusakan ada di byte itu sendiri, bukan pembungkusnya. Langkah yang tepat adalah meminta pengirim asli untuk mengirim ulang, atau memulihkan dari backup jika ada.

Pencegahan ke depan

Untuk file yang penting, kirim checksum bersama zip. Penerima dapat memverifikasi unduhan adalah byte-sempurna sebelum menghabiskan waktu pada perbaikan. MD5 atau SHA-256 keduanya bekerja; SHA-256 adalah pilihan yang lebih aman. Lihat mengapa MD5 tidak dapat didekripsi untuk konteks keamanan. Untuk semua yang berhubungan dengan zip, lihat hub alat zip.
