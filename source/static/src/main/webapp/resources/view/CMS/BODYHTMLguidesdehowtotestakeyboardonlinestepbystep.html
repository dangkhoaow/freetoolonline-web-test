So testen Sie eine Tastatur online, Schritt fuer Schritt

Zuletzt geprueft 2026-05-02. Oeffnen Sie den Tastaturtest, um die Schritte unten auszufuehren - keine Installation, kein Konto, kein Upload.

60-Sekunden-Antwort. Oeffnen Sie den Tastaturtest in Ihrem Browser, druecken Sie jede Taste auf Ihrer physischen Tastatur und beobachten Sie, wie jede auf dem Bildschirmlayout aufleuchtet. Eine Taste, die beim Druecken nicht aufleuchtet, funktioniert nicht. Eine Taste, die nach dem Loslassen aufleuchtet bleibt, klemmt. Modifier-Tasten (Shift, Ctrl, Alt, Cmd, Fn) und Sperrtasten (Caps, Num, Scroll) werden auf die gleiche Weise geprueft.

Warum eine Tastatur im Browser testen

Ein Browser-Test ist schneller als das Installieren eines Hersteller-Dienstprogramms und ehrlicher als das Tippen in ein Dokument - Tippen offenbart nur die Tasten, die Sie zufaellig gedrueckt haben, waehrend ein layoutgesteuerter Test Sie zwingt, jede Taste in Reihenfolge durchzugehen. Der Browser sieht die gleichen Tastenanschlag-Ereignisse, die das Betriebssystem an jede andere Anwendung liefert, also wird eine Taste, die im Test fehlschlaegt, auch in Ihrem Editor, Terminal und Spiel fehlschlagen.

Schritt fuer Schritt

  1. Oeffnen Sie den Test. Besuchen Sie /keyboard-test.html. Die Seite laedt ein Tastaturlayout auf dem Bildschirm. Keine Anmeldung, keine Berechtigungsanfrage.
  2. Gehen Sie jede Taste durch. Beginnen Sie bei Esc (oben links), druecken Sie sie, beobachten Sie das Aufleuchten im Layout. Bewegen Sie sich nach rechts durch die Funktionstastenreihe, dann nach unten durch die Zahlenreihe, die QWERTY-Reihen und enden Sie an der Leertaste und Pfeiltasten. Jeder Druck sollte die entsprechende Taste auf dem Bildschirm aufleuchten lassen.
  3. Pruefen Sie die Modifier- und Sperrtasten. Druecken Sie Shift, Ctrl, Alt, Cmd (macOS) oder die Windows-Taste und die Fn-Taste, falls Ihre Tastatur eine hat. Druecken Sie dann Caps Lock, Num Lock und Scroll Lock. Die Sperrtasten sind Toggles - einmal druecken zum Aktivieren, erneut druecken zum Loesen.
  4. Testen Sie Akkord-Eingabe. Halten Sie Shift und eine Buchstabentaste zusammen - beide sollten waehrend des Haltens aufleuchten bleiben. Lassen Sie los; beide sollten sich loeschen. Wiederholen Sie mit Ctrl + Buchstabe, dann Alt + Buchstabe. Dies erfasst Modifier-Tasten-Fehler, die nur unter Kombination auftauchen.
  5. Testen Sie n-key-Verhalten. Halten Sie W, A, S, D gleichzeitig (der Standard-Gaming-Akkord). Alle vier sollten aufleuchten. Fuegen Sie Q hinzu, waehrend Sie die anderen noch halten. Auf einer 6-key-Rollover-Tastatur sollten Sie alle fuenf sehen; auf einer 2-key-Rollover-Tastatur wird der fuenfte Druck fehlschlagen.
  6. Suchen Sie nach klemmenden Tasten. Nach dem Loslassen jeder Taste sollte das Bildschirmlayout vollstaendig geloescht sein. Eine Taste, die nach dem Loslassen aufleuchtet bleibt, klemmt - meist Schmutz unter der Keycap, manchmal ein abgenutzter Switch.

Was jeder Fehler bedeutet

  • Taste leuchtet beim Druecken nicht auf. Nicht funktionierende Taste - Schmutz, abgenutzter Switch oder gebrochene Leiterbahn. Versuchen Sie zuerst Druckluft; wenn das fehlschlaegt, muss der Switch wahrscheinlich auf einer mechanischen Tastatur ausgetauscht werden oder die Tastatur selbst auf einer Membrantastatur.
  • Taste bleibt nach Loslassen aufleuchten. Klemmende Taste - meist Schmutz zwischen Kappe und Switch. Keycap abnehmen, Schmutz herausblasen, wieder aufsetzen. Wenn es weiter besteht, ist der Switch beschaedigt.
  • Modifier-Taste schlaegt nur unter Akkord fehl. Die Matrix oder die Firmware verliert den Modifier unter Last. Auf einer Budget-Tastatur ist das normal (2KRO). Zum Tippen ist es in Ordnung; fuer Gaming oder Musik nicht.
  • Mehrere Tasten schlagen zusammen fehl. Oft dieselbe Matrixreihe oder -spalte. Wenn Q, A und Z alle fehlschlagen, ist die Spalte, die sie teilen, der Verdaechtige. Wenn Q, W, E, R, T alle fehlschlagen, ist die Reihe der Verdaechtige.
  • Drahtlose Tastatur verpasst Tasten. Bluetooth- oder 2.4-GHz-Interferenz. Bringen Sie den Empfaenger naeher oder testen Sie verkabelt, falls die Tastatur das unterstuetzt.

Was der Browser nicht pruefen kann

Der Browser sieht Tastenanschlaege, nachdem das Betriebssystem sie verarbeitet hat. Einige Dinge bleiben fuer den Test unsichtbar:

  • USB-Berichtsrate. Polling mit 125 Hz, 500 Hz oder 1000 Hz ist eine Einstellung auf Hardware-Ebene; der Browser sieht die Anschlaege, wie sie eintreffen, misst aber den Intervall nicht mit Sub-Millisekunden-Aufloesung.
  • Remapping auf Firmware-Ebene. Gaming-Tastaturen mit Onboard-Speicher mappen Tasten neu, bevor sie das Betriebssystem erreichen. Der Browser sieht die neu zugewiesene Taste, nicht die physische.
  • Akustisches Tastengeraeusch. Klick und Klacken sind physisch; der Browser sieht das Druck-Ereignis unabhaengig davon, wie laut oder leise der Switch ist.
  • Hintergrundbeleuchtungs- oder RGB-Fehler. Beleuchtung ist nur Hardware; der Browser sieht nicht, wie Ihre Tasten aussehen.
