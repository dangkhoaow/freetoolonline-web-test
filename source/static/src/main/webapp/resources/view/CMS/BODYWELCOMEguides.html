Each entry in the index is one line: the title that names the reader task, and a short summary sentence that says who the guide is for. A reader who scans only the summaries can pick the correct walkthrough before clicking, instead of opening three pages and discovering the second one was the right answer. Topic blocks group the guides by job-to-be-done so the scan converges quickly on the input the reader actually has.

Every guide linked from this index ends in the same place: the bytes the reader works with - photos, archives, documents, hashes - never get uploaded to a server because the tools each guide drives run locally in the reader's browser. The hub does not repeat that contract per-guide. It states it once at the bottom so the privacy posture is the last thing a reader sees before clicking through to a walkthrough.
