Como extraer un archivo en linea - ZIP, RAR, 7z (que ruta en este sitio hace que)

Ultima revision 2026-05-04. Una regla directa de enrutamiento para la consulta de busqueda "extraer archivo en linea" / "file zipper" / "extractor de archivo en linea". La direccion inversa de la guia de enrutamiento del compresor-de-archivos del ciclo-51: en vez de "tengo una carpeta, la quiero zipeada", tienes un archivo .zip / .rar / .7z y quieres los archivos dentro de vuelta en tu navegador. La respuesta honesta depende de la extension del archivo - y uno de los tres formatos que este sitio no extrae server-side. La guia nombra que ruta hace que, y a donde ir para los formatos que este sitio no maneja.

Respuesta en 30 segundos. Si tu archivo es un .zip, abre Unzip File - suelta el archivo, se sube al servicio AWS freetoolonline, y el contenido vuelve como una descarga. Si el .zip necesita contrasena, la misma herramienta acepta una entrada de contrasena en el mismo flujo. Si tu archivo es .rar o .7z, este sitio no tiene extractor server-side para esos formatos - la ruta correcta es la app integrada del SO o app local (7-Zip, Keka, p7zip, unrar) y la guia explica por que la honestidad importa aqui. Si no recuerdas la contrasena de un .zip, Remove ZIP Password es una operacion diferente - cubierta en la regla 4.

Por que "extraer archivo en linea" usualmente significa una de tres operaciones

La frase "extraer archivo en linea" es una consulta de busqueda para tres diferentes tareas-del-lector. La primera es "tengo un .zip y quiero los archivos dentro" - la direccion-inversa rutinaria de zipear. La segunda es "tengo un .rar o .7z y quiero los archivos dentro" - mismo objetivo, diferente contenedor de archivo. La tercera es "olvide la contrasena de un .zip y no puedo abrirlo" - que no es extraccion en absoluto sino recuperacion de contrasena. Elegir la herramienta correcta depende de en cual de las tres estas, y en este sitio la respuesta difiere por extension.

La razon importa: poner un .rar en una herramienta que solo acepta .zip no devuelve un mensaje de error que la mayoria de los lectores encuentra util - devuelve "tipo de archivo incorrecto" o silenciosamente rechaza la subida. Y el tiempo perdido re-subiendo el mismo archivo no cambia la respuesta. Las cuatro reglas de enrutamiento abajo cubren los casos reales, incluyendo que hacer cuando este sitio no tiene el extractor para tu formato - porque es mas util leer "abre 7-Zip o Keka localmente" que seguir haciendo clic en una herramienta que no maneja el formato.

Regla de enrutamiento 1 - tu archivo es un .zip: abre Unzip File (server-upload, contrasena opcional)

Si el archivo en tu disco tiene extension .zip, Unzip File es la ruta. Suelta el archivo en el widget de subida, el servicio AWS freetoolonline descomprime el archivo server-side, y el contenido vuelve como una descarga. Senales concretas de que estas en este caso:

  • Recibiste un .zip como adjunto de correo y tu telefono o laptop de trabajo no tiene una herramienta de escritorio para descomprimir a mano. La herramienta de navegador devuelve el contenido en un viaje de ida-y-vuelta.
  • El .zip contiene muchos archivos (una carpeta de proyecto, un lote de recibos, una carpeta de fotos) y quieres inspeccionar el contenido antes de decidir que mantener. Unzip File devuelve los archivos contenidos; guardas los que quieres.
  • El .zip esta protegido por contrasena y conoces la contrasena. Unzip File expone una entrada de contrasena en el mismo flujo - ingresa cuando se solicite y la extraccion procede. La guia companera para la direccion inversa (zipear una carpeta con contrasena) es Como zipear una carpeta en linea paso a paso.

Razon: Unzip File esta construido especificamente para la tarea-del-lector de extraccion .zip. El filtro de archivo aceptado en el widget de subida es solo .zip - esto esta documentado en la fuente de la herramienta, y la guia hace el limite explicito para que no desperdicies una subida en un archivo que la ruta no puede manejar. Si tu archivo es una variante .zip (.zipx) o un .zip envuelto en otro archivo, ver la regla 3.

Regla de enrutamiento 2 - tu archivo es un .rar o .7z: este sitio no extrae esos server-side

Si tu archivo en disco tiene extension .rar o .7z, este sitio no tiene extractor server-side para esos formatos. No hay ruta /unrar.html o /un7z.html, y el filtro de archivo aceptado de Unzip File es solo .zip - subir un .rar o .7z a esa ruta no produce un resultado util. La respuesta honesta es: abre el archivo localmente. Rutas concretas por sistema operativo:

  • Windows: 7-Zip (gratis, open-source, maneja .zip / .rar / .7z), WinRAR (nativo .rar, trial gratuito), o PeaZip (gratis, soporte amplio de formatos). Clic derecho en el archivo, "Extraer a una carpeta", listo.
  • macOS: Keka (gratis / pago, App Store) o The Unarchiver (gratis, App Store) manejan .zip / .rar / .7z. El Archive Utility integrado de macOS abre .zip; para .rar / .7z necesitas una de las apps de arriba.
  • Linux: 7z x archive.7z (p7zip; apt install p7zip-full) para .7z, unrar x archive.rar (unrar; apt install unrar) para .rar.

Razon: escribir "extraemos RAR en linea" cuando el sitio no tiene extractor .rar es el tipo de afirmacion falsa que desperdicia el tiempo del lector y rompe confianza. Cuando una ruta en linea no existe para un formato, el fallback de herramienta-local es la respuesta correcta, y la guia es mas util cuando lo dice directamente. La comparacion companera ZIP en linea vs 7z vs RAR - cual elegir recorre que formato elegir cuando controlas el lado productor; esta guia es para el lado consumidor (ya tienes el archivo enfrente).

Regla de enrutamiento 3 - extension mixta o no obvia: elige por lo que el archivo realmente es

Los casos limpios arriba cubren la mayoria de la consulta de busqueda. Los casos mixtos necesitan una pregunta extra - que es el archivo realmente? Sub-casos concretos:

  • La extension dice .zip pero Unzip File lo rechaza: el archivo esta mal-etiquetado (alguien renombro un .rar a .zip) o el .zip esta malformado. El flujo de diagnostico esta en Opciones para recuperar archivo ZIP corrupto - que dispone los tres cubos (verdaderamente corrupto, mal-etiquetado, contrasena-requerida-pero-prompt-no-disparado).
  • El archivo no tiene extension, o tiene .tar / .gz / .tar.gz / .bz2: este sitio no tiene ruta TAR / GZIP. El fallback de herramienta-local de la regla 2 aplica - 7-Zip maneja .tar.gz en Windows; tar -xzvf archive.tar.gz en macOS / Linux desenvuelve ambas capas en un paso.
  • El archivo es un .zipx (la variante WinZip): la mayoria de las herramientas de escritorio de descompresion manejan .zipx (7-Zip, WinRAR, Keka) pero el Unzip File de este sitio acepta solo el filtro estandar .zip - intentalo una vez; si rechaza la subida, recurre a una herramienta local.
  • El "archivo" es en realidad un instalador (.dmg, .pkg, .msi, .exe): esos no son archivos en el mismo sentido. No los subas a Unzip File. Abrelos con su instalador nativo o usa una herramienta de escritorio que conoce el formato especifico.

Razon: la consulta de busqueda "extraer archivo en linea" barre formatos de archivo que este sitio no extrae. Elegir por lo que el archivo realmente es (en vez de por lo que la extension dice) evita el bucle subir-y-fallar. La guia companera ZIP vs ZIPx vs RAR vs 7z formatos de archivo explicados es el explicador-de-formato; esta guia es la regla operacional de enrutamiento.

Regla de enrutamiento 4 - olvidaste la contrasena de un .zip: es una herramienta diferente

Si tu .zip esta protegido por contrasena y no sabes la contrasena, no sigas intentando abrirlo en Unzip File - la entrada de contrasena es para ingresar una contrasena conocida, no para adivinar. La ruta mas cercana en este sitio es Remove ZIP Password, que es una operacion diferente: intenta recuperar o quitar la proteccion de contrasena de un .zip. Tres advertencias valen decir directamente:

  • Recuperacion no esta garantizada. Contrasenas fuertes con cifrado ZIP AES-256 moderno estan disenadas para ser dificiles de romper; la herramienta puede devolver un resultado "no se pudo recuperar" en contrasenas intencionalmente fuertes.
  • Contenido sensible no debe subirse. El mismo limite de privacidad que aplica a Unzip File aplica aqui - el archivo se sube al servicio AWS freetoolonline y se procesa server-side. Si el .zip contiene material que no puedes subir (medico, legal, NDA), el fallback de herramienta-local de la regla 2 mas una herramienta local de recuperacion de contrasena es la ruta correcta, y la regla 5 abajo nombra dos opciones de escritorio.
  • La guia companera Tipos de contrasena PDF - owner vs user cubre una distincion relacionada para PDFs (contrasenas owner-set vs user-set); para .zip la distincion es mas simple - o la contrasena esta puesta o no.

Razon: recuperar una contrasena olvidada es una tarea-del-lector diferente de "tengo un archivo, quiero el contenido". Nombrar el limite mantiene la regla de enrutamiento limpia: Unzip File para ZIPs con-contrasena-conocida y sin-contrasena, Remove ZIP Password para ZIPs con-contrasena-olvidada, herramientas locales para todo lo demas. Para el conjunto completo de herramientas ZIP en este sitio, ver ZIP Tools.

Regla de enrutamiento 5 - el archivo es enorme o sensible: extrae localmente en vez de subir

Unzip File sube tu archivo al servicio AWS freetoolonline y la extraccion corre server-side. Esa es la ruta correcta para casi cada tarea-del-lector de herramienta de navegador, pero dos casos piden el fallback de herramienta-local. Primero, archivos muy grandes (un archivo de proyecto multi-gigabyte, un backup .zip en el rango de gigabytes) son lentos para subir y lentos para descargar incluso en una buena conexion - si controlas una maquina de escritorio, el extractor del SO integrado o app local (regla 2) es mas rapido extremo-a-extremo porque los bytes nunca dejan tu disco. Segundo, archivos que no puedes subir por razones de confidencialidad (imagenes medicas, evidencia legal, codigo bajo NDA) necesitan quedarse locales - incluso cuando el archivo es pequeno, la herramienta local es la ruta correcta.

El trade-off tiene la misma forma que las decisiones equivalentes para comprimir archivos (Compresor de archivos en linea: ZIP una carpeta vs Comprimir una imagen) y convertir archivos de video / imagen (FFmpeg en linea vs FFmpeg local - cuando cada uno gana, ImageMagick en linea vs herramientas especificas). El limite es el mismo: rutas en linea ganan en conveniencia para entradas de tamano normal, no-sensibles; rutas locales ganan en velocidad y privacidad para entradas enormes o sensibles. Para extraccion .zip la guia del compresor-de-archivos del ciclo-51 nombra el mismo limite en la direccion inversa (comprimir, no extraer), y la respuesta sitewide completa a la pregunta "sin subida" esta en Herramientas en linea gratis que funcionan sin subir archivos - Unzip File no esta en esa lista, exactamente porque el paso de extraccion corre en el servidor.

Que pasa con tu archivo despues de subirlo (y cuando saltarse la subida)

Cuando sueltas un .zip en Unzip File, el archivo se sube al servicio AWS freetoolonline (region us-east-1), se descomprime server-side, y el contenido se escribe de vuelta al bucket de salida del mismo pipeline de subida para que tu navegador pueda descargarlo. El archivo y la salida extraida no se mantienen indefinidamente - el pipeline de subida corre una limpieza programada que remueve archivos despues de una ventana de retencion fija (configurada site-wide; en el orden de decenas de minutos, no horas o dias). Esta es la misma ventana de retencion que toda otra herramienta basada en subida en este sitio usa (Compress Image, Zip File, HEIC to JPG y asi sucesivamente). Dos consecuencias practicas para ti como lector:

  • Descarga tu salida antes de cerrar la pestana. Una vez que la ventana de limpieza pasa, el mismo upload-id ya no resuelve a un archivo descargable - si cerraste la pestana sin guardar, re-subes desde tu copia local del archivo. Marcar la URL de resultado no es una solucion alternativa; la ventana de retencion aplica al archivo, no a la URL.
  • Para archivos muy grandes o sensibles a confidencialidad, saltate la subida completamente. Archivos multi-gigabyte son mas lentos de subir que de extraer localmente, y contenido que no puedes subir (medico, legal, NDA) necesita quedarse en disco. El fallback de SO local de la regla 2 es la ruta correcta en ambos casos - 7-Zip, Keka, o el comando de plataforma 7z x / unrar x extrae el archivo sin enviar los bytes al servidor, y no hay ventana de retencion para planear.

El limite es identico al trade-off de subida en Compresor de archivos en linea: ZIP una carpeta vs Comprimir una imagen (regla 4 en esa guia) y la respuesta de privacidad en direccion-inversa en Herramientas en linea gratis que funcionan sin subir archivos - Unzip File NO esta en esa lista, exactamente porque el paso de extraccion usa el pipeline server-upload. Si el limite de subida es el factor decisivo para tu archivo, el fallback de herramienta-local de la regla 2 es la ruta a seguir.

Si la subida se detuvo o la salida esta mal: troubleshoot antes de re-ejecutar

Si Unzip File acepta tu subida pero ninguna descarga aparece en un par de minutos, no re-subas. Re-subir el mismo archivo no arregla la causa subyacente - encola un segundo trabajo que choca con la misma pared. Los atascos comunes son: limite de tamano de entrada alcanzado en la subida (el archivo es mas grande que el tope por-archivo; regla 5 aplica), pestana de navegador refrescada durante el pipeline de conversion (el resultado descarga a la pestana cerrada y se pierde), o una entrada no-.zip dada al filtro solo-.zip (regla 1 vs regla 2 fue la llamada incorrecta - re-lee la extension y enruta de acuerdo).

Si la salida es de la forma incorrecta - por ejemplo el .zip se extrajo pero los archivos dentro son ellos mismos archivos - eso es un caso de archivo anidado: extrae el .zip externo primero, luego enruta cada archivo interno por su propia extension (un .rar dentro de un .zip todavia es un .rar y la regla 2 aplica). Si el destinatario no puede abrir el .zip que produjiste desde la direccion inversa (zipeando una carpeta), pueden necesitar Unzip File en su navegador, y si una contrasena esta en el camino Remove ZIP Password cubre el flujo de recuperacion. Para todo lo demas - una operacion comun, la ruta correcta por extension, el fallback correcto para archivos enormes o sensibles - la respuesta toma 30 segundos; la respuesta incorrecta toma otro minuto de tiempo de subida que no necesitabas gastar.
