Kompres Ukuran ZIP: Seberapa Lebih Kecil Arsip Sebenarnya?

"Kompres ukuran ZIP" terdengar seperti slider yang bisa Anda dorong, tetapi jawabannya ada di sisi input, bukan di sisi arsip. Folder berisi teks biasa, kode sumber, log, atau CSV sering menyusut 60 hingga 80 persen di dalam .zip, karena langkah DEFLATE menghapus bentangan pola berulang. Folder berisi foto JPG, video MP4, atau audio MP3 menyusut sekitar 0 hingga 5 persen, karena setiap file sudah dikompresi gaya DEFLATE oleh codec-nya sendiri sebelum mendarat di folder. Pemotongan ukuran ditentukan oleh apa yang Anda masukkan ke arsip.

Di mana sebenarnya membangun arsip: pembuat ZIP di browser ada di /zip-tools/zip-file.html. Seret folder ke dalam, alat mengemas setiap file ke dalam satu .zip di perangkat Anda, dan unduhan tiba tanpa konten pernah menyentuh server. Pemotongan ukuran yang Anda baca di tabel di atas adalah yang akan Anda lihat di properti file hasil.

Bagaimana dengan pengaturan "level kompresi"? Dalam praktiknya, biarkan pada default DEFLATE standar. Maximum atau Ultra menambah waktu CPU di sisi Anda dan jarang memotong lebih dari beberapa persen ekstra dari arsip yang berat teks; Store atau Fast hanya membantu ketika kecepatan lebih penting daripada ukuran. Kompresibilitas input (tabel di atas) menetapkan plafon seberapa kecil arsip dapat menjadi; slider level sebagian besar mengubah berapa lama build berjalan, bukan apakah arsip menyusut.

Jika Anda membutuhkan target ukuran tertentu. Ketika tujuannya adalah "muat di bawah 25 MB untuk email" daripada "jadikan sekecil mungkin", alat yang tepat adalah varian target ukuran di cara mengompres ZIP ke ukuran lebih kecil (panduan batas 2 MB / 25 MB / 100 MB). Untuk pertanyaan konseptual mendasar "kapan ZIP benar-benar menghemat ruang", panduan saudara di kompres ZIP mencakup disambiguasinya. Halaman ini adalah jawaban praktis "seberapa banyak akan menyusut" untuk folder yang sudah Anda miliki.

Jika sebagian besar folder Anda adalah media. Pisahkan pengiriman sebelum mengarsipkan. Zip file mirip teks bersama untuk kemenangan ukuran nyata, lalu biarkan foto, video, dan audio apa adanya atau encode ulang dengan alat-alat sendiri (resize, kualitas-ulang JPEG, atau transcoding MP4). Dua bagian lebih kecil biasanya lebih kecil daripada satu ZIP yang membungkus media yang tidak punya apa-apa lagi untuk dikompresi.

Apa yang tidak bisa dibantu arsip. PDF terenkripsi, ZIP berkata sandi, MP4, dan file apa pun yang sudah dienkode dengan codec lossless atau lossy modern tidak dapat dikompresi secara aritmatik oleh ZIP. Algoritma DEFLATE membutuhkan pola berulang untuk menemukan simbol yang lebih pendek; ciphertext dan output codec sengaja terlihat acak dan tidak memiliki apa pun yang berulang untuk dihapus. Output ZIP pada input ini kira-kira ukuran input ditambah sedikit overhead pengemasan, kadang sedikit lebih besar dari aslinya.
