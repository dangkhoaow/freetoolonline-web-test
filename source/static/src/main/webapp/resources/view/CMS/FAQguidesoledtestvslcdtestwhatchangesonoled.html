Frequently asked questions

Does the LCD test work on an OLED screen?

Yes. The full-screen red, green, blue, black, and white color cycle from the LCD test runs on any pixel-based screen, OLED included. The dead-pixel and stuck-pixel checks read the same way on either panel; only the backlight-bleed check from the LCD test does not apply on OLED, because OLED has no separate backlight layer.

Can I check for OLED burn-in in the browser?

The browser test can SHOW the static frames needed for a burn-in or image-retention judgement, but the verdict is reader judgement, not an automated diagnosis. Leave a solid color frame for two to three minutes; switch to the next frame; faint outlines of the prior frame mean image retention, and outlines that remain after the color cycle has run for several minutes and the screen has been off for a while are burn-in.

Why does backlight-bleed not apply to OLED?

OLED pixels emit their own light per pixel; there is no separate backlight layer. The backlight-bleed check on the LCD test looks for light leaking around the edges of a dark frame from the LED backlight behind the LCD panel — on OLED, that backlight layer does not exist, so the leak it looks for cannot occur.

What is the difference between burn-in and image retention?

Image retention is temporary persistence of a static frame after the frame has been shown for a few minutes; it typically clears after the screen has run the color cycle for a few more minutes or has been off for a while. Burn-in is permanent pixel wear that does not clear after running the cycle or after the screen has been off; the screen-test tool cannot recover burn-in, it can only confirm it.
