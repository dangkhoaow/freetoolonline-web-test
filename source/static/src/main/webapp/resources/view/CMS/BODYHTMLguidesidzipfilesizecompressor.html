Apa yang sebenarnya dilakukan kompresor ukuran file zip

Kompresor mengambil folder file, menjalankan kompresi ZIP pada setiap entri, dan menulisnya ke satu arsip .zip. File hasil akhir lebih kecil dari jumlah file asli untuk sebagian besar input - penghematan datang dari menghapus redundansi dalam konten berbentuk teks (log, kode sumber, CSV, JSON). Untuk input yang sudah dikompres (foto JPG, video MP4, dokumen Office modern), pembungkus .zip menghemat byte hanya pada level struktur direktori, bukan di dalam tubuh file. Harapkan pemotongan ukuran sederhana pada folder foto dan pemotongan besar pada folder teks.

Cara mengompres folder menjadi satu .zip

  1. Buka Zip File di tab browser ini.
  2. Jatuhkan folder ke zona unggah (atau pilih dari pemilih file).
  3. Opsional: masukkan password di kolom password jika Anda ingin arsip terkunci.
  4. Klik Convert. Arsip dibundel di host layanan freetoolonline.
  5. Unduh .zip kembali ke perangkat Anda. Yang asli di mesin Anda tetap utuh.

Kapan panduan ini menjadi pilihan yang tepat

Panduan ini menjawab tugas "saya punya folder, saya ingin satu .zip yang lebih kecil". Jika input adalah satu gambar bukan folder, beralih ke Compress Image. Jika input adalah PDF, beralih ke Compress PDF. Jika Anda sudah punya .zip dan ingin membuatnya lebih kecil, alat yang tepat adalah Cara Membuat File Zip Lebih Kecil - yang mencakup mengulang zip dengan media dikeluarkan dan memecah .zip besar menjadi beberapa volume.

Pertanyaan yang sering diajukan

Apakah kompresor ukuran file zip sama dengan mengompres setiap file di dalam zip?

Alat Zip File menjalankan kompresi ZIP pada setiap file saat membundelnya. Untuk file yang sudah dalam format terkompresi (JPG, MP4, Office modern), sebagian besar penghematan datang dari pembungkus direktori, bukan tubuh file. Untuk file berbentuk teks (CSV, JSON, kode sumber, log), kompresi per-file itu sendiri adalah penghematan utama.

Apakah password ZIP akan menjaga konten tetap privat jika ada yang mencegat unduhan?

Password ZIP menambahkan lapisan yang memblokir ekstraksi kasual tanpa password. Ini bukan enkripsi end-to-end - host layanan freetoolonline melihat konten file selama pembundelan, dan unduhan bergerak melalui HTTPS. Untuk konten sensitif tinggi (laporan bank, rekam medis), enkripsi secara lokal sebelum mengunggah.

Berapa lama tautan unduhan berlangsung?

URL unduhan menunjuk ke salinan yang disimpan di host layanan freetoolonline untuk jendela rotasi singkat. Setelah rotasi berjalan, URL yang sama mengembalikan 404 bahkan jika Anda menandainya. Jalankan ulang kompres untuk mendapatkan URL baru.
