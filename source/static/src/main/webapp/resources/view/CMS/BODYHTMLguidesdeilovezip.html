Online-ZIP-Werkzeug (kostenlos, Browser-Build-Service)

Leser, die "i love zip" in eine Suchmaschine eingeben, meinen normalerweise eines von zwei Dingen: ein Online-Werkzeug, das eine .zip-Datei aus einem Ordner oder einer Mehrfachauswahl ohne Installation erstellt, oder einen schnellen Weg, eine Datei, die jemand gesendet hat, neu zu komprimieren / zu entpacken. Diese Anleitung verweist auf das richtige Ziel fuer jede Version der Aufgabe. Der Ersteller unter /zip-tools/zip-file.html erstellt neue .zip-Archive auf einem privaten Upload-und-Build-Service und gibt das Ergebnis als eine einzige herunterladbare Datei zurueck; Quelldateien werden nach einem kurzen Aufbewahrungsfenster automatisch vom Service geloescht. Fuer die umgekehrte Richtung (eine vorhandene .zip-Datei oeffnen) verwende /zip-tools/unzip-file.html; um eine bereits erstellte .zip-Datei zu schrumpfen, kuemmert sich /zip-tools/zip-file.html auch darum mit dem Schieberegler fuer Komprimierungsstufe im Einstellungs-Modal.

Waehle das richtige Ziel fuer das, was du tatsaechlich tun musst

Ein "Online-ZIP-Werkzeug" sind eigentlich drei Aufgaben, die einen Namen teilen: ein neues Archiv erstellen, ein vorhandenes oeffnen oder ein bereits vorhandenes schrumpfen. Der Erstellungsweg ist der haeufigste - du hast einen Ordner oder eine Reihe von Anhaengen und moechtest eine .zip-Datei, die du uebergeben kannst, ohne zuerst einen Desktop-Archiver zu installieren. Oeffne /zip-tools/zip-file.html, lege den Ordner in den Upload-Bereich, benenne das Archiv im Einstellungs-Modal, fuege optional ein Passwort hinzu, klicke auf Convert und lade das Ergebnis herunter. Der Oeffnungsweg fuehrt ueber /zip-tools/unzip-file.html: lege die .zip-Datei auf die Seite, und sie listet den Inhalt auf, damit du auswaehlen kannst, welche Dateien zurueck auf deine Maschine extrahiert werden sollen.

Was "online" hier bedeutet

"Online" auf dieser Seite bedeutet, dass der Ersteller auf einem privaten Build-Service laeuft, der von FreetoolOnline gehostet wird, nicht als heruntergeladenes Programm. Die Dateien verlassen dein Geraet ueber HTTPS fuer die wenigen Sekunden, die der Build dauert, dann werden die Quelldateien nach einem kurzen Aufbewahrungsfenster vom Service entfernt. Die .zip-Datei selbst wird zurueck an deinen Browser angeboten und nicht auf dem Service gespeichert, sobald der Download beginnt. Es gibt kein Konto und keine E-Mail-Einreichung. Der Trade-Off gegenueber einem rein browser-seitigen Archiver ist die Zuverlaessigkeit bei grossen Ordnern: der Build-Service handhabt Eingaben, die die Browser-Speicherlimits ueberschreiten, wo ein reiner JavaScript-Archiver abstuerzen oder haengen wuerde.
