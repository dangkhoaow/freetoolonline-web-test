Once the .zip arrives on the receiving end, recipients open it with the unzip already built into their OS - Windows users right-click and pick Extract All, macOS users double-click and the archive expands next to itself, Linux file managers expose an Extract option. No third-party install is needed for a standard ZIP. Recipients who would rather not run an install (a managed laptop that blocks third-party software, for example) can open the archive in the browser at /unzip-file.html without downloading any tool. The size-trade decision sat on the sender side; on the receiving end, the only requirement is that the archive is a plain .zip, which the creator at /zip-tools/zip-file.html always produces.

Name the archive in the Settings modal before downloading so the recipient sees the project or topic in the filename instead of the default myzipped-<date> stamp. A clear filename pays off twice: on the receiving end the .zip arrives with a self-describing name, and on your own machine the export-folder stays readable when several archives are built in the same session. The same Settings modal is where the password and encryption method are set, so the naming step and the locking step share one round trip.
