Beschaedigte Zip-Datei Wiederherstellen - Optionen

Zuletzt geprueft 2026-04-27. Browser-Werkzeuge koennen wiederherstellbare Teile einer beschaedigten Zip extrahieren; vollstaendige Reparatur erfordert normalerweise ein Desktop-Werkzeug.

30-Sekunden-Antwort. Eine Zip, die "beschaedigt" sagt, kann vier verschiedene Dinge bedeuten, mit vier verschiedenen Loesungen. Laden Sie die Datei erneut herunter - die meisten "beschaedigten" Zips sind unterbrochene Downloads. Wenn das fehlschlaegt, versuchen Sie eine Teilextraktion, die herauszieht, was lesbar ist. Wenn das fehlschlaegt, fuehren Sie ein Desktop-Reparaturwerkzeug aus. Wenn das fehlschlaegt, sind die Daten wahrscheinlich weg und der richtige Schritt ist, den Absender um erneutes Senden zu bitten.

Schritt 1 - erneut herunterladen

Die meisten "Zip beschaedigt"-Meldungen kommen von einem abgebrochenen Download. Der Browser hat die Uebertragung frueh gestoppt, oder ein CDN hat eine Teildatei geliefert. Loesungen, die in Sekunden vor jedem Reparaturwerkzeug funktionieren: leeren Sie den Browser-Cache und laden Sie erneut herunter; ziehen Sie die Datei mit einem anderen Werkzeug (curl auf Desktop, ein Download-Manager auf Mobil); oder bitten Sie um einen frischen Link, wenn das Original von einer Chat-App kam, in der Anhangs-URLs ablaufen.

Die Diagnose: fuehren Sie eine Pruefsumme auf der heruntergeladenen Zip aus und vergleichen Sie mit der des Absenders. Wenn sie sich unterscheiden, ist die Datei das Problem, nicht die Zip selbst.

Schritt 2 - Teilextraktion

Das zentrale Verzeichnis einer Zip sitzt am Ende der Datei. Wenn die Beschaedigung am Ende ist, ist jede Datei darin unerreichbar - auch wenn die Bytes noch da sind. Wenn die Beschaedigung in der Mitte ist, extrahieren Dateien vor dem beschaedigten Bereich noch.

Versuchen Sie eine Datei entpacken im Browser. Wenn es Inhalte auflistet, aber beim Extrahieren Fehler macht, ist das zentrale Verzeichnis intakt und ein oder mehrere Dateieintraege sind beschaedigt. Einige Entpackungswerkzeuge (7-Zip auf Windows, The Unarchiver auf macOS) koennen alles vor der Beschaedigung extrahieren und den Rest ueberspringen. Lohnt einen Versuch, wenn nur eine Teilmenge der Dateien wichtig ist.

Schritt 3 - Desktop-Reparaturwerkzeuge

Wenn die Teilextraktion fehlschlaegt, koennen Reparaturwerkzeuge das zentrale Verzeichnis durch Scannen nach Dateikoepfen im Koerper neu aufbauen. Zwei kostenlose Optionen:

  • 7-Zip auf Windows. Rechtsklick auf die beschaedigte Zip, waehle 7-Zip > Open. Wenn es den Koerper scannen kann, listet es, was es findet - sogar ohne zentrales Verzeichnis.
  • zip auf Linux/macOS. Der Befehl zip -FF broken.zip --out fixed.zip versucht, eine Zip mit beschaedigter Struktur zu reparieren. Funktioniert fuer viele Faelle, in denen die Datei mitten in einem Eintrag endet.

Browserbasierte Reparatur ist selten, weil die Algorithmen grosse Dateien scannen und auf lokaler Festplatte schneller laufen. Wenn Sie keinen Desktop verfuegbar haben, ist der richtige Weg normalerweise, den Absender um erneutes Senden zu bitten.

Schritt 4 - wann aufgeben

Wenn die Zip mit Verschluesselung erstellt wurde und die Beschaedigung irgendwo nahe den Dateikoepfen ist, sind die Inhalte wahrscheinlich nicht wiederherstellbar. Dasselbe, wenn der Speichertraeger (ein versagendes USB-Laufwerk, eine beschaedigte SD-Karte) die Quelle war - die Beschaedigung ist in den Bytes selbst, nicht in der Verpackung. Der richtige Schritt ist, den urspruenglichen Absender um erneutes Senden zu bitten oder aus einem Backup wiederherzustellen, wenn eines existiert.

Vorbeugung in der Zukunft

Fuer Dateien, die wichtig sind, senden Sie eine Pruefsumme neben der Zip. Der Empfaenger kann verifizieren, dass der Download Byte-perfekt ist, bevor er Zeit auf eine Reparatur verschwendet. MD5 oder SHA-256 funktionieren beide; SHA-256 ist die sicherere Wahl. Siehe warum MD5 nicht entschluesselt werden kann fuer den Sicherheitskontext. Fuer alles Zip-bezogene, siehe den Zip-Werkzeug-Hub.
