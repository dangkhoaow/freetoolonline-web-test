Every browser camera test on this site runs the same Web API under the hood: a call to navigator.mediaDevices.getUserMedia({ video: true }) asks the operating system for a live video stream, and the browser performs a permission check before any frame ever reaches the page. The blocked state you see in the address bar is not the page refusing to draw a preview - it is the user agent honouring a saved Block decision and short-circuiting the request before the camera light can come on. Clearing that decision is what lets the next call succeed.

If the preview activates but stays dark, the permission is not the cause; check the companion guide on camera test shows black screen: four fixes for the lens-cover, app-conflict, driver, and stale-state checks that resolve the black-frame case.
