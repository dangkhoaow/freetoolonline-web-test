Every browser camera test on this site runs the same Web API under the hood: a call to navigator.mediaDevices.getUserMedia({ video: true }) asks the operating system for a live video stream, and the browser performs a permission check before any frame ever reaches the page. The blocked state you see in the address bar is not the page refusing to draw a preview - it is the user agent honouring a saved Block decision and short-circuiting the request before the camera light can come on. Clearing that decision is what lets the next call succeed.

If the preview activates but stays dark, the permission is not the cause; check the companion guide on camera test shows black screen: four fixes for the lens-cover, app-conflict, driver, and stale-state checks that resolve the black-frame case.

A camera permission prompt resolves into one of three end states, and the address-bar icon is the quickest place to tell them apart. The allowed state activates the live preview within a second of the click, and the icon shows a plain camera glyph. The blocked or denied state silently rejects every future request from the same origin, and the icon shows a camera glyph with a red slash through it; opening the site permissions panel reveals a Block decision the browser will keep honouring until you clear it. The ignored state appears when the prompt opens but you click outside the dialog or close the tab before answering, and the browser treats the next call as if the prompt was never shown - the icon stays neutral and the next page load can ask again. The same address-bar glyph is also the fastest at-a-glance diagnostic on a page already loaded: a slashed icon means the origin is on the Block list and every future getUserMedia call will be short-circuited until you clear it, a plain icon means the live preview is allowed and the camera light should come on within a second, and no icon at all means either no recent request was made on this page or the browser is hiding the icon between requests so the next call has to wake the prompt back up.
