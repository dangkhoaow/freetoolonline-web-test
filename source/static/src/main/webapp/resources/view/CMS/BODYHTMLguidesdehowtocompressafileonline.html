Wie Man eine Datei Online Komprimiert (Schritt für Schritt)

Zuletzt geprüft: 2026-04-27. Jeder Schritt unten läuft in Ihrem Browser auf Ihrem eigenen Gerät. Die Datei, die Sie komprimieren, verlässt nie Ihren Computer oder Ihr Telefon.

30-Sekunden-Antwort. Identifizieren Sie zuerst den Dateityp, wählen Sie dann das passende Tool. JPG / PNG Foto: Compress Image bei Qualität 80 spart typischerweise 50-70%. HEIC Foto: HEIC to JPG - die Konvertierung ist der Größengewinn. PDF: Flatten PDF wenn Formulare / Anmerkungen die Datei aufgebläht haben. Einzelnes Video: bei niedrigerer Bitrate neu kodieren (Browser-Tools handhaben kurze Clips). Wenn Ihre Eingabe ein Ordner ist, keine einzelne Datei, springen Sie stattdessen zu wie man einen Ordner für E-Mail komprimiert.

Schritt 1 - identifizieren Sie den Dateityp

Die Suche "Datei online komprimieren" liefert generische Tools, die versuchen, jeden Typ mit demselben Algorithmus zu behandeln. Das funktioniert für einige Eingaben und unterperformt schlecht bei anderen. Schauen Sie zuerst auf die Erweiterung Ihrer Datei:

  • .jpg / .jpeg / .png -> Bildkomprimierungs-Pfad (Schritt 2A).
  • .heic / .heif -> zuerst in JPG konvertieren; die Konvertierung ist der größere Größengewinn als jeder Komprimierungslauf (Schritt 2B).
  • .pdf -> PDF-spezifischer Flatten- / Re-Save-Pfad (Schritt 2C).
  • .mp4 / .mov / .mkv -> bei niedrigerer Bitrate neu kodieren (Schritt 2D).
  • .docx / .xlsx / .pptx / .csv / .txt / .json / .xml / Quellcode -> ZIP-es-Pfad (Schritt 2E).
  • .mp3 / .wav / .flac -> Audio-Neukodierung (Schritt 2F).

Wenn Sie einen Ordner statt einer einzelnen Datei haben, ist ZIP die Antwort, weil ZIP sowohl bündelt als auch komprimiert in einem Schritt. Datei-Kompressor vs ZIP deckt Ordner-vs-Datei-Entscheidungen im Detail ab.

Schritt 2A - JPG / PNG Fotos

Öffnen Sie Compress Image, ziehen Sie Ihr Foto in die Drop-Zone, stellen Sie Qualität auf 80 (Standard), und laden Sie herunter. Ein 5 MB iPhone-JPG landet typischerweise bei 1-1,5 MB bei Qualität 80 ohne sichtbaren Unterschied, es sei denn Sie zoomen auf 200%. Wenn Sie ein kleineres Ergebnis brauchen, senken Sie die Qualität auf 70 - die Einsparungen werden darunter abnehmend.

PNG ist ein verlustfreies Format. Wenn Ihr "PNG-Foto" tatsächlich eine Fotografie ist, speichern Sie es stattdessen als JPG - der Größenschnitt ist viel größer. PNG sollte für Screenshots, UI-Mockups, Strichgrafik und jedes Bild, das Transparenz benötigt, reserviert sein. JPG vs PNG für das Web geht die Formatwahl durch.

Schritt 2B - HEIC Fotos

HEIC-Dateien vom iPhone sind bereits klein (HEVC-Kodierung), daher hilft Neukomprimierung selten. Das größere Problem ist Kompatibilität - HEIC öffnet nicht auf Windows, älteren Android, den meisten Browsern und den meisten E-Mail-Clients ohne kostenpflichtige Codec-Installation. Öffnen Sie HEIC to JPG, ziehen Sie die .heic-Datei hinein, laden Sie das .jpg herunter. Die JPG-Kopie hat ähnliche Größe oder ist etwas größer, öffnet aber überall. Für HEIC-Batch-Arbeit siehe wie man 100 HEIC-Fotos in JPG konvertiert.

Schritt 2C - PDF-Dateien

PDFs kommen in zwei Geschmacksrichtungen. Ein gescanntes Dokument-PDF (groß, weil jede Seite ein hochauflösendes Bild ist) komprimiert durch erneutes Speichern bei niedrigerer Bildqualität - dieser Pfad benötigt dedizierte PDF-Software außerhalb des Bereichs von Browser-Tools. Ein Text-PDF, das durch Formularfelder, Anmerkungen oder Revisionsverlauf aufgebläht wurde, komprimiert gut via Flatten PDF - Abflachen entfernt die bearbeitbaren Schichten, lässt ein 12 MB Formular-PDF auf 2-3 MB fallen. Wenn Sie einen Stapel kleiner PDFs an eine Person senden müssen, ist das Zusammenführen zu einem oft das Ziel statt jedes einzeln zu komprimieren - verwenden Sie Compose PDF.

Für mehr PDF-Routing siehe die PDF-Bearbeitungs-Leiter.

Schritt 2D - Video-Dateien

Reine Browser-Video-Komprimierung ist für kurze Clips machbar (unter 60 Sekunden bei 1080p). Öffnen Sie Video Converter, stellen Sie die Ausgabe-Bitrate auf 1500 kbps für 1080p oder 800 kbps für 720p, und kodieren Sie neu. Ein 60 MB Telefon-Clip landet typischerweise bei 8-15 MB. Für längere Videos oder 4K-Quelle werden Browser-Speicherlimits ein Problem; verwenden Sie stattdessen FFmpeg lokal - FFmpeg online vs lokales FFmpeg deckt den Grenzfall ab.

Schritt 2E - Dokumente und Quellcode

Das ist der Pfad, den die meisten "kostenlosen Datei-Kompressor"-Ergebnisse verpassen. Ein 30-seitiges DOCX, ein 200-Zeilen-CSV, eine 4.000-Zeilen-Quellcode-Datei - all diese haben enorme Komprimierungsverhältnisse unter DEFLATE. Öffnen Sie Zip File, lassen Sie die Datei fallen, laden Sie die .zip herunter. Ein 12 MB DOCX mit eingebetteten Screenshots landet typischerweise bei 4-5 MB. Klartext-Quellcode kann um 70-80% schrumpfen. Überspringen Sie diesen Schritt nicht, wenn Ihre Datei textlastig ist; es ist ein empfohlener Größenschnitt verfügbar ohne Treueverlust.

Schritt 2F - Audio-Dateien

Browser-ZIP hilft nicht, weil Audio-Formate bereits komprimiert sind. Der richtige Pfad ist Formatkonvertierung - WAV zu MP3 bei 192 kbps, oder FLAC zu OGG. Browser-Tools für Audio-Neukodierung sind begrenzt; dedizierte Software (Audacity, fre:ac) ist schneller für diesen Fall. Die Ausnahme: wenn Sie ein einzelnes 5 MB WAV haben, das in ein 2 MB Upload-Limit passen muss, ist Online-Konvertierung in Ordnung.

Schritt 3 - Überprüfen Sie die Ausgabe

Nach der Komprimierung prüfen Sie vor dem Senden drei Dinge:

  • Öffnen Sie die komprimierte Datei einmal. Bildkompressoren können selten ein korruptes JPG produzieren, aber ein korruptes PDF oder Video sieht im Datei-Manager gut aus und versagt erst, wenn der Empfänger es öffnet.
  • Überprüfen Sie die Größe gegen das Limit, das Sie brauchten. E-Mail-Caps sind typischerweise 25 MB Anhang, manchmal 10 MB. Einige Web-Formulare limitieren einzelne Felder auf 5 MB.
  • Wenn die Datei zu einem Vertrags- / Rechts- / Banking-System geht, behalten Sie das Original. Einige Empfänger lehnen verlustbehaftet-komprimierte Dateien ab, weil sie das bit-genau Original akzeptieren müssen.

Was wenn das Ergebnis immer noch zu groß ist?

Wenn Ihre komprimierte Datei immer noch über dem Limit ist, funktionieren normalerweise zwei Optionen. Format konvertieren: viele "komprimierte" Dateien sind immer noch im falschen Format. PNG-Foto zu JPG kann 90% Größenschnitt geben, sogar nach einer JPG-Komprimierung. Verwenden Sie einen Transfer-Link: Drive, Dropbox, WeTransfer geben Ihnen alle eine URL zum Senden statt eines Anhangs. Der Empfänger lädt die Originaldatei in voller Treue herunter. Für Dateien größer als 25 MB ist dies fast immer die richtige Antwort.

Wenn Ihre Datei eigentlich ein Ordner von Dateien sein sollte, ist ZIP die Antwort (ein ZIP, alle Dateien drin). Siehe wie man einen Ordner für E-Mail komprimiert. Wenn Sie bereits ein ZIP haben, das zu groß ist, siehe wie man ZIP-Dateigröße online reduziert.

Datenschutz

Jedes Tool, das in dieser Anleitung verlinkt ist, läuft in Ihrem Browser auf Ihrem eigenen Gerät. Die Datei, die Sie komprimieren, wird nie auf einen Server hochgeladen. Das ist wichtig, wenn Ihre Datei etwas Persönliches, Finanzielles, Medizinisches oder Arbeits-Vertrauliches enthält.
