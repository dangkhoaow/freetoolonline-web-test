FFmpeg in the browser: how the WebAssembly build handles your file

FFmpeg Online runs ffmpeg.wasm, a WebAssembly build of the same FFmpeg binary that powers most professional video pipelines. The browser loads the WASM module once per session, mounts your uploaded file into the virtual filesystem, executes the command you type, and writes the output back out for download. Because the work runs on your CPU via the browser, the file never leaves your device; the trade-off is that encoding is CPU-bound and slower than a native installation, so long videos or large resolutions (4K, 8K) are best handled in short segments or offloaded to a local FFmpeg install.


Command patterns that cover most jobs

Format conversion (MP4 → WebM). -i input.mp4 -c:v libvpx-vp9 -b:v 1M -c:a libopus output.webm - re-encodes to VP9 video and Opus audio for modern browsers.

Trim a segment (keep seconds 10–30). -i input.mp4 -ss 10 -t 20 -c copy output.mp4 - -c copy avoids a re-encode, so the trim is instant.

Extract audio only (MP4 → MP3). -i input.mp4 -vn -c:a libmp3lame -b:a 192k output.mp3.

Resize to 720p. -i input.mp4 -vf scale=1280:720 -c:v libx264 -crf 23 output.mp4 - bicubic scale + quality-targeted H.264 re-encode.

Probe a file before editing. Switch the binary to ffprobe and pass -v quiet -print_format json -show_format -show_streams input.mp4 to get codec, bitrate, duration, and stream metadata as JSON.

Remember: the forward-slash is reserved by this tool’s command parser, so express paths without leading slashes and avoid the / separator in filter chains.


Privacy and size limits

Nothing is uploaded. The file is parsed in-browser, encoded in-browser, and the output download is generated on the page. When you close the tab, both the input and the output are discarded from memory. The practical upper limit is what your device’s RAM can hold: expect comfortable operation up to ~500 MB source files on a modern laptop, with progressively slower runs as the file grows. For long archival re-encodes, run FFmpeg locally; for one-off trim, extract, and format change jobs, the browser path is the fastest no-install route.

Browser ffmpeg vs local ffmpeg - which one now? The guide ffmpeg online vs local ffmpeg when each wins covers the decision rule: in-browser is ideal for one-off conversions under ~200 MB and anything privacy-sensitive; local ffmpeg wins for long-running batches, hardware-accelerated encoding, and files larger than browser memory allows.
