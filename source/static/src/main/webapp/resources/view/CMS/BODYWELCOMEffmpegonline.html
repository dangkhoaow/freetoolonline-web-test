How the server-side FFmpeg pipeline handles your file

FFmpeg Online uploads your file over HTTPS to a hosted processing service, dispatches your ffmpeg or ffprobe command to a server-side worker, and returns the output as a one-click download. The browser holds the upload form and the command builder; the encoding, transmuxing, or metadata probe runs on the service host, so your device is not CPU-bound. For workflows that must keep the source on-device, run a local FFmpeg install instead.


Command patterns that cover most jobs

Format conversion (MP4 → WebM). -i input.mp4 -c:v libvpx-vp9 -b:v 1M -c:a libopus output.webm - re-encodes to VP9 video and Opus audio for modern browsers.

Trim a segment (keep seconds 10-30). -i input.mp4 -ss 10 -t 20 -c copy output.mp4 - -c copy avoids a re-encode, so the trim is instant.

Extract audio only (MP4 → MP3). -i input.mp4 -vn -c:a libmp3lame -b:a 192k output.mp3.

Resize to 720p. -i input.mp4 -vf scale=1280:720 -c:v libx264 -crf 23 output.mp4 - bicubic scale + quality-targeted H.264 re-encode.

Probe a file before editing. Switch the binary to ffprobe and pass -v quiet -print_format json -show_format -show_streams input.mp4 to get codec, bitrate, duration, and stream metadata as JSON.

Remember: the forward-slash is reserved by this tool's command parser, so express paths without leading slashes and avoid the / separator in filter chains.


FFmpeg vs FFprobe - pick the right binary first

The first dropdown at the top of the command builder switches the binary between ffmpeg and ffprobe; pick the right one before you type any parameters because the two binaries answer different questions. FFmpeg is the encoder and transmuxer - use it when you want to re-encode (MP4 to WebM), trim by seconds, resize to 720p, or extract an audio track. FFprobe is the read-only inspector - use it when you want the codec, bitrate, duration, frame rate, and stream list as JSON without producing an output file (handy when you are debugging "why does this file not play?"). The two share the same command-parameters input shape and the same forward-slash restriction, but only FFmpeg produces a downloadable result; FFprobe writes its findings to the console output area.

Your file is uploaded over HTTPS to service.us-east-1a.freetool.online and stored on the processing host only for as long as the conversion needs it; the source and the generated output are removed shortly after the command finishes (the retention window is the value the service emits as web.minsToDelFile at page load). The session is anonymous - no signup or sign-in is required. For workflows that must keep the source on-device for compliance, run FFmpeg locally; the hosted service is the fastest no-install route for one-off conversions, trims, and metadata probes.
