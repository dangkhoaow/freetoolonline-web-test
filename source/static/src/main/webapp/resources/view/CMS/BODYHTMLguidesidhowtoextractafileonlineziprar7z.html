Cara mengekstrak file online - ZIP, RAR, 7z (rute mana di situs ini melakukan apa)

Tinjauan terakhir 2026-05-04. Aturan perutean langsung untuk kueri pencarian "ekstrak file online" / "file zipper" / "ekstraktor file online". Arah terbalik dari panduan perutean file-compressor siklus-51: alih-alih "saya punya folder, saya ingin men-zip-nya", Anda memiliki arsip .zip / .rar / .7z dan ingin file di dalamnya kembali di browser Anda. Jawaban jujur tergantung pada ekstensi file - dan salah satu dari tiga format yang tidak diekstrak situs ini di sisi server. Panduan menyebutkan rute mana melakukan apa, dan ke mana pergi untuk format yang situs ini tidak menangani.

Jawaban 30 detik. Jika arsip Anda adalah .zip, buka Unzip File - jatuhkan file, ia diunggah ke layanan AWS freetoolonline, dan kontennya kembali sebagai unduhan. Jika .zip membutuhkan kata sandi, alat yang sama menerima input kata sandi pada alur yang sama. Jika arsip Anda adalah .rar atau .7z, situs ini tidak memiliki ekstraktor sisi server untuk format-format itu - rute yang tepat adalah aplikasi OS bawaan atau aplikasi lokal (7-Zip, Keka, p7zip, unrar) dan panduan menjelaskan mengapa kejujuran penting di sini. Jika Anda tidak dapat mengingat kata sandi ke .zip, Remove ZIP Password adalah operasi berbeda - dibahas di aturan 4.

Mengapa "ekstrak file online" biasanya berarti salah satu dari tiga operasi

Frasa "ekstrak file online" adalah satu kueri pencarian untuk tiga tugas-pembaca yang berbeda. Pertama adalah "saya punya .zip dan saya ingin file di dalamnya" - arah-terbalik rutin dari zip. Kedua adalah "saya punya .rar atau .7z dan saya ingin file di dalamnya" - tujuan sama, kontainer arsip berbeda. Ketiga adalah "saya lupa kata sandi ke .zip dan saya tidak dapat membukanya" - yang sama sekali bukan ekstraksi tetapi pemulihan kata sandi. Memilih alat yang tepat tergantung pada di mana dari ketiga itu Anda berada, dan di situs ini jawabannya berbeda berdasarkan ekstensi.

Alasannya penting: meletakkan .rar ke dalam alat yang hanya menerima .zip tidak mengembalikan pesan kesalahan yang dianggap berguna oleh sebagian besar pembaca - ia mengembalikan "tipe file salah" atau diam-diam menolak unggahan. Dan waktu yang terbuang mengunggah ulang file yang sama tidak mengubah jawaban. Empat aturan perutean di bawah mencakup kasus nyata, termasuk apa yang harus dilakukan ketika situs ini tidak memiliki ekstraktor untuk format Anda - karena lebih berguna membaca "buka 7-Zip atau Keka secara lokal" daripada terus mengklik alat yang tidak menangani format.

Aturan perutean 1 - arsip Anda adalah .zip: buka Unzip File (server-upload, kata sandi opsional)

Jika arsip pada disk Anda memiliki ekstensi .zip, Unzip File adalah rute. Jatuhkan file ke widget unggah, layanan AWS freetoolonline meng-unzip arsip di sisi server, dan kontennya kembali sebagai unduhan. Sinyal konkret bahwa Anda berada di kasus ini:

  • Anda menerima .zip sebagai lampiran email dan ponsel atau laptop kerja Anda tidak memiliki alat unzip desktop yang siap pakai. Alat browser mengembalikan kontennya dalam satu perjalanan pulang-pergi.
  • .zip berisi banyak file (folder proyek, batch tanda terima, folder foto) dan Anda ingin memeriksa kontennya sebelum memutuskan apa yang akan disimpan. Unzip File mengembalikan file-file yang terkandung; Anda menyimpan yang Anda inginkan.
  • .zip dilindungi kata sandi dan Anda tahu kata sandinya. Unzip File mengekspos input kata sandi pada alur yang sama - masukkan ketika diminta dan ekstraksi berlanjut. Panduan pendamping untuk arah terbalik (men-zip folder dengan kata sandi) adalah Cara men-zip folder online langkah demi langkah.

Alasan: Unzip File dibangun khusus untuk tugas-pembaca ekstraksi .zip. Filter file yang diterima pada widget unggah hanya .zip - ini didokumentasikan dalam sumber alat, dan panduan membuat batas eksplisit sehingga Anda tidak membuang unggahan pada file yang tidak dapat ditangani rute. Jika file Anda adalah varian .zip (.zipx) atau .zip yang dibungkus dalam arsip lain, lihat aturan 3.

Aturan perutean 2 - arsip Anda adalah .rar atau .7z: situs ini tidak mengekstrak itu di sisi server

Jika arsip pada disk Anda memiliki ekstensi .rar atau .7z, situs ini tidak memiliki ekstraktor sisi server untuk format-format itu. Tidak ada rute /unrar.html atau /un7z.html, dan filter file yang diterima Unzip File hanya .zip - mengunggah .rar atau .7z ke rute itu tidak menghasilkan hasil yang berguna. Jawaban jujur adalah: buka arsip secara lokal. Rute konkret berdasarkan sistem operasi:

  • Windows: 7-Zip (gratis, open-source, menangani .zip / .rar / .7z), WinRAR (.rar-native, trial gratis), atau PeaZip (gratis, dukungan format luas). Klik kanan arsip, "Extract to a folder", selesai.
  • macOS: Keka (gratis / berbayar, App Store) atau The Unarchiver (gratis, App Store) menangani .zip / .rar / .7z. Archive Utility bawaan macOS membuka .zip; untuk .rar / .7z Anda membutuhkan salah satu aplikasi di atas.
  • Linux: 7z x archive.7z (p7zip; apt install p7zip-full) untuk .7z, unrar x archive.rar (unrar; apt install unrar) untuk .rar.

Alasan: menulis "kami mengekstrak RAR online" ketika situs tidak memiliki ekstraktor .rar adalah jenis klaim palsu yang membuang waktu pembaca dan menghancurkan kepercayaan. Ketika rute online tidak ada untuk format, fallback alat-lokal adalah jawaban yang tepat, dan panduan lebih berguna ketika mengatakannya langsung. Perbandingan pendamping ZIP online vs 7z vs RAR - mana yang dipilih menjalani format mana yang dipilih ketika Anda mengendalikan sisi produksi; panduan ini untuk sisi konsumsi (Anda sudah memiliki arsip di depan Anda).

Aturan perutean 3 - ekstensi campuran atau tidak jelas: pilih berdasarkan apa file sebenarnya

Kasus bersih di atas mencakup sebagian besar kueri pencarian. Kasus campuran membutuhkan satu pertanyaan tambahan - apa file sebenarnya? Sub-kasus konkret:

  • Ekstensi mengatakan .zip tetapi Unzip File menolaknya: file salah diberi label (seseorang mengganti nama .rar menjadi .zip) atau .zip cacat. Alur diagnostik ada di Opsi memulihkan file ZIP rusak - yang meletakkan tiga ember (benar-benar rusak, salah diberi label, kata-sandi-diperlukan-tetapi-prompt-tidak-dipicu).
  • File tidak memiliki ekstensi, atau memiliki .tar / .gz / .tar.gz / .bz2: situs ini tidak memiliki rute TAR / GZIP. Fallback alat-lokal dari aturan 2 berlaku - 7-Zip menangani .tar.gz di Windows; tar -xzvf archive.tar.gz di macOS / Linux membuka kedua lapisan dalam satu langkah.
  • File adalah .zipx (varian WinZip): sebagian besar alat unzip desktop menangani .zipx (7-Zip, WinRAR, Keka) tetapi Unzip File situs ini menerima hanya filter .zip standar - coba sekali; jika menolak unggahan, kembali ke alat lokal.
  • "Arsip" sebenarnya adalah penginstal (.dmg, .pkg, .msi, .exe): ini bukan arsip dalam arti yang sama. Jangan unggah ke Unzip File. Buka dengan penginstal native mereka atau gunakan alat desktop yang mengetahui format spesifik.

Alasan: kueri pencarian "ekstrak file online" menyapu format arsip yang situs ini tidak ekstrak. Memilih berdasarkan apa file sebenarnya (alih-alih berdasarkan apa ekstensi mengatakan) menghindari loop unggah-dan-gagal. Panduan pendamping ZIP vs ZIPx vs RAR vs 7z format arsip dijelaskan adalah penjelas-format; panduan ini adalah aturan perutean operasional.

Aturan perutean 4 - Anda lupa kata sandi .zip: itu alat yang berbeda

Jika .zip Anda dilindungi kata sandi dan Anda tidak tahu kata sandinya, jangan terus mencoba membukanya di Unzip File - input kata sandi untuk memasukkan kata sandi yang diketahui, bukan untuk menebak. Rute terdekat di situs ini adalah Remove ZIP Password, yang adalah operasi berbeda: ia mencoba memulihkan atau menghapus perlindungan kata sandi dari .zip. Tiga peringatan layak dikatakan langsung:

  • Pemulihan tidak dijamin. Kata sandi kuat dengan enkripsi ZIP AES-256 modern dirancang agar sulit dipecahkan; alat mungkin mengembalikan hasil "tidak dapat memulihkan" pada kata sandi yang sengaja kuat.
  • Konten sensitif tidak boleh diunggah. Batas privasi yang sama yang berlaku untuk Unzip File berlaku di sini - file diunggah ke layanan AWS freetoolonline dan diproses di sisi server. Jika .zip berisi materi yang tidak dapat Anda unggah (medis, hukum, NDA), fallback alat-lokal dari aturan 2 ditambah alat lokal pemulihan kata sandi adalah rute yang tepat, dan aturan 5 di bawah menyebutkan dua opsi desktop.
  • Panduan pendamping Tipe kata sandi PDF - owner vs user mencakup distingsi terkait untuk PDF (kata sandi owner-set vs user-set); untuk .zip distingsinya lebih sederhana - kata sandi ditetapkan atau tidak.

Alasan: memulihkan kata sandi yang dilupakan adalah tugas-pembaca berbeda dari "saya punya arsip, saya ingin kontennya". Menyebutkan batas menjaga aturan perutean tetap bersih: Unzip File untuk ZIP kata-sandi-diketahui dan tanpa-kata-sandi, Remove ZIP Password untuk ZIP kata-sandi-dilupakan, alat lokal untuk semua yang lain. Untuk set lengkap alat ZIP di situs ini, lihat ZIP Tools.

Aturan perutean 5 - file sangat besar atau sensitif: ekstrak secara lokal alih-alih mengunggah

Unzip File mengunggah arsip Anda ke layanan AWS freetoolonline dan ekstraksi berjalan di sisi server. Itu adalah jalur yang tepat untuk hampir setiap tugas-pembaca alat browser, tetapi dua kasus meminta fallback alat-lokal. Pertama, arsip yang sangat besar (arsip proyek multi-gigabyte, backup .zip dalam rentang gigabyte) lambat untuk mengunggah dan lambat untuk mengunduh bahkan pada koneksi yang baik - jika Anda mengendalikan mesin desktop, ekstraktor OS bawaan atau aplikasi lokal (aturan 2) lebih cepat ujung-ke-ujung karena byte tidak pernah meninggalkan disk Anda. Kedua, file yang tidak dapat Anda unggah karena alasan kerahasiaan (pencitraan medis, bukti hukum, kode di bawah NDA) perlu tetap lokal - bahkan ketika file kecil, alat lokal adalah rute yang tepat.

Trade-off memiliki bentuk yang sama dengan keputusan setara untuk mengompresi file (Kompresor file online: ZIP folder vs Kompres gambar) dan mengkonversi file video / gambar (FFmpeg online vs FFmpeg lokal - kapan masing-masing menang, ImageMagick online vs alat spesifik-tugas). Batasnya sama: rute online menang dalam kenyamanan untuk input ukuran normal, non-sensitif; rute lokal menang dalam kecepatan dan privasi untuk yang besar atau sensitif. Untuk ekstraksi .zip panduan kompresor-file siklus-51 menyebutkan batas yang sama di arah terbalik (kompres, bukan ekstrak), dan jawaban sitewide lengkap untuk pertanyaan "tanpa unggah" ada di Alat online gratis yang bekerja tanpa mengunggah file - Unzip File tidak ada dalam daftar itu, tepatnya karena langkah ekstraksi berjalan di server.

Apa yang terjadi pada arsip Anda setelah Anda mengunggahnya (dan kapan melewati unggahan)

Ketika Anda menjatuhkan .zip ke Unzip File, arsip diunggah ke layanan AWS freetoolonline (wilayah us-east-1), di-unzip di sisi server, dan kontennya ditulis kembali ke bucket output pipeline unggah yang sama sehingga browser Anda dapat mengunduhnya. Arsip dan output yang diekstrak tidak disimpan tanpa batas - pipeline unggah menjalankan pembersihan terjadwal yang menghapus file setelah jendela retensi tetap (dikonfigurasi sitewide; dalam urutan puluhan menit, bukan jam atau hari). Ini adalah jendela retensi yang sama yang digunakan setiap alat berbasis unggah lainnya di situs ini (Compress Image, Zip File, HEIC to JPG dan seterusnya). Dua konsekuensi praktis untuk Anda sebagai pembaca:

  • Unduh output Anda sebelum menutup tab. Setelah jendela pembersihan lewat, upload-id yang sama tidak lagi menyelesaikan ke file yang dapat diunduh - jika Anda menutup tab tanpa menyimpan, Anda mengunggah ulang dari salinan lokal arsip Anda. Bookmark URL hasil bukan solusi; jendela retensi berlaku untuk file, bukan URL.
  • Untuk arsip yang sangat besar atau sensitif kerahasiaan, lewati unggahan sepenuhnya. Arsip multi-gigabyte lebih lambat untuk diunggah daripada diekstrak secara lokal, dan konten yang tidak dapat Anda unggah (medis, hukum, NDA) perlu tetap di disk. Fallback OS lokal dari aturan 2 adalah rute yang tepat dalam kedua kasus - 7-Zip, Keka, atau perintah platform 7z x / unrar x mengekstrak arsip tanpa mengirim byte ke server, dan tidak ada jendela retensi untuk direncanakan.

Batas identik dengan trade-off unggah di Kompresor file online: ZIP folder vs Kompres gambar (aturan 4 dalam panduan itu) dan jawaban privasi arah-terbalik di Alat online gratis yang bekerja tanpa mengunggah file - Unzip File TIDAK ada dalam daftar itu, tepatnya karena langkah ekstraksi menggunakan pipeline server-upload. Jika batas unggah adalah faktor penentu untuk file Anda, fallback alat-lokal dari aturan 2 adalah rute untuk diikuti.

Jika unggahan macet atau output salah: troubleshoot sebelum menjalankan ulang

Jika Unzip File menerima unggahan Anda tetapi tidak ada unduhan yang muncul dalam beberapa menit, jangan unggah ulang. Mengunggah ulang file yang sama tidak memperbaiki penyebab dasar - itu mengantrekan pekerjaan kedua yang menabrak tembok yang sama. Kemacetan umum adalah: batas ukuran input dicapai pada unggahan (arsip lebih besar dari tutup per-file; aturan 5 berlaku), tab browser disegarkan selama pipeline konversi (hasil mengunduh ke tab yang ditutup dan hilang), atau input non-.zip diberikan ke filter hanya-.zip (aturan 1 vs aturan 2 adalah panggilan yang salah - baca ulang ekstensi dan rute sesuai).

Jika output dalam bentuk yang salah - misalnya .zip diekstrak tetapi file di dalamnya sendiri adalah arsip - itu adalah kasus arsip bersarang: ekstrak .zip luar dulu, lalu rute setiap file dalam berdasarkan ekstensinya sendiri (.rar di dalam .zip masih .rar dan aturan 2 berlaku). Jika penerima tidak dapat membuka .zip yang Anda hasilkan dari arah terbalik (men-zip folder), mereka mungkin membutuhkan Unzip File di browser mereka, dan jika kata sandi menghalangi Remove ZIP Password mencakup alur pemulihan. Untuk semua yang lain - satu operasi umum, rute yang tepat berdasarkan ekstensi, fallback yang tepat untuk file yang besar atau sensitif - jawabannya membutuhkan 30 detik; jawaban yang salah membutuhkan satu menit lagi waktu unggah yang Anda tidak perlu habiskan.
