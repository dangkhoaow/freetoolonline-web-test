Coming from a HEIC source instead of an SVG?

SVG and HEIC are different starting points: SVG is a vector path that renders to PNG at any resolution you ask for, while HEIC is an iPhone photo container that needs a pixel-level decode before it can ship as a JPG. If the file in front of you is an iPhone HEIC and the destination wants JPG, the right tool is HEIC to JPG - the upload and download flow is the same as this page, the difference is the format reader on the service side. The decision rule: pick HEIC to JPG when the source extension is .heic or .heif, pick this page when the source is .svg.
