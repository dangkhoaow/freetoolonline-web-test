Como Compactar um Arquivo Online (Passo a Passo)

Última revisão: 2026-04-27. Cada passo abaixo roda no seu navegador no seu próprio dispositivo. O arquivo que você compacta nunca deixa seu computador ou telefone.

Resposta de 30 segundos. Identifique o tipo de arquivo primeiro, depois escolha a ferramenta correspondente. Foto JPG / PNG: Compress Image em qualidade 80 tipicamente economiza 50-70%. Foto HEIC: HEIC to JPG - a conversão é a vitória de tamanho. PDF: Flatten PDF quando formulários / anotações incharam o arquivo. Vídeo único: re-codifique em bitrate mais baixo (ferramentas de navegador lidam com clipes curtos). Documento / planilha / arquivo de código fonte: ZIP it - DEFLATE corta arquivos com muito texto em 50-80%. Se sua entrada é uma pasta, não um único arquivo, pule para como compactar uma pasta para email em vez disso.

Passo 1 - identifique o tipo de arquivo

A busca "compactar um arquivo online" retorna ferramentas genéricas que tentam lidar com cada tipo com o mesmo algoritmo. Isso funciona para algumas entradas e tem desempenho ruim em outras. Olhe a extensão do seu arquivo primeiro:

  • .jpg / .jpeg / .png -> caminho de compressão de imagem (Passo 2A).
  • .heic / .heif -> converter para JPG primeiro; a conversão é a maior vitória de tamanho do que qualquer execução de compressão (Passo 2B).
  • .pdf -> caminho específico de PDF flatten / re-save (Passo 2C).
  • .mp4 / .mov / .mkv -> re-codificar em bitrate mais baixo (Passo 2D).
  • .docx / .xlsx / .pptx / .csv / .txt / .json / .xml / código fonte -> caminho ZIP-it (Passo 2E).
  • .mp3 / .wav / .flac -> re-codificação de áudio (Passo 2F).

Se você tem uma pasta em vez de um único arquivo, ZIP é a resposta porque ZIP tanto empacota quanto comprime em um passo. Compressor de arquivo vs ZIP cobre decisões pasta-vs-arquivo em detalhe.

Passo 2A - Fotos JPG / PNG

Abra Compress Image, arraste sua foto para a zona de drop, defina qualidade para 80 (o padrão), e baixe. Uma foto JPG de 5 MB do iPhone tipicamente termina em 1-1.5 MB em qualidade 80 sem diferença visível a menos que você dê zoom em 200%. Se você precisa de um resultado menor, abaixe qualidade para 70 - as economias são retorno decrescente abaixo disso.

PNG é um formato sem perdas. Se sua "foto PNG" é na verdade uma fotografia, salve-a como JPG em vez disso - a economia de tamanho é muito maior. PNG deve ser reservado para screenshots, mockups de UI, line art, e qualquer imagem que precisa de transparência. JPG vs PNG para a web percorre a escolha de formato.

Passo 2B - Fotos HEIC

Arquivos HEIC do iPhone já são pequenos (codificação HEVC), então re-compressão raramente ajuda. O maior problema é compatibilidade - HEIC não abre no Windows, Android mais antigo, maioria dos navegadores, e maioria dos clientes de email sem instalação de codec pago. Abra HEIC to JPG, arraste o arquivo .heic, baixe o .jpg. A cópia JPG é de tamanho similar ou ligeiramente maior mas abre em todo lugar. Para trabalho em lote de HEIC veja como converter 100 fotos HEIC para JPG.

Passo 2C - Arquivos PDF

PDFs vêm em duas variedades. Um PDF de documento escaneado (grande porque cada página é uma imagem de alta resolução) comprime re-salvando em qualidade de imagem mais baixa - esse caminho precisa de software PDF dedicado fora do escopo de ferramentas de navegador. Um PDF de texto que foi inchado por campos de formulário, anotações, ou histórico de revisão comprime bem via Flatten PDF - achatar remove as camadas editáveis, derrubando um PDF de formulário de 12 MB para 2-3 MB. Se você tem uma pilha de PDFs pequenos para enviar a uma pessoa, mesclá-los em um é frequentemente o objetivo em vez de comprimir cada individualmente - use Compose PDF.

Para mais roteamento de PDF veja a escada de edição de PDF.

Passo 2D - Arquivos de vídeo

Compressão de vídeo apenas no navegador é viável para clipes curtos (abaixo de 60 segundos em 1080p). Abra Video Converter, defina bitrate de saída para 1500 kbps para 1080p ou 800 kbps para 720p, e re-codifique. Um clipe de telefone de 60 MB tipicamente termina em 8-15 MB. Para vídeos mais longos ou fonte 4K, limites de memória do navegador se tornam um problema; use FFmpeg localmente em vez disso - FFmpeg online vs FFmpeg local cobre o caso de fronteira.

Passo 2E - Documentos e código fonte

Este é o caminho que a maioria dos resultados de "compressor de arquivo grátis" perde. Um DOCX de 30 páginas, um CSV de 200 linhas, um arquivo de código fonte de 4.000 linhas - todos estes têm enormes proporções de compressão sob DEFLATE. Abra Zip File, solte o arquivo, baixe o .zip. Um DOCX de 12 MB com screenshots embutidos tipicamente termina em 4-5 MB. Código fonte em texto puro pode encolher 70-80%. Não pule este passo se seu arquivo tem muito texto; é um corte de tamanho recomendado disponível sem perder fidelidade.

Passo 2F - Arquivos de áudio

ZIP de navegador não ajuda porque formatos de áudio já estão comprimidos. O caminho certo é conversão de formato - WAV para MP3 em 192 kbps, ou FLAC para OGG. Ferramentas de navegador para re-codificação de áudio são limitadas; software dedicado (Audacity, fre:ac) é mais rápido para este caso. A exceção: se você tem um único WAV de 5 MB que precisa caber em um limite de upload de 2 MB, conversão online está bem.

Passo 3 - verifique a saída

Após compressão, verifique três coisas antes de enviar:

  • Abra o arquivo comprimido uma vez. Compressores de imagem raramente podem produzir um JPG corrompido, mas um PDF ou vídeo corrompido parece bom no gerenciador de arquivos e só falha quando o destinatário o abre.
  • Verifique o tamanho contra o limite que você precisava. Limites de email são tipicamente anexo de 25 MB, às vezes 10 MB. Alguns formulários web limitam campos individuais a 5 MB.
  • Se o arquivo está indo para um sistema de contrato / legal / bancário, mantenha o original. Alguns destinatários rejeitam arquivos comprimidos com perdas porque devem aceitar o original bit-exato.

E se o resultado ainda for muito grande?

Se seu arquivo comprimido ainda está acima do limite, duas opções geralmente funcionam. Converter formato: muitos arquivos "comprimidos" ainda estão no formato errado. Foto PNG para JPG pode dar 90% de corte de tamanho, mesmo após uma compressão JPG. Use um link de transferência: Drive, Dropbox, WeTransfer todos dão a você uma URL para enviar em vez de um anexo. O destinatário baixa o arquivo original em fidelidade total. Para arquivos maiores que 25 MB esta é quase sempre a resposta certa.

Se seu arquivo era para ser uma pasta de arquivos, ZIP é a resposta (um ZIP, todos os arquivos dentro). Veja como compactar uma pasta para email. Se você já tem um ZIP que é muito grande, veja como reduzir tamanho de arquivo zip online.

Privacidade

Cada ferramenta ligada neste guia roda no seu navegador no seu próprio dispositivo. O arquivo que você compacta nunca é enviado para um servidor. Isso importa se seu arquivo contém algo pessoal, financeiro, médico, ou confidencial de trabalho. Para fundo veja ferramentas online gratuitas que funcionam sem subir arquivos.
