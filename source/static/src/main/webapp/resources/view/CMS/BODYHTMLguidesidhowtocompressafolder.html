Cara mengkompres folder menjadi satu ZIP

Diterbitkan 30 April 2026

Folder penuh file menjadi satu arsip yang lebih kecil dan dapat dikirim setelah dikompres menjadi ZIP. ZIP adalah format kontainer yang kompatibel secara universal - Windows, macOS, Linux, iOS, dan Android semuanya membukanya secara native, tidak diperlukan plugin - dan alat yang tepat mengubah seluruh pekerjaan menjadi alur browser tiga langkah. Panduan ini berjalan melalui langkah-langkah, trade-off yang mempengaruhi seberapa kecil hasilnya, dan skenario paling umum yang ditemui orang.


Alur tiga langkah

Langkah 1. Buka Kompres, File dan Folder ZIP. Alat ini berjalan sepenuhnya di browser Anda dan di layanan kami - tidak diperlukan akun.

Langkah 2. Seret seluruh folder ke zona unggahan (atau klik Pilih file dan pilih setiap file dalam folder menggunakan Cmd+A di Mac atau Ctrl+A di Windows). Alat ini mempertahankan struktur folder di dalam ZIP - subfolder Anda tetap bersarang persis seperti sebelumnya.

Langkah 3. Klik Proses. Setelah ZIP dibangun, klik Unduh. Hasilnya adalah satu file .zip yang dapat dibuka oleh sistem operasi modern dengan satu klik ganda.


Mengapa mengkompres folder?

Tiga alasan mendorong sebagian besar pekerjaan kompresi folder:

Lampiran email. Gmail membatasi lampiran pada 25 MB, Outlook pada 20 MB, dan webmail perusahaan sering pada 10 MB. Folder dengan 30 foto pada 4 MB masing-masing (total 120 MB) tidak akan lewat; folder yang sama dikompres pada kompresi default sering turun ke 70-90 MB - masih terlalu besar dengan sendirinya, tetapi jika foto adalah JPG Anda perlu mengkompres gambar terlebih dahulu (lihat Compress Image) dan kemudian ZIP versi yang lebih kecil. Untuk folder non-gambar (dokumen, kode sumber, spreadsheet) langkah ZIP sering melakukan kerja keras sendiri.

Unggahan file tunggal. Banyak portal, sistem manajemen pembelajaran, dan formulir pengiriman menerima satu file pada satu waktu. ZIP mengubah "30 dokumen" menjadi "satu .zip" sehingga unggahan selesai dalam sekali.

Penyimpanan dan berbagi. Biaya penyimpanan cloud biasanya ditagih berdasarkan jumlah file ditambah byte. Folder berisi 1.000 file kecil dikompres dengan baik (teks dan kode biasanya menyusut 60-80%) dan mengurangi kedua metrik.


Seberapa kecil ZIP sebenarnya?

Itu tergantung sepenuhnya pada apa yang ada di folder.

Konten folderRasio kompresi tipikalCatatan
Teks biasa, kode sumber, JSON, CSVPengurangan 60-80%Kompresi sangat baik - banyak redundansi
Dokumen Office (.docx, .xlsx, .pptx)Pengurangan 5-15%Ini sudah file ZIP secara internal; sedikit ruang ekstra
Dokumen PDFPengurangan 5-20%PDF biasanya pra-dikompres
Foto JPG, HEIC, WebPPengurangan 0-3%Sudah dikompres lossy; ZIP tidak dapat menyusut lebih jauh. Kompres gambar terlebih dahulu.
Tangkapan layar PNGPengurangan 5-15%PNG memiliki kompresinya sendiri; ZIP menambahkan sedikit
Video MP3, MP4, AVIPengurangan 0-2%Sudah dikompres oleh codec. ZIP terutama untuk bundling.
Folder campuran (dokumen + foto + kode)Pengurangan 15-40%Didorong oleh bagian yang paling dapat dikompres

Aturan praktis yang jujur: jika folder sebagian besar adalah foto atau video, ZIP memberi Anda kontainer pengiriman file tunggal, bukan payload yang lebih kecil. Kompres gambar atau transcoding video sebelum zip jika tujuannya adalah unggahan yang lebih kecil.


Skenario umum

Kirim folder proyek ke rekan satu tim. Seret folder proyek ke Kompres, File dan Folder ZIP. Struktur folder dipertahankan di dalam .zip, jadi ketika rekan satu tim Anda membuka di sisi lain (menggunakan Unzip File atau unzip OS bawaan apa pun) mereka mendapatkan folder asli kembali persis seperti yang Anda miliki.

Kompres folder untuk email di bawah 25 MB. Jalankan folder melalui alat ZIP terlebih dahulu. Jika hasilnya masih di atas batas dan sebagian besar ukurannya adalah foto, rutekan foto melalui Compress Image pada kualitas 80% (biasanya memotong ukurannya menjadi setengah tanpa kehilangan kualitas yang terlihat) dan re-ZIP. Jika hasilnya masih melebihi batas, lihat cara mengurangi ukuran ZIP untuk strategi berikutnya.

Bundel kode sumber untuk pengajuan pekerjaan. Jatuhkan folder proyek. Alat menjaga struktur direktori (src/, tests/, docs/) tetap utuh, sehingga penerima dapat mengekstrak dan menjalankan kode dengan segera. Jika proyek memiliki node_modules atau artefak build, hapus sebelum zip - mereka besar dan penerima akan membangunnya kembali.

Tambahkan kata sandi ke ZIP. Jika konten folder sensitif, gunakan Kompres, File dan Folder ZIP dengan opsi kata sandi diaktifkan (cari bidang kata sandi di pengaturan alat). Atau, untuk folder yang sudah Anda zip tanpa kata sandi, lihat bagaimana kata sandi bekerja pada format arsip.

Kompres folder yang berisi file lebih besar dari 4 GB. ZIP standar memiliki batas per-file pada 4 GB (format warisan ZIP32); ZIP64 mengangkat ini. Kompres, File dan Folder ZIP menangani file di atas ambang ini secara transparan - cukup unggah.


Bagaimana dengan 7z, RAR, atau tar.gz?

7z dikompres lebih kuat dari ZIP - biasanya 10-20% lebih kecil untuk konten yang sama - tetapi membutuhkan 7-Zip, The Unarchiver, atau aplikasi serupa di pihak penerima. macOS dan iOS tidak dapat membuka file .7z tanpa perangkat lunak pihak ketiga. Gunakan 7z ketika Anda mengontrol kedua sisi dan ukuran lebih penting daripada kenyamanan.

RAR mirip dengan 7z dalam kompresi tetapi formatnya adalah eksklusif; membuat .rar membutuhkan WinRAR (berbayar). Sebagian besar panduan modern merekomendasikan melewatkan RAR demi ZIP atau 7z.

tar.gz / tar.bz2 mendominasi alur kerja Linux dan sisi server. Mereka tidak didukung secara native di Windows pra-Windows-10. Jika penerima menggunakan Linux atau menjalankan skrip, tar.gz baik-baik saja; jika tidak ZIP adalah taruhan yang lebih aman.

Untuk skenario folder-untuk-email, kirim-ke-rekan-tim, kirim-ke-portal, ZIP adalah jawaban yang tepat 99% dari waktu.


Privasi dan apa yang diunggah

Kompresi berjalan pada layanan pemrosesan kami - folder Anda diunggah, dikompres, dan hasilnya dikembalikan ke browser Anda untuk diunduh. File diproses dan dihapus; kami tidak menyimpan konten folder di luar durasi sesi aktif. Jika folder Anda sangat sensitif (hukum, medis, penggajian), pertimbangkan untuk menggunakan alat ZIP hanya-desktop (terintegrasi di Windows, macOS, dan sebagian besar distribusi Linux) untuk menyimpan file sepenuhnya lokal.


Rekap cepat

1. Seret folder ke Kompres, File dan Folder ZIP.
2. Klik Proses dan tunggu ZIP dibangun.
3. Klik Unduh - Anda mendapatkan satu .zip dengan struktur folder yang dipertahankan.

Jika ZIP masih terlalu besar untuk tujuan, hambatan biasanya adalah foto atau video di folder - kompres yang pertama, lalu re-ZIP.


Alat dan panduan terkait

  • Kompres, File dan Folder ZIP - alat yang dilalui panduan ini.
  • Unzip File - ekstrak .zip kembali ke folder.
  • Compress Image - mengurangi ukuran foto sebelum mengkompres ketika folder berat foto.
