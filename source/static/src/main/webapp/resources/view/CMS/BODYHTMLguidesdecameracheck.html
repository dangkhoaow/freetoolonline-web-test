Was Leute mit "Kamera-Check" meinen

Der Begriff "Kamera-Check" taucht in Suchanfragen von Lesern auf, die bestätigen wollen, dass die Webcam vor einem Videoanruf funktioniert, nach dem Einstecken einer externen Kamera, nach einem Browser-Update, oder bevor sie eine Aufnahme senden. Der Leser fragt nicht nach einem Kalibrierungstest oder einem Bildqualitäts-Benchmark; er stellt die einfachere Frage: "Schaltet sich die Webcam ein, und sehe ich mich in der Vorschau?" Das ist dieselbe Frage, die "Kamera-Test" beantwortet. Die Tools auf dieser Seite behandeln die beiden Formulierungen austauschbar und leiten beide zur gleichen Browser-Vorschau weiter.

Die Prüfung ausführen

Die Prüfung selbst braucht einen Klick. Öffnen Sie Camera Test Online, akzeptieren Sie die Berechtigungsanfrage, die der Browser in der Adressleiste anzeigt, und eine Live-Vorschau Ihrer Webcam sollte auf der Seite erscheinen. Wenn die Vorschau aktiviert wird und Sie sich sehen, ist die Kamera erlaubt und funktioniert. Wenn die Vorschau dunkel oder leer bleibt, traf die Seite auf einen der beiden Fehlermodi, die im nächsten Abschnitt behandelt werden. Kein Frame wird hochgeladen, keine Aufnahme wird gespeichert, und das Stoppen des Tests gibt die Kamera frei, sodass die Anzeigeleuchte erlischt.

Drei Endzustände, die Sie sehen könnten

Erlaubt. Der Browser zeigte beim ersten Mal, als die Seite getUserMedia aufrief, eine Berechtigungsanfrage an, Sie klickten auf Erlauben, und die Vorschau aktiviert sich. Ein kleines Kamera-Symbol (ohne Querstrich) sitzt in der Adressleiste und zeigt, dass die Seite eine aktive Berechtigung hält.

Blockiert / verweigert. Entweder Sie klickten zuvor auf Blockieren in der Anfrage, oder die seitenspezifische Einstellung ist auf Blockieren per Richtlinie gesetzt. Der Browser weist zukünftige getUserMedia-Aufrufe still ab und die Vorschau bleibt dunkel. Das Adressleisten-Symbol zeigt eine Kamera mit Querstrich. Kamera-Test-Berechtigung blockiert: wie man Kamerazugriff erlaubt geht den Erlauben-Pfad pro Browser durch.

Ignoriert. Die Anfrage erschien, aber Sie schlossen sie oder wechselten den Tab, ohne zu antworten. Der Browser behandelt dies als keine-Entscheidung und fragt beim nächsten Seitenladen erneut. Das Neuladen der Kamera-Test-Seite löst eine frische Anfrage aus; klicken Sie diesmal auf Erlauben.

Wenn etwas nicht funktioniert

Eine dunkle Vorschau ohne Berechtigungsanfrage bedeutet normalerweise, dass der Browser eine frühere Blockieren-Entscheidung gegen diesen Ursprung erinnerte - lesen Sie Kamera-Test-Berechtigung blockiert für den Erlauben-Pfad pro Browser. Eine Vorschau, die sich aktiviert, aber einen schwarzen Frame zeigt, bedeutet normalerweise, dass die Linse abgedeckt ist, eine andere App das Gerät hält, oder ein Treiber fehlerhaft ist - lesen Sie Kamera-Test zeigt schwarzen Bildschirm: vier Behebungen. Wenn Sie eine vollständige Vor-Meeting-Prüfung durchführen, geht vor einem Videoanruf: welche Tools auszuführen die Sequenz (Kamera, Mikrofon, Bildschirm, Tastatur) in unter einer Minute durch.

Datenschutz-Haltung

Der Kamera-Test auf dieser Seite läuft vollständig in Ihrem Browser. Die Seite ruft navigator.mediaDevices.getUserMedia({ video: true }) auf, hängt den resultierenden Videostream an ein verstecktes <video>-Element an und liest Frames in ein Canvas nur zur Anzeige. Kein Frame wird hochgeladen, keine Aufnahme wird gespeichert, und der Stream wird in dem Moment freigegeben, in dem Sie die Seite verlassen oder den Test stoppen. Die Datenschutzrichtlinie dokumentiert den Kein-Upload, Keine-Aufnahme-Vertrag für die Kamera-Test-Oberfläche in klarer Sprache.
