Mikrofon-Test Ohne Ton: 4 Loesungen Die Wirklich Funktionieren

Zuletzt geprueft 2026-05-03. Wenn Ihr Browser-Mikrofon-Test oeffnet, aber der Pegelanzeiger waehrend des Sprechens flach bleibt, ist das Mikrofon selten defekt - eine der vier Ursachen unten ist weit wahrscheinlicher. Gehen Sie die Liste von oben nach unten durch; jede Loesung dauert unter einer Minute.

30-Sekunden-Antwort. Eine flache Pegelanzeige bei einem Browser-Mikrofon-Test bedeutet fast immer, dass das Geraet stummgeschaltet, blockiert oder von einer anderen App gehalten wird - nicht defekt. Schritt 1: Mikrofon-Berechtigung im Schloss-Symbol der Adressleiste erneut erteilen. Schritt 2: bestaetigen, dass das Mikrofon auf OS-Ebene aktiviert ist (Datenschutz und Sicherheit → Mikrofon). Schritt 3: Meeting-Client (Zoom, Teams, Google Meet, Discord) oder virtuelle Mikrofon-Treiber beenden, die das Geraet bereits halten koennten. Schritt 4: physischen Stummschaltungs-Schalter an einem USB-Headset oder Laptop-Funktionstaste pruefen. Die meisten "Kein Ton"-Berichte loesen sich in Schritt 1 oder Schritt 3.

Loesung 1 - Browser-Berechtigung wurde verweigert oder ist veraltet

Der Browser ist die naechste Schicht zur Testseite, sodass sein Berechtigungsstatus die haeufigste Ursache fuer eine flache Wellenform ist. Der Browser kann zeigen, dass die Seite "erlaubt" ist, waehrend er immer noch einen veralteten Berechtigungsdatensatz von einem anderen Netzwerk oder nach einer Profil-Synchronisierung haelt.

Chrome / Edge: klicken Sie auf das Schloss-Symbol (oder "Website-Informationen") in der Adressleiste → Mikrofon → waehlen Sie Zulassen, dann laden Sie die Seite neu. Wenn das Dropdown Mikrofon: Blockieren anzeigt, wechseln Sie zu Zulassen und laden neu.

Safari (macOS): Safari-Menue → Einstellungen fuer diese Website → Mikrofon: Zulassen. Safari zeigt die Anfrage nicht ueber die Adressleiste, also ist das Aendern ueber das Menue der einzige Weg.

Firefox: klicken Sie auf das Schloss-Symbol → Verbindung sicher → Weitere Informationen → Berechtigungen → loeschen Sie "Mikrofon verwenden". Laden Sie die Seite neu, damit Firefox erneut nachfragt.

Wenn das Adressleisten-Symbol zeigt, dass das Mikrofon erlaubt ist, und der Pegelanzeiger immer noch flach ist, ist die Browser-Berechtigung in Ordnung; gehen Sie zu Loesung 2.

Loesung 2 - Das Betriebssystem hat das Mikrofon stummgeschaltet

Moderne Betriebssysteme verwalten einen Hauptschalter, der jede Browser-Berechtigung ueberschreibt. macOS fuehrte dies in Mojave ein; Windows 10 fuegte es in 1903 hinzu. Eine flache Pegelanzeige mit der Browser-Berechtigung bereits auf "Zulassen" gesetzt bedeutet fast immer, dass der OS-Schalter aus ist.

macOS: Systemeinstellungen → Datenschutz und Sicherheit → Mikrofon. Ihr Browser muss aktiviert sein. Wenn er nicht in der Liste ist, oeffnen Sie den Mikrofon-Test einmal und macOS fuegt den Eintrag hinzu. Pruefen Sie auch das Kontrollzentrum der Menueleiste auf einen orangefarbenen Mikrofon-Punkt - das OS zeigt ihn, wann immer eine App das Geraet haelt.

Windows 11 / 10: Einstellungen → Datenschutz und Sicherheit → Mikrofon. Aktivieren Sie sowohl Apps Zugriff auf Ihr Mikrofon erlauben als auch Desktop-Apps Zugriff auf Ihr Mikrofon erlauben. Beide Flags muessen aktiviert sein - die zweite deckt Chrome, Edge und Firefox auf Windows ab. Bestaetigen Sie, dass Mikrofonzugriff oben auf der Seite auch Ein ist.

iOS / iPadOS: Einstellungen → Safari → Mikrofon → waehlen Sie Fragen oder Erlauben. iOS behaelt die Mikrofon-Berechtigung pro Site; das Loeschen der Site-Daten setzt sie zurueck. Pruefen Sie auch den kleinen orangefarbenen Punkt oben am Bildschirm - wenn er waehrend des Tests fehlt, hat iOS das Mikrofon nicht an die Seite vergeben.

Android (Chrome): Einstellungen → Apps → Chrome → Berechtigungen → Mikrofon → Nur waehrend der Verwendung der App erlauben. Die Android-Einstellung ueberschreibt das Zulassen im Browser, wenn der App-Ebene-Schalter aus ist.

Loesung 3 - Eine andere App haelt das Mikrofon

Mikrofone exponieren auf den meisten Consumer-Treibern einen einzigen Capture-Stream. Der erste Prozess, der ihn oeffnet, erhaelt exklusiven Zugriff; jede spaetere Anfrage erhaelt Stille, weil das Geraet "beschaeftigt" aber nicht "blockiert" ist. Meeting-Clients und virtuelle Mikrofon-Treiber sind die haeufigsten Schuldigen.

Halter beenden / killen: schliessen Sie Zoom, Teams, Google Meet, Webex, Discord, OBS Studio, Krisp, NVIDIA Broadcast, VoiceMeeter, Loopback und jede DAW (Audacity, GarageBand, Logic, Reaper), die einen Hintergrund-Stream gelassen haben koennte. Auf macOS gibt ein Force Quit der App das Mikrofon innerhalb von zwei Sekunden frei; auf Windows tut das Beenden des Prozesses im Task-Manager dasselbe.

Freigabe bestaetigen: macOS zeigt einen orangefarbenen Punkt oben am Bildschirm und einen Eintrag im Kontrollzentrum, waehrend eine App das Mikrofon haelt. Windows 11 zeigt ein Mikrofon-Symbol in der Taskleiste. iOS / iPadOS zeigen den orangefarbenen Datenschutz-Punkt. Wenn der Indikator nach dem Schliessen jedes Meeting-Clients an bleibt, ist ein virtueller Mikrofon-Treiber noch geladen - neu starten, um vollstaendig zu loeschen.

Erneut testen: laden Sie /microphone-test.html neu. Der Pegelanzeiger sollte innerhalb einer Sekunde reagieren, nachdem die Seite die Berechtigung erteilt hat und Sie sprechen.

Loesung 4 - Hardware-Stummschaltung, Funktionstaste oder Bluetooth-Headset-Pairing-Konflikt

Die letzte Schicht ist die Hardware. Viele USB-Headsets und Webcams werden mit einem physischen Stummschaltungs-Schalter am Kabel oder Hoerer geliefert; einige Laptops leiten den gleichen Schalter ueber eine Funktionstaste. Bluetooth-Headsets fuegen einen vierten Fehlermodus hinzu: das OS kann sich automatisch mit einem anderen Geraet pairen als dem, das Sie testen wollten.

USB-Headset-Stummschaltung: suchen Sie nach einem kleinen Schalter oder Knopf in der Naehe des Inline-Mikrofons am Kabel. Er ist normalerweise mit einem Mikrofon-Symbol und einem Schraegstrich gekennzeichnet. Schalten Sie ihn aus und testen Sie erneut. Der Pegelanzeiger sollte sofort springen. Einige Headsets schalten auch beim langen Druecken der Lautstaerketaste stumm - pruefen Sie das Handbuch.

Laptop-Funktionstaste: Lenovo-, Dell- und HP-Laptops haben eine Fn + F-Taste-Verknuepfung (Fn + F4 bei den meisten ThinkPads, Fn + F1 bei einigen Dells), die das integrierte Mikrofon auf Firmware-Ebene umschaltet. Druecken Sie es und testen erneut. Die Mikrofon-LED auf der Tastatur leuchtet normalerweise, wenn das Mikrofon auf Firmware-Ebene stummgeschaltet ist.

Bluetooth-Headset-Pairing-Konflikt: wenn zwei Bluetooth-Headsets gekoppelt sind, koennen macOS und Windows automatisch das auswaehlen, das zuletzt gekoppelt wurde, auch wenn es nicht das in Ihrem Ohr ist. Oeffnen Sie Systemeinstellungen → Ton → Eingabe (macOS) oder Einstellungen → System → Ton → Eingabe (Windows) und bestaetigen Sie, dass das aktive Mikrofon das ist, das Sie erwartet haben. Wechseln Sie das Eingabegeraet bei Bedarf; der Test erkennt die Aenderung sofort.

USB-Hub-Stromproblem: einige USB-C-Hubs unterversorgen ihre Downstream-Ports unter Last, wodurch das Mikrofon intermittierend abfaellt. Stecken Sie das Headset direkt in den Laptop und testen erneut - wenn das Mikrofon zurueckkommt, ist der Hub die Ursache.

Immer noch kein Ton nach allen vier? - der Diagnoseablauf

  1. Versuchen Sie eine zweite Seite. Oeffnen Sie mictests.com in derselben Browser-Sitzung. Wenn diese Seite auch einen flachen Pegelanzeiger zeigt, liegt die Ursache noch vor der Seite (Browser / OS / App / Hardware).
  2. Versuchen Sie einen zweiten Browser. Wenn Chrome flach ist, aber Safari den Pegel reagieren zeigt, liegt die Ursache auf Browser-Ebene (Erweiterung, Profil oder eine veraltete Berechtigung). Setzen Sie die Chrome-Site-Einstellungen zurueck oder verwenden Sie ein Gastprofil.
  3. Versuchen Sie ein zweites Benutzerkonto. Wechseln Sie zu einem frisch erstellten OS-Benutzer und oeffnen Sie den Mikrofon-Test. Wenn es dort funktioniert, liegt die Ursache in Ihrem Benutzerprofil (ein Berechtigungseintrag, eine Browser-Erweiterung oder ein virtueller Mikrofon-Treiber, der fuer Ihren Benutzer registriert ist).
  4. Versuchen Sie eine zweite Maschine. Wenn ein Telefon oder Tablet den Mikrofon-Test sofort funktionieren laesst, hat die fehlerhafte Maschine ein Hardware- oder Treiberproblem. Oeffnen Sie den Geraetemanager (Windows) oder Systeminformationen → Hardware → Audio (macOS) und suchen Sie das Mikrofon unter Eingabe.

Die vier Schritte grenzen die Ursache von der Testseite nach aussen bis zur Hardware in weniger als fuenf Minuten ein.

Was ist mit "schwarzer Bildschirm"-Problemen beim Kamera-Test?

Dasselbe Vier-Schichten-Modell gilt fuer den Kamera-Test: Browser-Berechtigung, OS-ebene Datenschutz und Sicherheit-Schalter, eine andere App haelt das Geraet, Hardware-Verschluss oder BIOS-Schalter. Wenn ein Videoanruf komplett fehlschlaegt, fuehren Sie sowohl den Kamera-Test als auch den Mikrofon-Test in derselben Browser-Sitzung aus - wenn beide funktionsfaehig zurueckkommen, ist der Anruf-Client selbst die Ursache; wenn nur einer fehlschlaegt, zeigt der Test, der defekt zurueckkam, auf die Schicht. Haeufig gestellte Fragen

Das OS zeigt den orangefarbenen Datenschutz-Punkt, waehrend der Pegelanzeiger flach ist - was bedeutet das?

Der Punkt bedeutet, dass das OS der aktiven Seite Mikrofon-Zugriff gewaehrt hat; die Stille liegt unterhalb der OS-Schicht. Das verweist auf Loesung 3 (eine andere App haelt den eigentlichen Stream) oder Loesung 4 (Hardware-Stummschaltung oder falsches Eingabegeraet).

Warum reagiert der Pegelanzeiger auf Tastatureingaben, aber nicht auf meine Stimme?

Das Mikrofon funktioniert, ist aber zu weit von Ihrem Mund entfernt oder weggerichtet. Bewegen Sie das Mikrofon innerhalb 15 cm und testen erneut. Wenn Sie das eingebaute Laptop-Mikrofon verwenden, ist es am Bildschirm-Bezel - sprechen Sie in Richtung Bildschirm, nicht in Richtung Tastatur.

Bluetooth-Headset verbunden, aber der Pegelanzeiger ist flach - ist das Headset defekt?

Fast nie. Bluetooth-Geraete haben ein separates SCO-Profil fuer Zwei-Wege-Audio und ein A2DP-Profil fuer Ein-Wege-Hoeren. Wenn das OS A2DP gewaehlt hat, ist das Mikrofon unerreichbar, obwohl die Wiedergabe funktioniert. Wechseln Sie in den "Headset"-Modus in den OS-Toneinstellungen oder Anruf-Einstellungen im Browser, um SCO zu erzwingen.

Der Test funktioniert, aber die Aufnahme beim Anruf hat immer noch kein Audio - dieselbe Loesung?

Nein - das ist Meeting-App-Routing, kein Test-Fehler. Im Anruf-Client (Zoom / Teams / Meet) oeffnen Sie die Audio-Einstellungen und waehlen das gleiche Mikrofon, das der Browser-Test verwendet. Der Test bestaetigt die Mikrofon- und OS-Schichten; der Anruf-Client waehlt sein eigenes Geraet.
