Converting HEIC photos to JPG rather than vectorising PNG?

This page traces a raster PNG into a scalable SVG, so a HEIC photo from an iPhone is not a match for the trace algorithm running here - HEIC is a compressed photo format, not a clean-edged graphic, and a photo rarely vectorises into useful curves. The HEIC to JPG conversion lives at /heic-to-jpg.html, which reads the HEIC container and writes a JPG copy without attempting any vector trace. PNG photos that are really iPhone screenshots end up the same way: re-save them as JPG first if photo-quality is the goal, then keep the SVG trace for logos and flat-colour icons that have hard edges to follow.
