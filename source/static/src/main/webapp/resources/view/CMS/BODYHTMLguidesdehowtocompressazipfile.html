Die drei Schritte

  1. Oeffnen Sie den In-Browser-ZIP-Ersteller unter /zip-file.html. Ziehen Sie den Ordner (oder die losen Dateien) in den Upload-Slot.
  2. Waehlen Sie das Komprimierungsziel. Eine regulaere .zip ist der Standard; wenn die Empfaengerseite passwortgeschuetzte Archive akzeptiert, verwenden Sie stattdessen zip mit Passwort. Das Tool verpackt das gewaehlte Ziel inline.
  3. Laden Sie die .zip herunter, wenn der Fortschrittsbalken fertig ist. Das Archiv ist das, was Sie hochladen, anhaengen oder teilen.

Wann ZIP-Komprimierung wirklich Bytes spart

Die ehrliche Antwort vorab: ZIP-Verpackung verkleinert komprimierbare Eingaben (Klartext, Quellcode, CSV, JSON, unkomprimierte Bilder) spuerbar; sie verkleinert bereits komprimierte Medien (JPG, MP4, MP3, PDF) nicht um mehr als ein paar Prozent, und der ZIP-Verpackungs-Overhead kann das Archiv in diesem Fall sogar leicht groesser machen als die Summe der Eingaben. Wenn das Ziel ein kleineres Archiv von Mediendateien ist, kommt der Byte-Gewinn vom Neukodieren der Medien zuerst - der zugehoerige Leitfaden ZIP komprimieren - wann es Platz spart erklaert die Entscheidungsregel. Wenn das Ziel eine bestimmte Zielgroesse ist (unter 25 MB fuer E-Mail, unter 100 KB fuer ein Formular), deckt der Schwester-Leitfaden So komprimieren Sie eine ZIP-Datei auf eine bestimmte Groesse den Qualitaet-vs-Groesse-Trade-off ab.

Haeufig gestellte Fragen

Wird die Datei auf einen Server hochgeladen?

Die Upload-Pipeline unter /zip-file.html laeuft serverseitig - das In-Browser-Tool sendet die Dateien ueber HTTPS an den Verarbeitungsdienst, das Archiv wird dort geschrieben, und der Download-Link wird zurueck an die Seite geliefert. Dateien werden nach einem kurzen Aufbewahrungsfenster vom Verarbeitungsdienst entfernt. Der gemeinsame Upload-Vertrag fuer jedes serverseitige Tool ist auf jeder Tool-Seite dokumentiert.

Wie gross ist die maximale Dateigroesse, die ich komprimieren kann?

Das Pro-Upload-Groessenlimit ist im Seiten-Bootstrap festgelegt (der Wert, der beim Laden in die Tool-Seite gerendert wird). Fuer ein routinemaessiges E-Mail-freundliches Archiv ist das praktische Limit die Anhang-Obergrenze des empfangenden Posteingangs (Gmail 25 MB, Outlook und Unternehmens-Posteingaenge oft 10-20 MB). Fuer ein groesseres Archiv teilen Sie ueber einen Cloud-Drive-Link statt eines E-Mail-Anhangs.

Sollte ich ZIP oder 7z oder RAR verwenden?

ZIP ist die sicherste Wahl fuer plattformuebergreifendes Teilen, weil jedes wichtige Betriebssystem eine .zip oeffnen kann, ohne zusaetzliche Software zu installieren. 7z komprimiert auf einigen Eingaben staerker, aber der Empfaenger benoetigt moeglicherweise einen Extraktor. RAR ist selten die richtige Wahl fuer anonymes Teilen, weil es auf den meisten Systemen standardmaessig nicht oeffenbar ist.
