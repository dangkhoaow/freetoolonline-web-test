HEIC vs JPG vs WebP - cuando usar cada formato de imagen

Publicado el 19 de Abril de 2026

Tres formatos de imagen dominan la fotografia diaria y la entrega web: HEIC (predeterminado en iPhones desde 2017), JPG (el fallback universal) y WebP (el reemplazo de archivos mas pequenos de Google para JPG y PNG). Cada uno codifica fotos de manera diferente, y la eleccion correcta depende de donde se vaya a usar el archivo. Esta guia recorre lo que cada formato realmente hace, los trade-offs practicos y los pasos exactos de conversion para las cuatro situaciones que las personas encuentran mas.


Que es (y que no es) cada formato

HEIC (High Efficiency Image Container) envuelve fotos codificadas con el codec HEVC - el mismo codec usado para video H.265. Apple lo hizo predeterminado del iPhone porque los archivos son aproximadamente la mitad del tamano de JPG con la misma calidad visual, y HEIC puede llevar multiples imagenes, mapas de profundidad y metadatos de live-photo en un solo contenedor. El detalle: HEVC esta gravado por patentes, asi que Windows, Android antiguo, la mayoria de los navegadores y muchos servicios web no decodifican HEIC nativamente.

JPG (JPEG) ha sido el formato predeterminado de foto web desde 1992. Usa compresion con perdidas basada en DCT, viene en cada biblioteca de imagen en cada plataforma y no tiene sorpresas de compatibilidad. Los archivos son mas grandes que HEIC o WebP con calidad visual equivalente, pero si algo necesita abrir la imagen de manera confiable - correo, contratos, impresion, archivado - JPG es la apuesta mas segura.

WebP es la respuesta de Google a "mas pequeno que JPG, mejor calidad que PNG". Viene con modos con y sin perdidas, soporta transparencia (a diferencia de JPG) y es decodificado nativamente en cada navegador actual (Chrome, Firefox, Safari 14+, Edge). Permanece menos ampliamente soportado en clientes de correo y algunas herramientas de escritorio antiguas, asi que aun no es un reemplazo universal.

Cuando elegir cada formato

Elige JPG cuando: estas enviando una foto por correo, enviando un documento a un sistema contractual/legal, imprimiendo, subiendo a un servicio que rechaza cualquier otra cosa (portales bancarios, gubernamentales, de seguros), o archivando una foto a largo plazo donde no puedes garantizar soporte futuro de decodificador para HEIC o WebP.

Elige WebP cuando: la imagen va en tu propio sitio web y controlas el servidor. WebP corta significativamente el presupuesto de peso de pagina - importante para cargas mas rapidas y para visitantes en datos moviles. Todos los navegadores modernos lo decodifican; el servidor aun puede servir un fallback JPG via el elemento <picture> para el <1% de visitantes en clientes antiguos.

Manten HEIC cuando: las fotos permanecen en el ecosistema Apple (app Photos, iCloud, Mensajes entre iPhones, AirDrop entre Macs). Los archivos son la mitad del tamano, la calidad es impecable y no hay paso de conversion para introducir artefactos. En el momento en que necesites enviarlos fuera de Apple, convierte a JPG o WebP primero.


Caminos rapidos de conversion para las cuatro situaciones comunes

Foto de iPhone → correo o carga web. Usa HEIC to JPG - suelta el archivo .heic, descarga el .jpg, adjunta. Tambien puedes configurar Ajustes de iPhone → Camara → Formatos → Mas Compatible para guardar fotos futuras como JPG por defecto.

Lote de iPhone → WebP para un sitio web. Convierte HEIC → JPG primero, luego JPG → WebP con Compress JPEG o una tuberia en tiempo de construccion. Una conversion en dos pasos sigue siendo mucho mas pequena que el HEIC original cuando se entrega por HTTP.

Captura de pantalla PNG → WebP. Usa SVG to PNG al reves via ImageMagick o pegalo en ImageMagick Online con -define webp:lossless=true. WebP sin perdidas de capturas de pantalla UI tipicamente corta el tamano un 25-45%.

HEIC con capa de transparencia → WebP o PNG. JPG eliminara el canal alfa, asi que evitalo para cualquier imagen con transparencia. HEIC to JPG aplana el alfa a blanco; exporta a PNG en su lugar si la transparencia importa, o a WebP (sin perdidas) para un balance recomendado calidad/tamano.


Soporte de navegador y dispositivo de un vistazo

Chrome / Edge: WebP (desde 2014), HEIC no. Firefox: WebP si, HEIC no. Safari (macOS): WebP desde 14, HEIC nativo. Safari (iOS): WebP desde 14, HEIC nativo. Windows Explorer: HEIC requiere la extension paga Microsoft HEVC Video; WebP nativo desde Windows 10. Android Photos: WebP nativo, HEIC nativo desde Android 10. Clientes de correo: JPG es el unico formato garantizado para renderizar en Gmail, Outlook, Apple Mail, ProtonMail y correo web empresarial. Planifica en consecuencia.


Arbol de decision resumido

1. El archivo va fuera de tu control (correo, portal, sistema legal, impresion)? → JPG.
2. Va en un sitio web que controlas con trafico de navegador moderno? → WebP (fallback JPG via <picture>).
3. Permaneciendo dentro de Apple (Photos, iCloud, iMessage)? → HEIC.
4. Transparencia requerida? → WebP (sin perdidas) o PNG; nunca JPG.
