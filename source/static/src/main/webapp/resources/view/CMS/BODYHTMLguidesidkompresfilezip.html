Kompres File Zip

Kompres file zip adalah cara mengemas satu file, beberapa file, atau seluruh folder menjadi satu arsip .zip di browser. Anda unggah berkas, layanan membundelnya di server lewat HTTPS, file sumber dihapus otomatis setelah jeda singkat, lalu arsip kembali sebagai satu unduhan.

Alat pelaksana/zip-file.html

Apa yang sebenarnya dilakukan alat kompres zip ini

Alat di /zip-file.html mengerjakan tiga pekerjaan utama dalam satu halaman. Pertama, ia membundel: satu file, kumpulan file, atau satu folder lengkap (sub-folder tetap dipertahankan) berubah menjadi satu arsip .zip yang penerima unduh sekali klik. Kedua, ia memberi nama arsip lewat panel Settings sehingga penerima melihat topik atau nama proyek pada nama berkas alih-alih stempel waktu generik. Ketiga, ia dapat mengunci arsip dengan kata sandi opsional: pilih metode enkripsi Standard, AES-128, atau AES-256 ketika kata sandi akan dikirim lewat saluran terpisah dari arsipnya. Seluruh pemrosesan terjadi di server kami melalui koneksi HTTPS, dan file sumber dihapus otomatis setelah jeda retensi singkat.

Apakah kompres file zip benar-benar memperkecil berkas

Hasil arsip dari halaman /zip-file.html ini hanya lebih kecil dari jumlah masukannya ketika berkas yang Anda satukan memang dapat dipadatkan oleh algoritma deflate ZIP. Bila Anda membundel file teks panjang, log harian, ekspor CSV, kode sumber, atau dokumen Office, urutan byte yang berulang di dalamnya membiarkan ZIP memadatkan keluaran ke kisaran 30 hingga 60 persen dari ukuran sumber. Sebaliknya, masukan yang sudah berupa media terkompresi - kamera digital JPG, klip video MP4, lagu MP3, atau lampiran PDF yang isinya gambar - sudah dipadatkan codec masing-masing sebelum tiba di sini, jadi ZIP hanya membungkusnya: ukuran akhir bergerak sekitar 0 hingga 5 persen dan kadang sedikit bertambah karena overhead nama berkas dan tabel arsip. Ringkasannya tetap satu: ZIP memadatkan apa yang belum dipadat; ZIP membungkus apa yang sudah dipadat. Bila tujuan Anda adalah memenuhi batas 25 MB lampiran email atau batas unggah lain, perkecil dulu media berat di alat kompres gambar atau video, baru gabungkan hasilnya menjadi satu arsip ZIP di halaman ini.

Privasi dan penanganan data

Halaman ini berjalan tanpa akun: Anda tidak perlu mendaftar, masuk, atau memasang ekstensi browser sebelum membundel arsip. Berkas masukan dikirim lewat HTTPS ke server pemroses kami, arsip .zip dibangun di sana, dan berkas sumber dihapus otomatis setelah jeda retensi singkat sehingga tidak menumpuk di sisi server. Tautan unduhan yang dihasilkan menunjuk ke arsip akhir saja - bukan ke salinan masukan Anda. Untuk arsip yang akan melewati saluran berbagi tepercaya rendah, aktifkan kata sandi dan pilih metode AES-128 atau AES-256 di panel Settings, lalu kirim kata sandinya di saluran berbeda dari arsipnya supaya pembaca yang mencegat satu sisi tidak otomatis membuka sisi yang lain.

Cara kompres file zip di browser, langkah demi langkah

  1. Buka /zip-file.html di tab baru.
  2. Seret atau pilih file yang ingin dibundel. Sebuah folder juga diterima; sub-folder tetap dipertahankan di dalam arsip.
  3. Opsional: buka Settings, beri nama arsip, dan tambahkan kata sandi plus metode enkripsi (Standard, AES-128, atau AES-256) bila penerima sudah punya saluran terpisah untuk kata sandi.
  4. Klik tombol Zip. Halaman akan mengunggah masukan lewat HTTPS, membangun arsip di server, dan menampilkan tautan unduhan.
  5. Simpan berkas .zip yang dihasilkan. Jika Anda memasang kata sandi, kirim kata sandi tersebut lewat jalur yang berbeda dari arsipnya.

Pertanyaan yang sering muncul

Apakah perlu mendaftar atau memasang aplikasi?

Tidak. Halaman ini berjalan di browser dan tidak meminta akun. Anda hanya menyeret berkas, menekan Zip, dan mengunduh hasilnya.

Apakah file saya aman?

Unggahan dikirim lewat HTTPS ke server pemroses, arsip dibangun di sisi server, dan file sumber dihapus otomatis setelah jeda retensi singkat. Untuk lapisan tambahan, gunakan kata sandi dengan metode AES-128 atau AES-256 dan kirim kata sandinya lewat saluran terpisah.

Mengapa ukuran arsip saya tidak banyak berkurang?

Kemungkinan besar masukan Anda berisi media yang sudah dipadatkan oleh codec masing-masing (JPG, MP4, MP3, atau PDF berisi gambar). ZIP hanya membungkus berkas semacam itu tanpa banyak menyusutkannya. Kompres media berat lebih dulu dengan alat gambar atau video, lalu bundel hasilnya.
