Izin Tes Kamera Diblokir: Cara Mengizinkan Akses Kamera di Browser Anda

Ditargetkan pada saat halaman tes kamera browser dimuat, area pratinjau langsung gelap atau kosong, dan baik prompt izin duduk di bilah alamat yang Anda abaikan, atau satu prompt sudah-ditolak telah menyembunyikan dirinya. Mengarahkan tindakan tes aktual ke Tes Kamera Online (di browser via Web API MediaDevices.getUserMedia(); tanpa unggah, tanpa instalasi).

Jawaban 30 detik. Sebagian besar kasus "tes kamera tidak mulai" adalah keadaan izin, bukan kesalahan perangkat keras. Buka bilah alamat, cari ikon kamera kecil (Chrome, Edge), ikon perisai (Firefox), atau gembok "Pengaturan" (Safari). Klik ikon, lalu pilih Izinkan untuk izin kamera freetoolonline.com, dan muat ulang halaman. Jika Anda tidak melihat ikon, prompt telah dijawab sebelumnya dan browser ingat; perbaikannya sama - temukan izin kamera per-situs dan ubah kembali ke Izinkan. Jalur klik persisnya di bawah per browser.

Mengapa pratinjau kamera tetap gelap: tiga keadaan izin dijelaskan

Tes kamera sisi-browser meminta sistem operasi untuk frame video langsung melalui Web API MediaDevices.getUserMedia(). Sebelum halaman dapat melukis satu frame di area pratinjau, user agent (browser Anda) berkonsultasi dengan penyimpanan izin per-asalnya dan memutuskan apakah akan menghormati permintaan. Tiga keadaan akhir dimungkinkan, masing-masing menghasilkan hasil yang berbeda di halaman bagi pembaca yang memegang pratinjau gelap:

  • Diizinkan. Prompt izin baik muncul sekali dan Anda mengklik Izinkan, atau browser mengingat Izinkan sebelumnya dari situs ini dan melewati prompt sepenuhnya. Dalam kedua kasus, area pratinjau langsung merender frame dalam satu atau dua detik setelah halaman menyelesaikan lukisan awalnya.
  • Diblokir (ditolak). Anda menjawab prompt dengan Blokir pada suatu titik - di halaman ini atau di halaman lain dari asal yang sama - dan browser mengingat pilihannya. Tidak ada prompt baru; halaman meminta akses kamera, browser secara diam-diam menolak permintaan, dan pratinjau langsung tetap gelap atau menampilkan pemberitahuan "izin ditolak". Ikon kamera-dengan-garis kecil biasanya terlihat di bilah alamat.
  • Diabaikan (tanpa keputusan). Prompt memang muncul, tetapi Anda mengklik di tempat lain, beralih tab, atau prompt menggulir ke bagian layar yang tidak Anda lihat. Browser mempertahankan permintaan tertunda tanpa berkomitmen pada salah satu keadaan. Pratinjau langsung tetap gelap, tetapi tidak seperti kasus yang diblokir, browser belum mengingat "tidak" - muat ulang halaman berikutnya akan memunculkan prompt lagi.

Mengetahui Anda berada di keadaan mana dari tiga ini menghemat waktu. Jika ikon kecil muncul di bilah alamat, Anda berada di kasus (a) atau (b); jika tidak ada ikon dan halaman tidak mengatakan apa-apa, Anda berada di kasus (c) dan satu muat ulang sering memperbaikinya. Jika ikon menampilkan kamera-dengan-garis, Anda berada di kasus (b) dan langkah-langkah di bawah mengalihkan izin per-situs kembali ke Izinkan.

Chrome (desktop)

Chrome menyimpan izin kamera per asal di profil pengguna. Untuk memberikan atau memberikan kembali akses:

  1. Lihat paling kiri bilah alamat. Jika ikon kamera menampilkan garis merah melaluinya, klik ikon itu.
  2. Di popup kecil, atur Kamera ke Izinkan (kadang berlabel "Selalu izinkan di situs ini"). Tutup popup.
  3. Muat ulang halaman (Ctrl/Cmd + R). Pratinjau langsung harus aktif dalam satu detik.

Jika Anda tidak melihat ikon kamera, izin mungkin sudah diatur ke "Blokir" dan Chrome mungkin menyembunyikan ikon ketika tidak ada permintaan baru-baru ini yang berjalan. Buka chrome://settings/content/camera langsung, gulir ke daftar Tidak diizinkan menggunakan kamera Anda, temukan https://freetoolonline.com, klik ikon tempat sampah di sebelahnya untuk membersihkan aturan, lalu muat ulang halaman tes kamera; Chrome akan memunculkan prompt izin segar saat halaman lain memanggil getUserMedia. Pada perangkat terkelola (kebijakan sekolah atau tempat kerja), akses kamera dapat diblokir oleh kebijakan di tingkat OS atau konsol-admin - dalam hal itu Chrome menampilkan "Dikelola oleh organisasi Anda" dan toggle per-situs berwarna abu-abu.

Firefox (desktop)

Firefox menampilkan izin aktif sebagai ikon kamera kecil di sebelah kiri URL ketika halaman telah meminta atau diberikan akses. Langkah:

  1. Klik ikon kamera di sebelah kiri URL. Panel berlabel Izin terbuka.
  2. Temukan baris Gunakan Kamera. Jika menunjukkan Diblokir Sementara, klik x di sebelah entri untuk membersihkannya, lalu muat ulang halaman; Firefox akan meminta ulang. Jika menunjukkan Diblokir tanpa x, pilihan dibuat permanen - buka about:preferences#privacy, gulir ke Izin → Kamera → Pengaturan, temukan entri freetoolonline.com dalam daftar, ubah ke Izinkan atau hapus, lalu simpan dan muat ulang.
  3. Ketika prompt muncul kembali, klik Izinkan (dan opsional centang Ingat keputusan ini agar Firefox tidak meminta lagi).

Firefox menjaga dua keadaan izin kamera tetap berbeda: Diblokir Sementara sementara yang dihapus dengan satu muat ulang halaman, dan Diblokir permanen yang bertahan melalui muat ulang sampai Anda mengeditnya di bawah about:preferences#privacy → Izin → Kamera → Pengaturan. Sebagian besar pembaca yang mendarat di panduan tes kamera ini menghantam keadaan sementara pada kunjungan pertama dan mengasumsikan webcam dinonaktifkan secara permanen; ikon kamera-dengan-garis kecil Firefox di bilah URL adalah diagnostik yang membedakan dua keadaan sebelum Anda mengubah aturan per-situs apa pun.

Safari (macOS)

Safari mengikat izin kamera ke pengaturan per-situs untuk profil Safari aktif, bukan ke sesi halaman tunggal. Untuk mengizinkan:

  1. Dengan halaman tes kamera terbuka, pilih Safari → Pengaturan untuk freetoolonline.com dari bilah menu (atau Safari → Pengaturan → Situs Web → Kamera).
  2. Atur dropdown kamera untuk freetoolonline.com ke Izinkan.
  3. Tutup panel pengaturan dan muat ulang halaman (Cmd + R).

Jika "Izinkan" tidak ada di dropdown, macOS sendiri mungkin memblokir akses Safari ke kamera di tingkat OS - buka Pengaturan Sistem → Privasi & Keamanan → Kamera, pastikan Safari diaktifkan, lalu kembali ke Safari dan coba lagi. macOS memunculkan prompt sekali pertama kali Safari mencoba menggunakan kamera di halaman mana pun; setelah itu diizinkan, preferensi Safari per-situs mengambil alih.

Edge (desktop)

Edge mengikuti model izin Chromium dengan alur UI yang sama dengan Chrome. Klik ikon kamera di sebelah kiri bilah alamat, pilih Izinkan, muat ulang. Jika ikon tersembunyi, buka edge://settings/content/camera dan bersihkan atau ubah aturan per-situs untuk freetoolonline.com. Edge pada instalasi Windows terkelola juga menghormati Group Policy - jika toggle berwarna abu-abu, admin IT telah mengunci pengaturan kebijakan dan override per-pengguna tidak mungkin dari UI browser.

Izin kamera ponsel - iOS Safari dan Android Chrome

Kamera ponsel duduk di belakang dua gerbang izin pada telepon - satu yang dimiliki browser (Kamera per-asal = Izinkan / Blokir) dan satu kedua yang dimiliki sistem operasi di atasnya (Pengaturan → Privasi & Keamanan → Kamera membiarkan Safari atau Chrome menyentuh lensa sama sekali). Pratinjau gelap di halaman tes kamera setelah Anda mengetuk Izinkan di prompt browser adalah tanda klasik bahwa toggle tingkat-OS adalah pemblokir, bukan aturan per-situs. iOS menambahkan kerutan ketiga: pemilihan kamera depan vs belakang di dalam browser diatur oleh izin Kamera yang sama, tetapi petunjuk facingMode halaman dapat mendarat di lensa yang LED privasinya tidak dapat Anda lihat, jadi muat-ulang-sekali-setelah-izinkan adalah langkah verifikasi yang direkomendasikan.

iOS Safari. Buka Pengaturan → Aplikasi → Safari → Kamera. Atur default untuk "Kamera" ke Izinkan (atau Tanya). Kemudian, dengan halaman tes kamera terbuka, ketuk tombol kecil aA di bilah alamat dan pilih Pengaturan Situs Web → Kamera → Izinkan. Tutup dan buka kembali tab. iOS juga mengharuskan Safari sendiri memiliki akses Kamera di Pengaturan → Privasi & Keamanan → Kamera → Safari - pada perangkat baru atau setelah reset privasi ini dapat dimatikan.

Android Chrome. Ketuk ikon kunci atau info di sebelah kiri URL, lalu Izin → Kamera → Izinkan. Jika toggle hilang, buka Pengaturan → Pengaturan situs → Kamera, temukan entri freetoolonline.com di bawah "Diblokir", bersihkan, lalu muat ulang halaman sehingga Chrome meminta ulang. OS Android juga memiliki lapisannya sendiri - Pengaturan → Aplikasi → Chrome → Izin → Kamera harus diatur ke Izinkan hanya saat menggunakan aplikasi (default) untuk situs apa pun yang menggunakan kamera.

Reset keadaan kamera-diblokir yang macet ketika ikon garis tidak hilang

Diagnostik paling andal untuk "Saya terus ditolak" adalah ikon kamera-dengan-garis kecil di bilah alamat - itu berarti browser mengingat keputusan Blokir sebelumnya terhadap asal freetoolonline.com dan secara diam-diam menolak panggilan getUserMedia di masa depan tanpa memunculkan prompt baru. Menutup tab, memuat ulang halaman, atau bahkan keluar dari browser tidak akan menghapus keputusan ini karena aturan kamera per-situs disimpan terhadap asal, bukan terhadap tab atau sesi. Perbaikannya adalah menghapus entri per-situs yang diingat sehingga kunjungan berikutnya memicu prompt izin yang benar-benar baru, setelah itu ikon kamera kembali ke biasa (diizinkan) dan pratinjau langsung aktif tanpa me-restart browser. Setiap browser desktop mengekspos entri kamera per-situs di tempat yang sedikit berbeda:

  • Chrome / Edge: chrome://settings/content/camera atau edge://settings/content/camera → hapus entri freetoolonline.com di bawah "Tidak diizinkan menggunakan kamera Anda".
  • Firefox: about:preferences#privacy → Izin → Kamera → Pengaturan → hapus entri.
  • Safari (macOS): Safari → Pengaturan → Situs Web → Kamera → pilih entri dan klik Hapus.

Setelah membersihkan, muat ulang halaman tes kamera; browser memperlakukan panggilan getUserMedia berikutnya sebagai permintaan pertama-kali dan memunculkan prompt segar. Ini juga rute yang tepat ketika akun komputer bersama telah meninggalkan keputusan Blokir yang basi dari pengguna sebelumnya.

Privasi dan apa yang sebenarnya dilakukan tes kamera

Tes kamera browser di situs ini merender pratinjau langsung sepenuhnya di browser Anda. Halaman memanggil navigator.mediaDevices.getUserMedia({ video: true }) untuk meminta aliran video, melampirkan aliran ke elemen <video> tersembunyi, dan membaca frame ke <canvas> untuk ditampilkan - semuanya di perangkat Anda. Tidak ada frame yang diunggah, tidak ada rekaman yang disimpan, dan menghentikan tes melepaskan aliran dan lampu kamera padam. Jika lampu kamera tetap menyala setelah Anda meninggalkan halaman, itu hampir selalu tab lain atau aplikasi lain memegang perangkat, bukan halaman ini; menutup tab yang menyinggung melepaskan kamera dalam satu detik.

Izin yang Anda berikan dari ikon kamera-dengan-garis di bilah alamat hanya mengizinkan halaman pada freetoolonline.com untuk memanggil getUserMedia({ video: true }) saat tab dari asal ini aktif. Hibah tidak berlaku lintas asal, tidak meluas ke sub-halaman domain lain, dan browser mencabutnya secara otomatis pada saat Anda membalik pengaturan per-situs kembali ke "Tanya" atau "Blokir". Menutup tab juga melepaskan pratinjau langsung - lampu indikator kamera mati segera setelah aliran berhenti, jadi akses tersisa dibatasi oleh masa hidup tab daripada panjang sesi. Kebijakan Privasi mendokumentasikan kontrak tanpa-unggah, tanpa-rekam untuk permukaan tes kamera dalam bahasa yang jelas.
