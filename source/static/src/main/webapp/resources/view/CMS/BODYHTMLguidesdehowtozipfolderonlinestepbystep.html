Wie man einen Ordner online zippt, Schritt fuer Schritt

Zuletzt geprueft 2026-04-27. Oeffnen Sie das Zip-Tool, um einen Ordner in Ihrem Browser zu komprimieren, ohne ihn hochzuladen.

30-Sekunden-Antwort. Auf dem Desktop ziehen Sie den Ordner auf das Zip-Tool - der Browser durchlaeuft den Verzeichnisbaum und behaelt ihn im Archiv bei. Auf dem Handy koennen Browser Ordner nicht direkt lesen, daher ist der Workaround, lange auf den Ordner zu tippen, "compress" zu waehlen (in iOS und modernem Android eingebaut), oder den Ordner ueber einen Datei-Manager zu ziehen. Der Desktop-Weg ist schneller und funktioniert in jedem modernen Browser.

Fuenf Schritte auf dem Desktop

  1. Ordner waehlen. Ziehen Sie den Ordner in die Ablagezone bei dem Zip-Tool. Der Browser verwendet die File System Access API oder einen Verzeichnis-Picker-Fallback und liest jede Datei darin, einschliesslich verschachtelter Unterordner.
  2. Struktur behalten. Das Tool behaelt standardmaessig das Ordner-Layout im Zip bei. Empfaenger entpacken und sehen my-folder/sub/file.txt genau so, wie Sie es gesendet haben. Nicht abflachen, ausser Sie brauchen ein flaches Layout - Entpack-Tools handhaben verschachtelte Strukturen nativ, und die Metadaten machen das Archiv selbsterklaerend.
  3. Kompression einstellen. Standard (Stufe 6) ist richtig fuer gemischte Inhalte. Beste (Stufe 9) hilft bei Ordnern, die von Dokumenten, Quellcode oder Logs dominiert werden. Speichern (Stufe 0) ist die richtige Wahl fuer Ordner mit Fotos, Videos oder Audio - alles darin ist bereits komprimiert.
  4. Ausgabe benennen. Verwenden Sie einen aussagekraeftigen Dateinamen: project-handover-2026-04.zip schlaegt archive.zip. Vermeiden Sie Leerzeichen und Sonderzeichen beim Versenden per E-Mail - manche Clients kodieren den Dateinamen immer noch neu.
  5. Herunterladen. Das Zip wird lokal erstellt. Der Originalordner bleibt unberuehrt.

Der Workaround fuer das Handy

Mobile Browser koennen einer Website keinen direkten Zugriff auf einen Ordner geben - sie zeigen nur einzelne Dateien an. Drei funktionierende Wege:

  • iOS Files-App. Lange auf den Ordner tippen, Compress waehlen. Die Ausgabe ist ein Zip am gleichen Ort. Oeffnen Sie es in einer beliebigen App oder teilen Sie es.
  • Android. Die meisten Datei-Manager-Apps (Files by Google, Solid Explorer, MiXplorer) enthalten eine Compress-Aktion mit der gleichen Form - lange tippen, Compress.
  • Mehrfachauswahl auf Browser-Seite. Waehlen Sie alle Dateien innerhalb des Ordners aus (einzeln in iOS Safari, Mehrfachauswahl in den meisten Android-Browsern) und legen Sie sie im Zip-Tool ab. Sie verlieren die verschachtelte Struktur, ausser Sie erstellen sie manuell neu.

Groessen-Fallen zu vermeiden

Ein Ordner mit einer riesigen Datei darin (ein Video, ein Datenbank-Dump) komprimiert sich kaum, weil diese Datei der Flaschenhals ist. Komprimieren oder kuerzen Sie sie zuerst. Ein Ordner mit 20.000 winzigen Dateien komprimiert stark, erstellt aber ein langsames Archiv - manche Entpack-Tools timen bei Archiven mit zu vielen Eintraegen aus. Wenn Sie dieses Muster sehen, gruppieren Sie kleine Dateien in Unterordner oder verpacken Sie sie vorher in ein tar, bevor Sie zippen.

Zip vor dem Versenden pruefen

Oeffnen Sie das heruntergeladene Zip lokal, um die Struktur zu bestaetigen und dass jede erwartete Datei darin ist. Eine Datei entpacken im Browser listet den Inhalt ohne Extraktion auf. Fuer den Hintergrund zu Format-Trade-offs siehe online zip vs 7z vs rar oder den Zip-Tools-Hub.
