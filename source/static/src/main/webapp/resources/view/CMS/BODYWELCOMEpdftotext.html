Why some PDFs extract as blank: text layer vs scanned image

A PDF can look identical on screen yet produce either a perfect paragraph of text or an empty result. The difference is whether the file has a real text layer or only a picture of text. PDFs exported from Word, Pages, Google Docs, and design tools carry a text layer that the extractor reads directly, preserving words, line breaks, and reading order. Scanned PDFs - receipts photographed with a phone, contracts fed through a flatbed, archived research papers - store each page as an image. To a text extractor those pages are pixels, not characters, so the output is blank. If you get an empty result, open the PDF and try to select a single word with your cursor: if nothing highlights, there is no text layer to extract.


When to reach for OCR instead

Scanned paper documents. Any PDF that originated as a photo or a flatbed scan needs OCR (optical character recognition) before the words become selectable text. Run the file through a dedicated OCR tool (Adobe Acrobat, Tesseract, ABBYY FineReader) first, then return here to extract the now-readable text.

Hybrid PDFs. Some archives wrap a scanned page inside a text-layer container. The extraction may return page numbers or headers but drop the body - that is a sign only part of the file was OCR'd.

Handwritten notes. OCR still struggles with free-form handwriting. For lecture notes or signed forms, a handwriting-specific model (for example, Google's Document AI) will outperform generic OCR.

Tables and forms. If your goal is to pull structured data out of a PDF form, prefer a form-aware parser. A plain text extract loses rows, columns, and checkbox state.


Preserving structure (headings, lists, table rows)

Plain-text output is lossy by design. Most PDF text layers encode absolute positions per glyph rather than logical paragraphs, so the extractor infers reading order from those positions. Multi-column layouts, sidebars, and footnotes occasionally come out interleaved. Use this tool for search, cut-and-paste into a doc, or feeding text into another program. If you need the hierarchy (headings, lists, table rows) to survive, export the source document as HTML from its original editor, or use a structure-aware PDF library (pdfplumber, pdf.js) in a small script.


Privacy

When you close the tab, the file and the extracted text are gone. That makes this tool safe for contracts, draft documents, and internal reports that shouldn't leave your device.
