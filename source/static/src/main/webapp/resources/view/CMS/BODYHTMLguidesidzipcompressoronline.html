Kompresor ZIP Online

Kompresor zip online adalah versi berbasis browser dari langkah "klik kanan lalu Kompres" di desktop: pilih folder file, klik satu tombol, dan halaman mengembalikan satu arsip .zip yang bisa Anda kirim melalui email, unggah, atau bagikan melalui tautan. Alasan untuk melakukannya di browser alih-alih desktop adalah kenyamanan tanpa instalasi - bekerja sama di Windows, macOS, ChromeOS, dan dari laptop terkelola di mana Anda tidak dapat menginstal alat desktop.

Kompresor berbasis browser di situs ini berada di /zip-tools/zip-file.html. Halaman panduan ini adalah tempat konteks keputusan berada - kapan arsip benar-benar menyusut, kapan tidak, dan pengaturan opsional mana yang penting; halaman alat adalah tempat unggahan terjadi dan arsip dibangun.

Kapan arsip benar-benar menyusut

Kompresor zip menghemat ruang hanya saat file input belum dikompresi oleh format mereka sendiri. Teks biasa, kode sumber, ekspor CSV, file log, dump JSON, dan bitmap tidak terkompresi semuanya mengompresi dengan baik di dalam .zip - folder berisi ini sering menyusut setengah atau lebih setelah pembangunan. Format yang sudah terkompresi - foto JPG, tangkapan PNG, video MP4, audio MP3, dan file .zip yang ada yang ditempatkan di dalam .zip lain - biasanya tumbuh dari nol hingga beberapa persen karena header arsip dan metadata per entri ditambahkan tanpa kompresi lebih lanjut yang mungkin pada muatan. Jika folder Anda didominasi foto atau video, harapkan hasil berada dalam beberapa persen dari byte input; manfaatnya adalah menggabungkan banyak file menjadi satu arsip yang bisa diunduh, bukan penyusutan.

Kata sandi opsional dan enkripsi AES

Kompresor zip berbasis browser mendukung kata sandi opsional dan metode enkripsi saat arsip perlu melewati saluran di mana kata sandi dibagikan secara terpisah. Buka modal Pengaturan di /zip-tools/zip-file.html untuk mengatur kata sandi dan memilih salah satu Standar (enkripsi ZIP lawas, kompatibel luas tetapi lemah secara kriptografis), AES-128, atau AES-256. Untuk konten sensitif, lebih suka AES-256 dan bagikan kata sandi melalui saluran berbeda dari arsip itu sendiri. Jika Anda hanya perlu memulihkan kata sandi yang diketahui dari arsip yang ada, alat saudara Remove ZIP password membuka kunci saat kata sandi diketahui; ia tidak memecahkan kata sandi yang tidak diketahui pada arsip enkripsi yang kuat.

Bagaimana pembangunan bekerja

File diunggah melalui HTTPS ke layanan pemrosesan berumur pendek yang merakit arsip dan mengembalikannya untuk diunduh; file sumber otomatis dihapus setelah jendela retensi singkat dengan tidak ada penyimpanan persisten input. Penamaan dapat dikonfigurasi di modal Pengaturan - default adalah myzipped-<tanggal>, tetapi mengganti nama sebelum mengunduh berarti penerima melihat proyek atau topik dalam nama file alih-alih cap tanggal.

Apa yang tidak dicakup panduan ini

Panduan ini tidak memandu pemilihan tingkat kompresi karena alat di browser menjalankan langkah DEFLATE standar secara default dan slider tingkat sebagian besar mengubah waktu pembangunan daripada ukuran arsip. Ia juga tidak mencakup format arsip pesaing (.7z, .rar, .tar.gz) - situs default ke .zip untuk jangkauan, karena setiap OS desktop membukanya tanpa instalasi dan sebagian besar pemindai lampiran webmail menerimanya. Untuk alur kerja sisi penerima (cara mengekstrak .zip setelah tiba), lihat /unzip-file.html atau gunakan unzip yang dibangun ke dalam sistem operasi Anda.

Pertanyaan umum

Apakah kompresor zip online mempertahankan struktur folder?

Ya. Arsip mempertahankan nama file dan struktur direktori relatif dari input secara verbatim. Pada perjalanan bolak-balik lintas sistem operasi, ia tidak mempertahankan kepemilikan file atau atribut yang diperluas (kepemilikan file Linux, resource fork macOS); tata letak folder yang terlihat identik di sisi penerima.

Apa batas ukuran per arsip?

Kompresor berbasis browser di /zip-tools/zip-file.html menggabungkan set file apa pun yang Anda pilih dalam satu unggahan; arsip yang sangat besar mendapat manfaat dari browser desktop di mana memori kurang dibatasi. Jika satu arsip melebihi batas lampiran webmail penerima (biasanya 20 hingga 25 MB untuk Gmail dan sebagian besar server email korporat), bagi paket: zip input yang berat teks secara terpisah untuk kemenangan ukuran yang nyata, dan kirim media yang sudah terkompresi sebagai lampiran terpisah atau melalui tautan berbagi.

Apakah file saya disimpan di server?

File dikirim melalui HTTPS ke layanan pemrosesan berumur pendek yang membangun arsip dan mengembalikannya untuk diunduh. File sumber otomatis dihapus setelah jendela retensi singkat; arsip tidak diindeks atau digunakan kembali. Tidak ada akun atau masuk yang diperlukan.

Bagaimana jika arsip membutuhkan kata sandi?

Kompresor berbasis browser mendukung kata sandi opsional dengan enkripsi Standar, AES-128, atau AES-256 - buka modal Pengaturan di /zip-tools/zip-file.html untuk mengaturnya sebelum mengklik Zip. Untuk arah sebaliknya (membuka kunci arsip yang ada yang kata sandinya Anda ketahui), gunakan Remove ZIP password. Tidak ada alat online di situs ini yang memecahkan kata sandi yang tidak diketahui pada arsip enkripsi yang kuat; pemulihan untuk kata sandi yang dilupakan tergantung pada memiliki file sumber untuk membuat ulang arsip atau bertanya kepada pengirim.

Apa perbedaan antara panduan ini dan panduan "kompresor zip" telanjang?

Panduan ini menjawab kueri "kompresor zip online" secara khusus - pembaca menginginkan alat berbasis browser, bukan instalasi desktop. Panduan saudara di /guides/id/zip-compressor.html menjawab kueri "kompresor zip" telanjang dengan konteks trade-off ukuran yang sama. Kedua panduan saling menautkan ke pembuat berbasis browser yang sama di /zip-tools/zip-file.html; kata-kata yang Anda ketik memilih panduan mana yang dibaca secara alami, tetapi tidak pernah memilih di mana arsip dibangun.
