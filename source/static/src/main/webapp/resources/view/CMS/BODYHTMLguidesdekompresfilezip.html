ZIP-Datei komprimieren

ZIP-Datei komprimieren ist der Weg, eine Datei, mehrere Dateien oder einen ganzen Ordner in eine einzige .zip-Datei im Browser zu packen. Du laedst die Dateien hoch, der Dienst buendelt sie auf dem Server ueber HTTPS, die Quelldateien werden nach einer kurzen Pause automatisch geloescht, und das Archiv kommt als einzelner Download zurueck.

Was das ZIP-Komprimierungswerkzeug tatsaechlich tut

Das Werkzeug unter /zip-file.html erledigt drei Hauptaufgaben auf einer Seite. Erstens buendelt es: eine Datei, eine Sammlung von Dateien oder ein vollstaendiger Ordner (Unterordner werden beibehalten) verwandelt sich in eine einzige .zip-Datei, die der Empfaenger mit einem Klick herunterlaedt. Zweitens benennt es das Archiv ueber das Einstellungs-Panel, damit der Empfaenger das Thema oder den Projektnamen im Dateinamen sieht statt eines generischen Zeitstempels. Drittens kann es das Archiv mit einem optionalen Passwort sperren: waehle die Verschluesselungsmethode Standard, AES-128 oder AES-256, wenn das Passwort ueber einen vom Archiv getrennten Kanal gesendet wird. Die gesamte Verarbeitung findet auf unseren Servern ueber eine HTTPS-Verbindung statt, und die Quelldateien werden nach einer kurzen Aufbewahrungspause automatisch geloescht.

Verkleinert das Komprimieren von ZIP-Dateien wirklich die Datei

Das Ausgabearchiv von /zip-file.html ist nur dann kleiner als die Summe seiner Eingaben, wenn die Dateien, die du zusammenfuegst, tatsaechlich vom ZIP-Deflate-Algorithmus komprimiert werden koennen. Wenn du lange Textdateien, taegliche Logs, CSV-Exporte, Quellcode oder Office-Dokumente buendelst, ermoeglicht die wiederholte Bytefolge darin ZIP, die Ausgabe auf den Bereich von 30 bis 60 Prozent der Quellgroesse zu komprimieren. Umgekehrt sind Eingaben, die bereits komprimierte Medien sind - JPG-Digitalkameras, MP4-Videoclips, MP3-Songs oder PDF-Anhaenge, deren Inhalt Bilder sind - bereits vor ihrer Ankunft hier von ihren jeweiligen Codecs komprimiert worden, sodass ZIP sie nur umhuellt: die endgueltige Groesse bewegt sich etwa 0 bis 5 Prozent und nimmt manchmal aufgrund des Overheads von Dateinamen und der Archivtabelle leicht zu. Die Zusammenfassung bleibt eine: ZIP komprimiert, was noch nicht komprimiert ist; ZIP umhuellt, was bereits komprimiert ist. Wenn dein Ziel darin besteht, die 25 MB E-Mail-Anhang-Grenze oder ein anderes Upload-Limit einzuhalten, verkleinere zuerst schwere Medien im Bild- oder Video-Komprimierungswerkzeug, dann kombiniere die Ergebnisse zu einem einzigen ZIP-Archiv auf dieser Seite.

Datenschutz und Datenhandhabung

Diese Seite laeuft ohne Konto: du musst dich nicht registrieren, anmelden oder eine Browser-Erweiterung installieren, bevor du das Archiv buendelst. Eingabedateien werden ueber HTTPS an unsere Verarbeitungsserver gesendet, das .zip-Archiv wird dort erstellt, und die Quelldateien werden nach einer kurzen Aufbewahrungspause automatisch geloescht, damit sie sich auf der Serverseite nicht ansammeln. Der erzeugte Download-Link verweist nur auf das endgueltige Archiv - nicht auf eine Kopie deiner Eingaben. Fuer Archive, die einen vertraulichen Freigabekanal durchlaufen, aktiviere das Passwort und waehle die Methode AES-128 oder AES-256 im Einstellungs-Panel, dann sende das Passwort ueber einen anderen Kanal als das Archiv, damit ein Leser, der eine Seite abfaengt, die andere Seite nicht automatisch oeffnet.

Wie man ZIP-Dateien im Browser komprimiert, Schritt fuer Schritt

  1. Oeffne /zip-file.html in einem neuen Tab.
  2. Ziehe oder waehle die Dateien aus, die du buendeln moechtest. Ein Ordner wird ebenfalls akzeptiert; Unterordner bleiben innerhalb des Archivs erhalten.
  3. Optional: oeffne Einstellungen, benenne das Archiv und fuege ein Passwort plus die Verschluesselungsmethode (Standard, AES-128 oder AES-256) hinzu, wenn der Empfaenger bereits einen separaten Kanal fuer das Passwort hat.
  4. Klicke auf den Zip-Knopf. Die Seite laedt die Eingaben ueber HTTPS hoch, erstellt das Archiv auf dem Server und zeigt den Download-Link an.
  5. Speichere die resultierende .zip-Datei. Wenn du ein Passwort festgelegt hast, sende es ueber einen anderen Weg als das Archiv.

Fragen, die haeufig auftauchen

Muss ich mich registrieren oder eine App installieren?

Nein. Diese Seite laeuft im Browser und verlangt kein Konto. Du ziehst nur Dateien hinein, drueckst Zip und laedst das Ergebnis herunter.

Sind meine Dateien sicher?

Uploads werden ueber HTTPS an den Verarbeitungsserver gesendet, das Archiv wird serverseitig erstellt, und die Quelldateien werden nach einer kurzen Aufbewahrungspause automatisch geloescht. Fuer eine zusaetzliche Schicht verwende ein Passwort mit der Methode AES-128 oder AES-256 und sende das Passwort ueber einen separaten Kanal.

Warum hat sich die Groesse meines Archivs nicht stark reduziert?

Hoechstwahrscheinlich enthalten deine Eingaben Medien, die bereits von ihren jeweiligen Codecs komprimiert wurden (JPG, MP4, MP3 oder PDF mit Bildern). ZIP umhuellt solche Dateien nur, ohne sie viel zu schrumpfen. Komprimiere zuerst schwere Medien mit einem Bild- oder Video-Werkzeug, dann buendele die Ergebnisse.
