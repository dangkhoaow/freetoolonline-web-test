Kompres Folder Online

Mengompres folder online berarti menyerahkan seluruh folder ke alat web dan mendapatkan kembali satu arsip .zip, siap dikirim via email atau diunggah sebagai satu file. Pembuat di /zip-file.html menerima folder via HTTPS, membangun arsip di layanan unggah-lalu-bangun privat, dan mengembalikan .zip ke browser Anda. Subfolder dipertahankan, jadi penerima mengekstrak arsip kembali ke layout direktori yang sama dengan yang Anda kirim.

Resep tiga langkah

Buka pembuat, seret folder ke zona unggah, dan klik Convert. Struktur folder tetap utuh di dalam arsip hasil, properti yang layak dikonfirmasi saat Anda menyerahkan proyek: penerima membongkar .zip dan menemukan layout bersarang yang sama dengan yang Anda miliki di disk, bukan tumpukan datar setiap file dari setiap subfolder. Beri nama arsip di modal Pengaturan sebelum membangunnya agar penerima melihat sesuatu yang jelas saat tiba ("keuangan-q3.zip" mengalahkan "archive.zip"). Outputnya adalah satu file .zip, disajikan kembali ke browser Anda sebagai unduhan.

Kapan Anda butuh kata sandi pada arsip

Jika folder melintas melalui kanal di mana Anda tidak bisa sepenuhnya mempercayai daftar penerima - email yang diteruskan, kotak masuk berbagi, grup chat - modal Pengaturan menampilkan field kata sandi opsional dengan tiga pilihan enkripsi. Enkripsi Standard ZIP diterima secara universal tetapi lemah menurut standar saat ini. AES-128 adalah default modern, diterima Windows Explorer (klik kanan Extract All) dan Archive Utility macOS. AES-256 adalah tingkat tertinggi dengan plafon kompatibilitas yang sama dengan AES-128. Kirim kata sandi melalui kanal terpisah - mengirim kata sandi dan file bersama-sama menggagalkan tujuan. Kata sandi diterapkan saat build, jadi penerima diminta saat ekstrak.

Apa yang tidak dilakukan pembuat

Pembuat membangun arsip baru - ia tidak memecahkan enkripsi pada .zip yang orang lain bangun. Ia tidak menghasilkan 7z atau RAR (keduanya lebih padat daripada ZIP tetapi memerlukan instalasi pihak ketiga di sebagian besar mesin penerima, jadi pembuat tetap pada ZIP untuk jangkauan lintas platform). Ia tidak menghapus metadata dari file input: EXIF pada JPG, field penulis pada dokumen Word, dan stempel waktu modifikasi semuanya berjalan ke arsip tanpa berubah. Jika metadata itu perlu dibersihkan, lakukan sebelum mengunggah folder.
