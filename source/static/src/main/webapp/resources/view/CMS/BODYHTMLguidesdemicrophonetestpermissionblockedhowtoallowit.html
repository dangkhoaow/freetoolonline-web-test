Mikrofon-Test-Berechtigung Blockiert: Wie Sie Mic-Zugriff In Ihrem Browser Erlauben

Zuletzt geprueft 2026-05-06. Gezielt auf den Moment, in dem eine Browser-Mikrofon-Test-Seite laedt, der Pegelanzeiger flach bleibt, waehrend Sie sprechen, und entweder eine Berechtigungsaufforderung in der Adressleiste sitzt, die Sie ignoriert haben, oder eine bereits abgelehnte sich versteckt hat. Leitet die eigentliche Test-Aktion zu Microphone Test Online (im Browser via Web API MediaDevices.getUserMedia(); kein Upload, keine Installation).

30-Sekunden-Antwort. Die meisten "der Mikrofon-Test reagiert nicht"-Faelle sind ein Berechtigungszustand, kein Hardware-Fehler. Oeffnen Sie die Adressleiste, suchen Sie ein kleines Mikrofon-Symbol (Chrome, Edge), Schild-Symbol (Firefox) oder "Einstellungen"-Schloss (Safari). Klicken Sie darauf, dann waehlen Sie Zulassen fuer die Mikrofon-Berechtigung von freetoolonline.com, und laden Sie die Seite neu. Wenn Sie das Symbol nicht sehen, wurde die Aufforderung frueher beantwortet und der Browser erinnert sich; die Loesung ist dieselbe - finden Sie die per-Site-Mikrofon-Berechtigung und schalten Sie sie zurueck auf Zulassen. Der exakte Klickpfad ist unten pro Browser.

Berechtigung blockiert vs ueberhaupt keine Aufforderung

Ein Browser-Mikrofon-Test fragt das Betriebssystem nach Live-Audio-Frames von einem Aufnahmegeraet. Bevor die Seite irgendein Sample lesen kann, muss der User Agent (Ihr Browser) eine Berechtigungspruefung durchfuehren. Drei Endzustaende sind moeglich, und sie sehen auf der Seite unterschiedlich aus:

  • Erlaubt. Die Berechtigungsaufforderung erscheint einmal und Sie klicken Zulassen, oder der Browser erinnert sich an ein frueheres Zulassen von dieser Site und ueberspringt die Aufforderung vollstaendig. In beiden Faellen beginnt der Pegelanzeiger innerhalb einer oder zwei Sekunden nach Abschluss des anfaenglichen Paintings der Seite, auf Ihre Stimme zu reagieren.
  • Blockiert (abgelehnt). Sie haben die Aufforderung irgendwann mit Blockieren beantwortet - auf dieser Seite oder auf einer anderen Seite desselben Ursprungs - und der Browser hat sich die Wahl gemerkt. Es gibt keine neue Aufforderung; die Seite bittet um Mikrofon-Zugriff, der Browser lehnt die Anfrage still ab, und der Pegelanzeiger bleibt flach oder zeigt einen "Berechtigung verweigert"-Hinweis. Ein kleines Mikrofon-mit-Schraegstrich-Symbol ist normalerweise in der Adressleiste sichtbar.
  • Ignoriert (keine Entscheidung). Die Aufforderung erschien, aber Sie klickten woanders, wechselten Tabs oder die Aufforderung scrollte in einen Bildschirmbereich, den Sie nicht angesehen haben. Der Browser haelt die Anfrage ausstehend, ohne sich zu einem Zustand zu verpflichten. Der Pegelanzeiger bleibt flach, aber im Gegensatz zum blockierten Fall erinnert sich der Browser noch nicht an "nein" - das naechste Neuladen der Seite wird die Aufforderung erneut anzeigen.

Zu wissen, in welchem dieser drei Zustaende Sie sind, spart Zeit. Wenn ein kleines Symbol in der Adressleiste erscheint, sind Sie in Fall (a) oder (b); wenn es kein Symbol gibt und die Seite nichts sagt, sind Sie in Fall (c) und ein einziges Neuladen behebt es oft. Wenn das Symbol das Mikrofon-mit-Schraegstrich zeigt, sind Sie in Fall (b) und die Schritte unten schalten die per-Site-Berechtigung zurueck auf Zulassen.

Chrome (Desktop)

Chrome speichert die Mikrofon-Berechtigung pro Origin im Benutzerprofil. Um Zugriff zu gewaehren oder erneut zu gewaehren:

  1. Schauen Sie ganz links in der Adressleiste. Wenn das Mikrofon-Symbol einen roten Schraegstrich durchgezogen hat, klicken Sie auf dieses Symbol.
  2. Im kleinen Popup setzen Sie Mikrofon auf Zulassen (manchmal beschriftet "Immer auf dieser Site zulassen"). Schliessen Sie das Popup.
  3. Laden Sie die Seite neu (Ctrl/Cmd + R). Sprechen Sie in Ihr Mikrofon; der Pegelanzeiger sollte innerhalb einer Sekunde springen.

Wenn Sie das Mikrofon-Symbol nicht sehen, kann die Berechtigung bereits auf "Blockieren" gesetzt sein und Chrome kann das Symbol verstecken, wenn keine kuerzliche Anfrage gefeuert hat. Oeffnen Sie chrome://settings/content/microphone direkt, scrollen Sie zur Liste Nicht erlaubt, Ihr Mikrofon zu verwenden, finden Sie https://freetoolonline.com, klicken Sie auf das Papierkorb-Symbol daneben, um die Regel zu loeschen, dann laden Sie die Mikrofon-Test-Seite neu; Chrome wird beim naechsten Mal, wenn die Seite getUserMedia aufruft, eine frische Berechtigungsaufforderung anzeigen. Auf verwalteten Geraeten (Schul- oder Arbeitsplatzrichtlinie) kann der Mikrofon-Zugriff auf OS- oder Admin-Konsolen-Ebene policy-blockiert sein - in diesem Fall zeigt Chrome "Von Ihrer Organisation verwaltet" und der per-Site-Toggle ist ausgegraut.

Firefox (Desktop)

Firefox zeigt die aktive Berechtigung als kleines Mikrofon-Symbol links von der URL, wenn eine Seite Zugriff angefordert oder erhalten hat. Schritte:

  1. Klicken Sie auf das Mikrofon-Symbol links von der URL. Ein Panel mit der Beschriftung Berechtigungen oeffnet sich.
  2. Finden Sie die Zeile Mikrofon verwenden. Wenn sie Voruebergehend Blockiert zeigt, klicken Sie auf das x neben dem Eintrag, um ihn zu loeschen, dann laden Sie die Seite neu; Firefox wird erneut nachfragen. Wenn sie Blockiert ohne x zeigt, wurde die Wahl dauerhaft gemacht - oeffnen Sie about:preferences#privacy, scrollen Sie zu Berechtigungen → Mikrofon → Einstellungen, finden Sie den freetoolonline.com-Eintrag in der Liste, aendern Sie ihn zu Zulassen oder entfernen Sie ihn, dann speichern und neu laden.
  3. Wenn die Aufforderung wieder erscheint, klicken Sie Zulassen (und optional Diese Entscheidung merken ankreuzen, damit Firefox nicht erneut fragt).

Firefox trennt "voruebergehend blockiert" (Entscheidung fuer eine Seitenladung) von "dauerhaft blockiert" (Entscheidung auf Origin-Ebene). Die meisten Benutzer treffen den voruebergehenden Fall bei einem ersten Besuch und merken nicht, dass ein einziges Neuladen der Seite erneut gefragt haette. Der dauerhafte Fall erfordert den about:preferences-Pfad oben.

Safari (macOS)

Safari bindet die Mikrofon-Berechtigung an die per-Website-Einstellungen fuer das aktive Safari-Profil, nicht an eine einzelne Seitensitzung. Um zuzulassen:

  1. Mit der Mikrofon-Test-Seite geoeffnet, waehlen Sie Safari → Einstellungen fuer freetoolonline.com aus der Menueleiste (oder Safari → Einstellungen → Websites → Mikrofon).
  2. Setzen Sie das Mikrofon-Dropdown fuer freetoolonline.com auf Zulassen.
  3. Schliessen Sie das Einstellungs-Panel und laden Sie die Seite neu (Cmd + R).

Wenn "Zulassen" nicht im Dropdown ist, blockiert macOS selbst moeglicherweise Safaris Zugriff auf das Mikrofon auf OS-Ebene - oeffnen Sie Systemeinstellungen → Datenschutz und Sicherheit → Mikrofon, stellen Sie sicher, dass Safari aktiviert ist, dann gehen Sie zurueck zu Safari und versuchen Sie es erneut. macOS zeigt eine einmalige Aufforderung beim ersten Mal, wenn Safari versucht, das Mikrofon auf irgendeiner Seite zu verwenden; sobald das zugelassen ist, uebernehmen die per-Site-Praeferenzen von Safari.

Edge (Desktop)

Edge folgt dem Chromium-Berechtigungsmodell mit demselben UI-Flow wie Chrome. Klicken Sie auf das Mikrofon-Symbol links von der Adressleiste, waehlen Sie Zulassen, neu laden. Wenn das Symbol versteckt ist, oeffnen Sie edge://settings/content/microphone und loeschen oder aendern Sie die per-Site-Regel fuer freetoolonline.com. Edge auf verwalteten Windows-Installationen respektiert auch Group Policy - wenn der Toggle ausgegraut ist, hat der IT-Admin die Einstellung policy-gesperrt und ein per-Benutzer-Override aus der Browser-UI ist nicht moeglich.

iOS Safari und Android Chrome (mobil)

Mobile Browser fuegen eine OS-Ebene vor der per-Site-Berechtigung hinzu, also kann ein "blockierter" Mikrofon-Test auf Mobile zwei unterschiedliche Loesungen haben - eine im Browser, eine in den OS-Einstellungen.

iOS Safari. Oeffnen Sie Einstellungen → Apps → Safari → Mikrofon. Setzen Sie den Standard fuer "Mikrofon" auf Zulassen (oder Fragen). Dann tippen Sie mit der Mikrofon-Test-Seite geoeffnet auf den kleinen aA-Button in der Adressleiste und waehlen Website-Einstellungen → Mikrofon → Zulassen. Schliessen Sie das Tab und oeffnen Sie es erneut. iOS erfordert auch, dass Safari selbst Mikrofon-Zugriff hat in Einstellungen → Datenschutz und Sicherheit → Mikrofon → Safari - auf einem frischen Geraet oder nach einem Datenschutz-Reset kann das ausgeschaltet sein.

Android Chrome. Tippen Sie auf das Schloss- oder Info-Symbol links von der URL, dann Berechtigungen → Mikrofon → Zulassen. Wenn der Toggle fehlt, oeffnen Sie Einstellungen → Site-Einstellungen → Mikrofon, finden Sie den freetoolonline.com-Eintrag unter "Blockiert", loeschen Sie ihn, dann laden Sie die Seite neu, damit Chrome erneut nachfragt. Das Android-OS hat auch seine eigene Ebene - Einstellungen → Apps → Chrome → Berechtigungen → Mikrofon muss auf Nur waehrend der App-Nutzung zulassen (der Standard) gesetzt sein, damit irgendeine Site das Mikrofon verwenden kann.

Wenn Sie versehentlich weiter ablehnen: saubere Zuruecknahme und Neu-Gewaehrung

Wenn Sie sich in verschiedenen Browsern durch mehrere "Blockieren"-Entscheidungen geklickt haben und ein sauberes Reset wollen, ist der zuverlaessigste Weg, die per-Site-Berechtigung vollstaendig zu loeschen, damit der Browser bei der naechsten Seitenladung frisch fragt. Jeder Browser hat eine leicht unterschiedliche Einstellungs-URL:

  • Chrome / Edge: chrome://settings/content/microphone oder edge://settings/content/microphone → entfernen Sie den freetoolonline.com-Eintrag unter "Nicht erlaubt, Ihr Mikrofon zu verwenden".
  • Firefox: about:preferences#privacy → Berechtigungen → Mikrofon → Einstellungen → entfernen Sie den Eintrag.
  • Safari (macOS): Safari → Einstellungen → Websites → Mikrofon → waehlen Sie den Eintrag und klicken Sie Entfernen.

Nach dem Loeschen laden Sie die Mikrofon-Test-Seite neu; der Browser behandelt den naechsten getUserMedia-Aufruf als Erstanfrage und zeigt eine frische Aufforderung. Das ist auch der richtige Weg, wenn gemeinsame Computer-Konten eine veraltete Blockieren-Entscheidung von einem frueheren Benutzer hinterlassen haben.

Wenn die Berechtigung erlaubt ist, aber der Pegelanzeiger immer noch flach ist, liegt die Ursache stromabwaerts des Browsers. Drei stromabwaerts-Ursachen erklaeren fast alle "Berechtigung OK aber kein Ton"-Berichte: (1) der OS-Ebene-Hauptschalter ist aus (macOS: Systemeinstellungen → Datenschutz und Sicherheit → Mikrofon; Windows: Einstellungen → Datenschutz und Sicherheit → Mikrofon mit beiden App-Zugriffs-Toggles an), (2) eine andere App haelt das Geraet (Zoom, Teams, Google Meet, Discord, OBS Studio oder ein virtueller Mikrofon-Treiber wie Krisp / NVIDIA Broadcast / VoiceMeeter / Loopback - beenden Sie die anstoessige App), oder (3) Hardware-Stummschaltung / falsches Eingabegeraet (USB-Headset-Kabel-Stummschalter, Laptop-Funktionstaste-Stummschaltung oder ein Bluetooth-Headset-Auto-Pairing zum falschen Geraet - bestaetigen Sie die aktive Eingabe in Systemeinstellungen → Ton → Eingabe). Die dedizierte Vier-Ursachen-Anleitung ist unter Microphone Test No Sound: 4 Fixes That Work - sie deckt jede stromabwaerts-Ursache im Detail ab.

Datenschutz und was der Mikrofon-Test tatsaechlich tut

Der Browser-Mikrofon-Test auf dieser Site liest den Live-Audio-Stream vollstaendig in Ihrem Browser. Die Seite ruft navigator.mediaDevices.getUserMedia({ audio: true }) auf, um einen Audio-Stream anzufordern, haengt den Stream an einen Web Audio-Analyser-Knoten an und rendert den Pegelanzeiger aus der Frequenzbereichs-Ausgabe des Analysers - alles auf Ihrem Geraet. Kein Sample wird hochgeladen, keine Aufnahme wird gespeichert, und das Stoppen des Tests gibt den Stream frei und der Mikrofon-Indikator geht aus. Wenn der Indikator nach dem Verlassen der Seite an bleibt, ist das fast immer ein anderes Tab oder eine andere Anwendung, die das Geraet haelt, nicht diese Seite; das Schliessen des anstoessigen Tabs gibt das Mikrofon innerhalb einer Sekunde frei.

Das Erteilen der Berechtigung an freetoolonline.com autorisiert nur die Seiten dieses Ursprungs, getUserMedia aufzurufen, waehrend Sie sich in einem Tab dieses Ursprungs befinden. Die Berechtigung bleibt nicht ueber Origins hinweg bestehen, erstreckt sich nicht auf Unterseiten anderer Domains, und der Browser wird sie automatisch widerrufen, wenn Sie die per-Site-Einstellung zurueck auf "Fragen" oder "Blockieren" aendern. Die Datenschutzerklaerung dokumentiert dies in einfacher Sprache.
