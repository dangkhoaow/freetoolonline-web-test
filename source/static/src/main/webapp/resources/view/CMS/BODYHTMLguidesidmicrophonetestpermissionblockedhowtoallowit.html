Izin Tes Mikrofon Diblokir: Cara Mengizinkan Akses Mic di Browser Anda

Ditinjau terakhir 2026-05-06. Ditargetkan pada momen ketika halaman tes mikrofon browser dimuat, meter level tetap datar saat Anda berbicara, dan baik ada permintaan izin di bilah alamat yang Anda abaikan, atau yang sudah ditolak telah menyembunyikan dirinya. Mengarahkan tindakan tes sebenarnya ke Microphone Test Online (di browser melalui Web API MediaDevices.getUserMedia(); tanpa upload, tanpa instalasi).

Jawaban 30 detik. Sebagian besar kasus "tes mikrofon tidak merespons" adalah status izin, bukan kesalahan hardware. Buka bilah alamat, cari ikon mikrofon kecil (Chrome, Edge), ikon perisai (Firefox), atau kunci "Pengaturan" (Safari). Klik, lalu pilih Izinkan untuk izin mikrofon freetoolonline.com, dan muat ulang halaman. Jika Anda tidak melihat ikon, permintaan telah dijawab sebelumnya dan browser mengingat; perbaikannya sama - temukan izin mikrofon per situs dan alihkan kembali ke Izinkan. Jalur klik yang tepat ada di bawah per browser.

Izin diblokir vs tidak ada permintaan sama sekali

Tes mikrofon browser meminta sistem operasi untuk frame audio langsung dari perangkat capture. Sebelum halaman dapat membaca sampel apapun, user agent (browser Anda) harus menjalankan pemeriksaan izin. Tiga status akhir mungkin terjadi, dan mereka terlihat berbeda di halaman:

  • Diizinkan. Permintaan izin muncul sekali dan Anda klik Izinkan, atau browser mengingat Izinkan sebelumnya dari situs ini dan melewati permintaan sepenuhnya. Dalam kedua kasus, meter level mulai merespons suara Anda dalam satu atau dua detik setelah halaman menyelesaikan paint awal.
  • Diblokir (ditolak). Anda menjawab permintaan dengan Blokir di suatu titik - di halaman ini atau di halaman lain dari origin yang sama - dan browser mengingat pilihan. Tidak ada permintaan baru; halaman meminta akses mikrofon, browser secara diam-diam menolak permintaan, dan meter level tetap datar atau menampilkan pemberitahuan "izin ditolak". Ikon mikrofon-dengan-garis-miring kecil biasanya terlihat di bilah alamat.
  • Diabaikan (tidak ada keputusan). Permintaan muncul, tetapi Anda mengklik di tempat lain, beralih tab, atau permintaan bergulir ke bagian layar yang tidak Anda lihat. Browser tetap memegang permintaan tertunda tanpa komitmen ke status mana pun. Meter level tetap datar, tetapi tidak seperti kasus diblokir, browser belum mengingat "tidak" - reload halaman berikutnya akan memunculkan permintaan lagi.

Mengetahui di mana dari tiga status ini Anda berada menghemat waktu. Jika ikon kecil muncul di bilah alamat, Anda di kasus (a) atau (b); jika tidak ada ikon dan halaman tidak berkata apa-apa, Anda di kasus (c) dan satu reload sering memperbaikinya. Jika ikon menunjukkan mikrofon-dengan-garis-miring, Anda di kasus (b) dan langkah-langkah di bawah ini mengubah izin per situs kembali ke Izinkan.

Chrome (desktop)

Chrome menyimpan izin mikrofon per origin di profil pengguna. Untuk memberikan atau memberikan ulang akses:

  1. Lihat di ujung paling kiri bilah alamat. Jika ikon mikrofon menunjukkan garis miring merah, klik ikon itu.
  2. Di popup kecil, atur Mikrofon ke Izinkan (kadang berlabel "Selalu izinkan di situs ini"). Tutup popup.
  3. Muat ulang halaman (Ctrl/Cmd + R). Bicara ke mikrofon Anda; meter level harus melompat dalam satu detik.

Jika Anda tidak melihat ikon mikrofon, izin mungkin sudah diatur ke "Blokir" dan Chrome mungkin menyembunyikan ikon saat tidak ada permintaan terbaru yang dipicu. Buka chrome://settings/content/microphone langsung, gulir ke daftar Tidak diizinkan menggunakan mikrofon Anda, temukan https://freetoolonline.com, klik ikon tong sampah di sebelahnya untuk menghapus aturan, lalu muat ulang halaman tes mikrofon; Chrome akan memunculkan permintaan izin segar saat halaman memanggil getUserMedia berikutnya. Pada perangkat terkelola (kebijakan sekolah atau tempat kerja), akses mikrofon dapat diblokir kebijakan di tingkat OS atau konsol admin - dalam kasus itu Chrome menampilkan "Dikelola oleh organisasi Anda" dan toggle per situs berwarna abu-abu.

Firefox (desktop)

Firefox menampilkan izin aktif sebagai ikon mikrofon kecil di sebelah kiri URL ketika halaman telah meminta atau diberikan akses. Langkah-langkah:

  1. Klik ikon mikrofon di sebelah kiri URL. Panel berlabel Izin terbuka.
  2. Temukan baris Gunakan Mikrofon. Jika menunjukkan Diblokir Sementara, klik x di sebelah entri untuk menghapusnya, lalu muat ulang halaman; Firefox akan meminta ulang. Jika menunjukkan Diblokir tanpa x, pilihan dibuat permanen - buka about:preferences#privacy, gulir ke Izin → Mikrofon → Pengaturan, temukan entri freetoolonline.com di daftar, ubah ke Izinkan atau hapus, lalu simpan dan muat ulang.
  3. Ketika permintaan muncul lagi, klik Izinkan (dan opsional centang Ingat keputusan ini agar Firefox tidak bertanya lagi).

Firefox memisahkan "diblokir sementara" (keputusan satu pemuatan halaman) dari "diblokir permanen" (keputusan tingkat origin). Sebagian besar pengguna mengalami kasus sementara saat kunjungan pertama dan tidak menyadari bahwa satu reload halaman akan meminta ulang. Kasus permanen memerlukan rute about:preferences di atas.

Safari (macOS)

Safari mengikat izin mikrofon ke pengaturan per-website untuk profil Safari aktif, bukan ke satu sesi halaman. Untuk mengizinkan:

  1. Dengan halaman tes mikrofon terbuka, pilih Safari → Pengaturan untuk freetoolonline.com dari bilah menu (atau Safari → Pengaturan → Websites → Mikrofon).
  2. Atur dropdown mikrofon untuk freetoolonline.com ke Izinkan.
  3. Tutup panel pengaturan dan muat ulang halaman (Cmd + R).

Jika "Izinkan" tidak ada di dropdown, macOS sendiri mungkin memblokir akses Safari ke mikrofon di tingkat OS - buka Pengaturan Sistem → Privasi dan Keamanan → Mikrofon, pastikan Safari diaktifkan, lalu kembali ke Safari dan coba lagi. macOS memunculkan permintaan satu kali pertama kali Safari mencoba menggunakan mikrofon di halaman manapun; setelah itu diizinkan, preferensi per-situs Safari mengambil alih.

Edge (desktop)

Edge mengikuti model izin Chromium dengan alur UI yang sama dengan Chrome. Klik ikon mikrofon di kiri bilah alamat, pilih Izinkan, muat ulang. Jika ikon disembunyikan, buka edge://settings/content/microphone dan hapus atau ubah aturan per-situs untuk freetoolonline.com. Edge pada instalasi Windows terkelola juga menghormati Group Policy - jika toggle berwarna abu-abu, admin IT telah mengunci pengaturan secara kebijakan dan override per pengguna tidak mungkin dari UI browser.

iOS Safari dan Android Chrome (mobile)

Browser mobile menambahkan lapisan tingkat OS di depan izin per-situs, jadi tes mikrofon "diblokir" di mobile dapat memiliki dua perbaikan berbeda - satu di browser, satu di pengaturan OS.

iOS Safari. Buka Pengaturan → Apps → Safari → Mikrofon. Atur default untuk "Mikrofon" ke Izinkan (atau Tanya). Lalu, dengan halaman tes mikrofon terbuka, ketuk tombol aA kecil di bilah alamat dan pilih Pengaturan Website → Mikrofon → Izinkan. Tutup dan buka kembali tab. iOS juga membutuhkan Safari sendiri memiliki akses Mikrofon di Pengaturan → Privasi dan Keamanan → Mikrofon → Safari - pada perangkat baru atau setelah reset privasi ini bisa dimatikan.

Android Chrome. Ketuk ikon kunci atau info di sebelah kiri URL, lalu Izin → Mikrofon → Izinkan. Jika toggle hilang, buka Pengaturan → Pengaturan situs → Mikrofon, temukan entri freetoolonline.com di bawah "Diblokir", hapus, lalu muat ulang halaman agar Chrome meminta ulang. OS Android memiliki lapisan sendiri juga - Pengaturan → Apps → Chrome → Izin → Mikrofon harus diatur ke Izinkan hanya saat menggunakan aplikasi (default) untuk situs apapun menggunakan mikrofon.

Ketika Anda terus menolak secara tidak sengaja: pencabutan bersih dan pemberian ulang

Jika Anda telah mengklik beberapa keputusan "Blokir" di browser yang berbeda dan ingin reset bersih, rute paling andal adalah menghapus izin per-situs sepenuhnya sehingga browser bertanya segar pada pemuatan halaman berikutnya. Setiap browser memiliki URL pengaturan yang sedikit berbeda:

  • Chrome / Edge: chrome://settings/content/microphone atau edge://settings/content/microphone → hapus entri freetoolonline.com di bawah "Tidak diizinkan menggunakan mikrofon Anda".
  • Firefox: about:preferences#privacy → Izin → Mikrofon → Pengaturan → hapus entri.
  • Safari (macOS): Safari → Pengaturan → Websites → Mikrofon → pilih entri dan klik Hapus.

Setelah menghapus, muat ulang halaman tes mikrofon; browser memperlakukan panggilan getUserMedia berikutnya sebagai permintaan pertama kali dan memunculkan permintaan segar. Ini juga rute yang tepat ketika akun komputer bersama telah meninggalkan keputusan Blokir lama dari pengguna sebelumnya.

Jika izin diizinkan tetapi meter masih datar, penyebabnya di hilir browser. Tiga penyebab hilir bertanggung jawab atas hampir semua laporan "izin OK tetapi tidak ada suara": (1) toggle master tingkat OS dimatikan (macOS: Pengaturan Sistem → Privasi dan Keamanan → Mikrofon; Windows: Pengaturan → Privasi dan keamanan → Mikrofon dengan kedua toggle akses aplikasi aktif), (2) aplikasi lain memegang perangkat (Zoom, Teams, Google Meet, Discord, OBS Studio, atau driver mikrofon virtual seperti Krisp / NVIDIA Broadcast / VoiceMeeter / Loopback - tutup aplikasi pelanggar), atau (3) mute hardware / perangkat input salah (saklar mute kabel headset USB, tombol fungsi mute laptop, atau auto-pair headset Bluetooth ke perangkat salah - konfirmasi input aktif di Pengaturan Sistem → Suara → Input). Panduan khusus empat penyebab ada di Microphone Test No Sound: 4 Fixes That Work - mencakup setiap penyebab hilir secara rinci.

Privasi dan apa yang sebenarnya dilakukan tes mikrofon

Tes mikrofon browser di situs ini membaca aliran audio langsung sepenuhnya di browser Anda. Halaman memanggil navigator.mediaDevices.getUserMedia({ audio: true }) untuk meminta aliran audio, menempelkan aliran ke node analyser Web Audio, dan merender meter level dari output domain frekuensi analyser - semuanya di perangkat Anda. Tidak ada sampel yang diunggah, tidak ada rekaman yang disimpan, dan menghentikan tes melepaskan aliran dan indikator mikrofon mati. Jika indikator tetap menyala setelah Anda meninggalkan halaman, itu hampir selalu tab atau aplikasi lain yang memegang perangkat, bukan halaman ini; menutup tab pelanggar melepaskan mikrofon dalam satu detik.

Memberikan izin ke freetoolonline.com hanya mengotorisasi halaman origin ini untuk memanggil getUserMedia selama Anda di tab dari origin ini. Izin tidak bertahan antar origin, tidak meluas ke sub-halaman domain lain, dan browser akan secara otomatis mencabutnya jika Anda mengubah pengaturan per situs kembali ke "Tanya" atau "Blokir". Kebijakan Privasi mendokumentasikan ini dalam bahasa sederhana.
