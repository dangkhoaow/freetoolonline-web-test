Alat online gratis - tidak perlu mengunggah

Tinjauan terakhir 2026-04-27. Buka peta situs untuk melihat setiap alat yang berjalan di browser Anda.

Jawaban 30 detik. Alat berbasis browser mengkonversi, mengkompres, mengedit, dan menganalisis file tanpa pernah mengunggahnya, karena browser modern dapat membuka data biner, menjalankan algoritma sebenarnya di JavaScript atau WebAssembly, dan menulis hasil kembali ke unduhan Anda. File Anda berpindah dari disk ke tab browser dan kembali ke disk. Tab adalah seluruh ruang kerja; tidak ada server yang melihat data Anda.

Alasan teknis mengapa ini berfungsi

Tiga kemampuan browser membuatnya mungkin. File API mengekspos file yang dipilih (atau dijatuhkan) pengguna sebagai typed array byte - tanpa unggah. WebAssembly menjalankan kode terkompilasi (C, C++, Rust) pada kecepatan hampir native di dalam tab; pustaka seperti Pillow, FFmpeg, dan 7-Zip telah dikompilasi ke WASM dan dapat membaca dan menulis format yang sama yang mereka tangani di desktop. Blob API dan Object URLs memungkinkan tab membangun file yang dapat diunduh di memori dan mengembalikannya ke pengguna.

Bersama-sama: file dibaca dari disk ke memori tab, diproses oleh modul WASM atau algoritma JavaScript murni, dan ditulis kembali ke disk melalui unduhan native browser. Server yang meng-host halaman hanya melayani byte HTML, JS, dan WASM - data pengguna tidak pernah meninggalkan perangkat.

Apa yang Anda dapatkan gratis

  • Privasi. Apa pun yang Anda tidak ingin ada di log server tetap di luar log server karena tidak ada yang masuk ke server. PDF rahasia, foto pribadi, dokumen kerja - semuanya diproses secara lokal.
  • Kecepatan untuk file kecil. Tidak ada waktu unggah, tidak ada antrian, tidak ada latensi pemrosesan server-side. Gambar 5 MB dikonversi dalam waktu yang dibutuhkan browser untuk membaca byte ditambah menjalankan algoritma.
  • Perilaku ramah offline. Setelah halaman dimuat, ia dapat berjalan tanpa koneksi jaringan. Beberapa alat menyimpan modul WASM ke cache sehingga mereka bekerja sepenuhnya offline setelah kunjungan pertama.
  • Tanpa akun. Sistem akun ada untuk menagih, melacak, atau membatasi pengguna. Tidak ada kebutuhan-kebutuhan itu yang berlaku ketika pekerjaan terjadi di perangkat pengguna.

Apa yang tidak dapat dilakukan alat browser

Trade-off-nya jujur. Alat hanya-browser memiliki batasan nyata:

  • File yang sangat besar. Video 4 GB mungkin tidak akan dimuat - tab memiliki batas memori (beberapa ratus MB hingga ~2 GB tergantung browser). Alat desktop menangani 4 GB tanpa masalah.
  • Operasi GPU-berat dalam skala. WebGPU ada di browser modern tetapi lebih baru dan kurang dioptimalkan daripada pipeline CUDA atau Metal native. Color-grading video 4K bingkai demi bingkai lambat di tab.
  • Operasi yang membutuhkan sistem file pengguna secara luas. Alat browser melihat file yang Anda berikan dan dapat menulis kembali satu file. Ia tidak dapat memindai folder, memantau file baru, atau mengubah file yang ada di tempat.
  • Alur kerja multi-file yang mencakup banyak sesi. Tanpa status persisten secara default. Refresh tab dan pekerjaan hilang. Beberapa aplikasi menambahkan persistensi localStorage atau IndexedDB; sebagian besar tidak.

Mulai dari mana

Untuk gambar: HEIC ke JPG, kompres gambar, alat konversi gambar. Untuk PDF: hub alat PDF. Untuk ZIP: hub alat ZIP. Untuk pekerjaan developer: hub alat developer. Untuk tes perangkat: hub alat tes perangkat. Katalog lengkap: peta situs.

Privasi yang Anda dapatkan adalah berita utama, tetapi kecepatan dan pengaturan tanpa gesekan sama pentingnya. Coba satu alat, refresh tab, lihat bahwa file tidak ada di server di suatu tempat - tidak pernah ada di sana.
