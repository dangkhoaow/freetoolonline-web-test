Wie man einen Ordner fuer E-Mail komprimiert - Schritt fuer Schritt (2026)

Zuletzt geprueft 2026-04-24. Funktioniert auf Windows 10/11, macOS 13+ (Ventura und neuer) und im Browser ueber Zip File, wenn ein nativer Shell-ZIP-Flow nicht verfuegbar ist.

30-Sekunden-Antwort. Rechtsklick auf den Ordner → "Senden an ZIP-Ordner (komprimiert)" auf Windows (oder "Komprimieren" auf macOS). Haengen Sie die resultierende .zip an die E-Mail an. Wenn die ZIP die Grenze des Anbieters ueberschreitet (typischerweise 20-25 MB, siehe E-Mail-Groessentabelle unten), dann (a) teilen Sie die Quelle in zwei Ordner und ZIPpen Sie jeden, oder (b) laden Sie zu einem Dateiuebertragungsdienst hoch und fuegen Sie den Link in den E-Mail-Koerper ein.

Warum einen Ordner fuer E-Mail komprimieren?

E-Mail-Anhaenge werden in Bytes gemessen. Unkomprimierte Ordner koennen ueberhaupt nicht angehaengt werden - E-Mail-Clients erfordern eine einzelne Datei, kein Verzeichnis. Komprimieren falten den Ordner in ein einzelnes ZIP-Archiv, das die interne Struktur, Dateinamen und Aenderungsdaten bewahrt, waehrend es die Gesamtgroesse durch LZ77 / DEFLATE-Kompression reduziert. Fuer Ordner mit gemischtem Inhalt (Dokumente, Bilder, Tabellen), erwarten Sie 20-50% Groessenreduzierung; fuer Ordner, die bereits von komprimierten Formaten (JPG, MP3, MP4) dominiert werden, erwarten Sie nur ein paar Prozent, weil diese Formate bereits entropie-kodiert sind.

Die drei praktischen Gruende, vor dem Senden zu ZIPpen:

  • Anbieter-Groessenlimits. Gmail, Outlook und iCloud begrenzen alle einzelne Anhaenge. Eine ZIP kann unter das Limit passen, wenn der rohe Ordner es nicht kann.
  • Strukturerhaltung. Der Empfaenger erhaelt den Ordnerbaum nach dem Entpacken intakt - nicht 47 lose Dateien im Wurzelverzeichnis seines Downloads-Ordners.
  • Integritaet im Transit. ZIP speichert CRC-32 pro Datei; beschaedigte Downloads markieren sich beim Extrahieren selbst, sodass Sie es erneut versuchen koennen, anstatt stillschweigend defekte Bytes zu erhalten.

Schritt fuer Schritt auf Windows 10 und 11

  1. Finden Sie den Ordner im Datei-Explorer. Wenn Sie eine Teilmenge von Dateien anstelle des gesamten Ordners komprimieren, erstellen Sie zuerst einen neuen leeren Ordner und kopieren Sie die Dateien hinein - ZIP kann lose Dateien nicht nativ "auswaehlen-und-archivieren", ohne sie zu gruppieren.
  2. Rechtsklicken Sie auf den Ordner. Auf Windows 11 erscheint das neue Shell-Menue; wenn Sie die ZIP-Option nicht sofort sehen, klicken Sie auf "Weitere Optionen anzeigen" unten, um das klassische Menue zu enthuellen.
  3. Waehlen Sie "Senden an" → "Komprimierter (gezippter) Ordner". Windows erstellt <ordner-name>.zip im selben Verzeichnis.
  4. Benennen Sie die ZIP um, wenn gewuenscht (der Standard erbt den Ordnernamen).
  5. Haengen Sie die ZIP an Ihre E-Mail an. Drag-and-drop in das Verfassen-Fenster oder verwenden Sie die Bueroklammer / "Datei anhaengen"-Schaltflaeche.

Windows 11 fuegte 2023 native 7z- und tar-Unterstuetzung hinzu; speziell fuer ZIP-Anhaenge bleiben Sie bei .zip, weil macOS und jedes Webmail-Vorschaufenster es ohne Download dekodiert. Wenn der Empfaenger bestaetigt, dass 7-Zip oder gleichwertig installiert ist, kann das Wechseln zu .7z mit LZMA2 weitere 10-20% von der Groesse abrasieren, zum Preis eines langsameren Kompressionsschritts.

Schritt fuer Schritt auf macOS 13+ (Ventura, Sonoma, Sequoia)

  1. Waehlen Sie den Ordner im Finder aus.
  2. Rechtsklick (oder Control-Klick) → "'<ordner-name>' komprimieren". macOS erstellt <ordner-name>.zip im selben Verzeichnis.
  3. Benennen Sie das Archiv um, falls noetig. macOS bewahrt Dateimetadaten und Resource Forks in __MACOSX-Ordnern, was einige Windows-Empfaenger als Unordnung betrachten; siehe die FAQ zum Entfernen dieser unten.
  4. Haengen Sie es an die E-Mail an ueber Standard-Drag-Drop oder Aktion "Datei anhaengen".

macOS unterstuetzt auch terminalbasiertes Archivieren mit zip -r archive.zip ordner/ (BSD zip) oder ditto -c -k --sequesterRsrc ordner archive.zip (Apples ditto-Befehl). Ditto ist das, was Finder unter der Haube aufruft, und ist vorzuziehen fuer jeden Ordner, der macOS-spezifische Resource Forks enthaelt.

E-Mail-Anhangsgrenzen, die einen ZIPpten Ordner in 2026 blockieren

Anbieterspezifische Grenzen verursachen die meisten "mein Ordner-ZIP wird nicht gesendet"-Fehler. Der Blocker ist fast immer die Obergrenze des Empfaengers, nicht Ihr Sendelimit - kennen Sie beide, bevor Sie komprimieren, weil eine 30-MB-Ordner-ZIP, die Ihr Gmail problemlos sendet, immer noch gegen einen 25-MB-Outlook-Empfaenger zurueckprallt. Ordner sind besonders betroffen, weil Verzeichnisarchive dazu neigen, Dutzende von Elementen auf einmal zu buendeln; ein uebergrosses Foto im Ordner reicht aus, um die gesamte ZIP ueber die Grenze eines Empfaengers zu schieben.

  • Gmail: 25 MB senden / 50 MB empfangen. Groessere Anhaenge werden automatisch ueber Google Drive umgeleitet.
  • Outlook (Microsoft 365): Standard-Sendelimit 20 MB; 25 MB empfangen fuer die meisten Plaene. Unternehmens-Mandanten koennen bis zu 150 MB konfiguriert werden, aber Empfaenger ausserhalb des Mandanten sehen oft immer noch 25 MB.
  • Outlook.com (kostenlos): 10 MB Sendelimit vor OneDrive-Umleitung.
  • iCloud Mail: 20 MB; Apples Mail Drop tritt fuer groessere Anhaenge in Aktion (iCloud-Server hostet die Datei fuer 30 Tage).
  • Yahoo Mail: 25 MB.
  • Unternehmens-/benutzerdefiniertes SMTP: variiert stark (10-150 MB). Fragen Sie Ihren Administrator oder fuehren Sie einen Test-Versand mit einer ~15-MB-Datei durch.

Wenn die ZIP die Grenze des Empfaengers ueberschreitet, sagt die Bounce-Nachricht normalerweise "Nachricht zu gross" oder "554 5.7.1". Wenn Sie diesen Fehler sehen, senden Sie nicht mit der gleichen ZIP erneut - wechseln Sie zu einer Dateiuebertragungslink-Strategie.

Wann ein Passwort zur ZIP hinzufuegen

Fuegen Sie ein Passwort hinzu, wenn der Ordner irgendetwas von Folgendem enthaelt: rechtliche Dokumente, unterzeichnete Vertraege, HR-Daten, Finanzunterlagen, Anmeldeinformationen oder alles, was einer Vertraulichkeitsvereinbarung unterliegt. Passwortgeschuetztes ZIP verschluesselt Dateiinhalte mit AES-256 (die ZIP-2.0-"traditionelle" Verschluesselung ist kaputt und sollte nicht verwendet werden). Der Empfaenger benoetigt das Passwort ueber einen separaten Kanal - setzen Sie das Passwort nicht in dieselbe E-Mail wie den Anhang.

Der schnellste In-Browser-Pfad ist Zip File, das passwortgeschuetzte ZIPs ohne Softwareinstallation erstellt. Wenn Sie nur eine bereits passwortgeschuetzte ZIP, die Sie kontrollieren, oeffnen muessen, ist das passende Tool ZIP-Passwort entfernen. Fuer die Kurzentscheidungsmatrix, wer was wissen sollte, siehe den verwandten Leitfaden zu PDF-Passworttypen: Eigentuemer vs Benutzer - die gleiche Taxonomie gilt fuer ZIP-Zugriffs- vs Bearbeitungsberechtigungen.

Wann ZIP ueberspringen und einen Dateiuebertragungsdienst verwenden

ZIP ist die richtige Antwort fuer Ordner unter der Anhangsobergrenze des Empfaengers. Darueber werden anhangsbasierte Workflows fragil - Empfaenger prallen ab, Spam-Filter markieren, Unternehmens-Posteingangs-Quoten fuellen sich. Drei Signale, dass ein Dateiuebertragungsdienst (Google Drive, OneDrive, Dropbox, WeTransfer oder Ihr eigenes Hosting) einen Anhang schlaegt:

  • ZIP groesser als 20 MB auf dem Weg zu einem Verbraucher-E-Mail-Konto.
  • Empfaenger auf restriktiver Unternehmens-E-Mail (rechtlich, Finanzen, Gesundheitswesen haben oft aggressive Groessen- und Inhaltsfilter).
  • Medienreiche Ordner (Video, RAW-Fotos), wo Kompression die Groesse nicht bedeutungsvoll reduziert - Hosting ist effizienter als Anhaengen.

Wenn Sie wollen, dass der Empfaenger den Inhalt sehen kann, ohne zuerst herunterzuladen (Nur-Lese-Ueberpruefung), schlaegt ein teilbarer Link zu einem Cloud-Ordner einen ZIP-Anhang auf jeder Achse: vorab anzeigbar, immer aktuell und individuell widerrufbar.

ZIP-Groesse vor dem Senden reduzieren

Wenn Ihr Ordner gerade knapp ueber dem Groessenlimit ZIPt, vier pragmatische Reduzierungen, bevor Sie zu Dateiuebertragung wechseln:

  • Komprimieren Sie Bilder zuerst. JPG bei Qualitaet 85 ist normalerweise nicht von der Quelle zu unterscheiden und 30-50% kleiner. Fuehren Sie die losen Bilder durch Compress Image aus, bevor Sie ZIPpen.
  • Konvertieren Sie HEIC-Fotos vom iPhone. HEIC-Dateien koennen, sobald sie in einer ZIP sind, nicht auf vielen Windows-Empfaengern vorab angezeigt werden. Konvertieren Sie zuerst ueber HEIC to JPG - JPG ist pro Datei groesser, oeffnet sich aber ueberall ohne Codec-Installation. Siehe HEIC vs JPG vs WebP fuer die vollstaendige Format-Entscheidung.
  • Entfernen Sie macOS-Metadaten. Auf macOS enthalten Finder-Archive __MACOSX- und .DS_Store-Rauschen. Der ditto-Befehl strippt diese; alternativ komprimieren Sie auf einer Linux- / Windows-Maschine, die sie nie produziert.
  • Teilen Sie in mehrere ZIPs. Viele E-Mail-Clients akzeptieren 2-3 Anhaenge eleganter als eine ueberdimensionierte ZIP. Gruppieren Sie nach Thema (dokumente.zip + fotos.zip) anstatt nach willkuerlicher Byte-Grenze.

Haeufig gestellte Fragen

Warum ueberschreitet meine ZIP nach der Kompression immer noch das Limit?

Weil der Quellordner von bereits komprimiertem Inhalt dominiert wird. JPG, PNG, MP3, MP4, DOCX und XLSX sind alle intern vorkomprimiert; der DEFLATE-Algorithmus von ZIP hat wenig zusaetzliche Entropie zu entfernen. Fuer Ordner, in denen 80%+ der Bytes von diesen Formaten kommen, erwarten Sie nur wenige Prozent Reduzierung. In diesen Faellen wechseln Sie zu einem Dateiuebertragungsdienst, anstatt zu versuchen, die ZIP unter das Limit zu pressen.

Kann ich eine Ordner-ZIP unter Windows ohne zusaetzliche Software passwortgeschuetzt machen?

Der Windows-Shell-Befehl "Senden an → Komprimierter Ordner" unterstuetzt nativ keinen Passwortschutz. Sie haben drei Optionen: (a) verwenden Sie unser browserbasiertes Zip File-Tool, um eine passwortgeschuetzte ZIP zu erstellen, (b) installieren Sie 7-Zip und waehlen Sie "Zum Archiv hinzufuegen..." mit einem Passwort, oder (c) verwenden Sie PowerShell mit dem Compress-Archive-Cmdlet plus einem Verschluesselungsschritt von Drittanbietern. Das Browser-Tool ist die Option mit der geringsten Reibung, weil keine Installation erforderlich ist.

Was, wenn der Empfaenger sagt "die ZIP ist beschaedigt" oder "Extraktion fehlgeschlagen"?

Drei haeufige Ursachen: (1) der Virenscanner des E-Mail-Anbieters hat das Archiv im Transit modifiziert (ungewoehnlich, aber passiert bei aelteren Antivirus-Gateways); (2) die ZIP wurde mit Windows Explorer erstellt und enthaelt Nicht-ASCII-Dateinamen, die der Extraktor des Empfaengers nicht dekodieren kann; (3) der Download wurde abgeschnitten (weniger wahrscheinlich auf modernem HTTP, aber immer noch moeglich auf unzuverlaessigen Mobilnetzen). Loesung: komprimieren Sie den Ordner auf einem anderen OS neu (wenn macOS-erstellt, versuchen Sie es erneut unter Windows oder verwenden Sie ditto mit --sequesterRsrc), oder laden Sie ueber einen Dateiuebertragungsdienst neu hoch, der Pruefsummen serverseitig verifiziert, bevor er den Download-Link freigibt.

Zusammenfassung

Einen Ordner fuer E-Mail zu komprimieren ist eine 10-Sekunden-native Shell-Operation auf Windows und macOS. Die echte Entscheidung ist, ob die resultierende ZIP unter die Anhangsobergrenze des Empfaengers passt - wenn ja, versenden Sie sie mit dem Rechtsklick-Shell-Menue; wenn nein, waehlen Sie zwischen Aufteilen in kleinere ZIPs und Wechseln zu einem Dateiuebertragungslink. Fuegen Sie ein Passwort hinzu, wann immer der Ordner irgendetwas Reguliertes oder Vertrauliches enthaelt, und denken Sie daran, dass HEIC und bereits komprimierte Medien nicht viel schrumpfen - konvertieren oder hosten Sie diese separat.
