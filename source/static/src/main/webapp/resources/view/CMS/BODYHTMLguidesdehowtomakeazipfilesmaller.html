Wie man eine Zip-Datei kleiner macht

Zuletzt geprueft 2026-04-27. Oeffnen Sie das Zip-Tool, um in Ihrem Browser zu komprimieren, ohne die Dateien hochzuladen.

30-Sekunden-Antwort. Die endgueltige Groesse eines Zip wird durch das festgelegt, was darin ist, nicht durch das Zip-Format. Drei Dinge senken die Zahl: waehlen Sie die richtige Komprimierungsstufe (9 fuer Dokumente, 0 fuer bereits komprimierte Medien), entfernen oder schrumpfen Sie zuerst die schweren Dateien (komprimieren Sie Bilder und PDFs vor dem Zippen), und teilen Sie das Archiv auf, wenn es ein festes Groessenlimit ueberwinden muss (Gmails 25 MB, 100 MB der meisten Chat-Apps). Oeffnen Sie das Zip-Tool, ziehen Sie die Dateien hinein, stellen Sie Best-Komprimierung fuer Dokumente und Store fuer Medien ein und laden Sie herunter.

Warum ein Zip manchmal genauso gross aussieht wie die Eingabe

Zip verwendet Deflate, eine verlustfreie Komprimierung, die auf Mustern arbeitet. Klartext und Quellcode sind 60-80% wiederholende Bytes; Deflate quetscht sie auf 20-40% des Originals. JPEG, PNG, MP4, MP3 und die meisten anderen Medien sind bereits durch ihr eigenes Format komprimiert, sodass Deflate fast nichts mehr zu entfernen findet. Ein Ordner mit Urlaubsfotos, der maximal gezippt wird, kommt 1-3% kleiner heraus als die Eingabe. Das ist normal, kein Tool-Fehler.

Die Loesung ist, die schweren Dateien vor dem Zippen zu komprimieren. Ein Bild komprimieren reduziert ein 5-MB-Foto auf 600 KB ohne sichtbaren Qualitaetsverlust; Ein GIF in Frames extrahieren gefolgt von Neu-Codierung gewinnt normalerweise; Ein PDF zusammenstellen mit zurueckgesetzten Bildqualitaetseinstellungen macht eine messbare Delle. Danach tut der Zip-Schritt, was er kann.

Waehlen Sie die Komprimierungsstufe bewusst

Das Browser-Tool zeigt Store, Fast, Default und Best. Verwenden Sie sie so:

  • Store (Stufe 0). Buendel ohne Komprimierung. Richtig, wenn jede Datei innen bereits komprimiert ist (ein Ordner mit MP4s, ein Ordner mit JPEGs). Spart CPU und produziert eine etwas kleinere Datei als aggressives Deflate auf bereits komprimierten Daten.
  • Fast (Stufe 1-3). Schnelles Packen fuer viele kleine Textdateien, wenn Ihnen die letzten paar Prozent egal sind.
  • Default (Stufe 6). Der vernuenftige Standard. Balanciert Groesse und Zeit bei gemischten Eingaben.
  • Best (Stufe 9). Pressen Sie jedes Bit heraus. Richtig fuer Dokumente, Quellcode, Logs, CSVs. Fuegt 2-5x Zeit vs Default fuer ~3-5% extra Ersparnis hinzu.

Ein Groessenlimit durch Aufteilen treffen

Wenn das Postfach oder die Chat-App des Empfaengers ein hartes Limit hat, ist das Aufteilen des Archivs in kleinere Volumes der richtige Weg. Gmail begrenzt Anhaenge auf 25 MB; Slack auf 100 MB auf kostenlosen Tarifen; die meisten Verbraucher-E-Mails auf 20-25 MB. Das Browser-Zip-Tool laesst Sie nach Groesse in N Volumes aufteilen. Senden Sie die Volumes; der Empfaenger legt sie alle in den gleichen Ordner und sein Zip-Tool liest sie als ein Archiv zurueck.

Was nicht funktioniert

Ein Zip erneut zippen hilft selten. Das innere Zip ist bereits komprimiert; es zu umhuellen erzeugt eine etwas groessere Datei aufgrund des zweiten Headers. Wenn ein Zip nach Best-Komprimierung zu gross ist, ist die Datei innen der Engpass, nicht das Archiv. Siehe Dateikompressor vs Zip fuer welchen Kompressor zuerst zu greifen.

Einseitiges Playbook: ein Zip in drei Entscheidungen schrumpfen

Die zugrunde liegende Aufgabe auf dieser Seite ist eine Entscheidung: schrumpfen Sie die Bytes, die der Empfaenger herunter- oder hochladen muss. Drei Entscheidungen erledigen das normalerweise. Oeffnen Sie das Browser-ZIP-Tool, waehlen Sie die Best-Stufe (Stufe 9) fuer Dokumente und Quellcode, wechseln Sie zu Store (Stufe 0), wenn der Inhalt bereits MP4 / JPEG / MP3 ist, und teilen Sie in Volumes auf, wenn das Postfach des Empfaengers das Ergebnis begrenzt. Wenn die Eingabe ein einzelnes Bild ist und Sie ueberhaupt kein Archiv brauchen, komprimieren Sie das Bild direkt - das resultierende JPEG ist oft kleiner als jedes Zip des Originals. Fuer eine portable Zusammenfassung des Stufenwaehlers siehe Zip-Dateigroessen-Kompressor, und fuer das Routing zwischen ZIP und Bildkomprimierung siehe wann ZIP vs wann Bild komprimieren.
