File sumber otomatis terhapus dari layanan build setelah jendela retensi singkat, jadi arsip tidak tertinggal di mana pun setelah Anda mengunduhnya.

Tiga bentuk input yang ditangani pembuat

Pembuat menerima satu file pada satu waktu, pilihan banyak file (pilih beberapa dokumen dari dialog file dengan Shift atau Ctrl), atau seluruh folder yang diseret ke zona unggah. Subfolder dipertahankan di dalam arsip sehingga penerima melihat pohon direktori yang sama dengan yang Anda kirim. Outputnya adalah satu .zip, dikembalikan ke browser Anda, siap dilampirkan ke email, dijatuhkan ke chat, atau diunggah ke tautan bagikan sebagai satu item. Nama arsip adalah yang Anda ketikkan ke modal Pengaturan sebelum mengeklik Konversi; pilih nama yang mengatakan apa isi paket (nama proyek, rentang tanggal, nama belakang penerima) sehingga file jelas saat tiba.

Apa arti "online" di sini

"Online" di halaman ini berarti pembuat berjalan di layanan build privat yang di-hosting FreetoolOnline, bukan sebagai program yang diunduh ke mesin Anda. File meninggalkan perangkat Anda via HTTPS selama beberapa detik build berlangsung, lalu file sumber dihapus dari layanan setelah jendela retensi singkat. File .zip sendiri ditawarkan kembali ke browser Anda dan tidak persisten di layanan setelah unduhan dimulai. Menamai arsip agar penerima membukanya

Modal Pengaturan menampilkan field "Nama arsip" yang menentukan nama file yang dilihat penerima. Default adalah "archive.zip", yang mudah dilupakan; ubah ke topik (misalnya "keuangan-q3.zip", "foto-pernikahan-2024.zip" atau "serah-terima-klien-source.zip") agar file tidak hilang di antara unduhan penerima. Nama tidak memengaruhi isi atau kompresi - murni label - tetapi meningkatkan kemungkinan penerima benar-benar membuka file daripada mengabaikannya sebagai spam. Jaga nama tetap pendek, gunakan tanda hubung atau garis bawah alih-alih spasi, dan tetap dengan karakter ASCII agar Windows, macOS, dan Linux semua membukanya tanpa mengkodekan ulang nama file.

Menambahkan kata sandi (Standard / AES-128 / AES-256)

Modal Pengaturan memuat field kata sandi opsional dengan tiga pilihan enkripsi. Enkripsi Standard ZIP adalah yang tertua dan paling diterima secara universal tetapi dianggap lemah menurut standar saat ini. AES-128 adalah default modern dan diterima oleh Windows Explorer saat ini (klik kanan Extract All) dan Archive Utility macOS. AES-256 adalah tingkat tertinggi dengan plafon kompatibilitas yang sama dengan AES-128. Pilih Standard untuk konten sensitivitas rendah ketika kompatibilitas adalah prioritas; pilih AES-128 atau AES-256 ketika kata sandi adalah penghalang privasi nyata dan Anda tahu sistem operasi penerima saat ini. Kirim kata sandi melalui kanal terpisah (chat, SMS) - email plus kata sandi dalam satu pesan menggagalkan tujuannya.

Apa yang TIDAK dilakukan pembuat

Pembuat tidak membuka .zip yang kata sandinya tidak diketahui - ia membangun arsip baru dan opsional mengenkripsinya dengan kata sandi yang Anda berikan, tetapi tidak memecahkan enkripsi pada arsip yang dibangun orang lain. Ia tidak menghasilkan arsip 7z atau RAR - kedua format sedikit lebih padat daripada ZIP tetapi memerlukan instalasi pihak ketiga di sebagian besar platform penerima, jadi pembuat tetap pada ZIP untuk jangkauan lintas platform. Ia tidak menghapus metadata dari file input - EXIF pada JPG, field penulis pada dokumen Word, dan stempel waktu modifikasi semuanya masuk ke arsip tanpa berubah. Jika itu perlu dibersihkan, lakukan sebelum mengunggah folder.
