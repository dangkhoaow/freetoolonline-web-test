Cara Mengkompres Folder untuk Email - Langkah Demi Langkah (2026)

Terakhir ditinjau 2026-04-24. Bekerja di Windows 10/11, macOS 13+ (Ventura dan lebih baru), dan di browser via Zip File ketika alur zip shell natif tidak tersedia.

Jawaban 30 detik. Klik kanan folder → "Kirim ke folder ZIP (terkompres)" di Windows (atau "Kompres" di macOS). Lampirkan .zip yang dihasilkan ke email. Jika ZIP melebihi batas penyedia (biasanya 20-25 MB, lihat tabel ukuran email di bawah), maka (a) pisahkan sumber menjadi dua folder dan ZIP masing-masing, atau (b) unggah ke layanan transfer file dan tempel tautan ke badan email.

Mengapa mengkompres folder untuk email?

Lampiran email diukur byte. Folder yang tidak dikompres tidak dapat dilampirkan sama sekali - klien email memerlukan satu file, bukan direktori. Mengkompres mengumpulkan folder menjadi satu arsip ZIP yang mempertahankan struktur internal, nama file, dan tanggal modifikasi, sambil menyusutkan ukuran total melalui kompresi LZ77 / DEFLATE. Untuk folder konten campuran (dokumen, gambar, spreadsheet), harapkan pengurangan ukuran 20-50%; untuk folder yang sudah didominasi oleh format yang dikompres (JPG, MP3, MP4), harapkan hanya beberapa persen karena format tersebut sudah dikodekan entropi.

Tiga alasan praktis untuk ZIP sebelum mengirim:

  • Batas ukuran penyedia. Gmail, Outlook, dan iCloud semuanya membatasi lampiran individu. ZIP dapat muat di bawah batas ketika folder mentah tidak bisa.
  • Pelestarian struktur. Penerima mendapat pohon folder utuh setelah unzip - bukan 47 file longgar di akar folder Unduhan mereka.
  • Integritas dalam perjalanan. ZIP menyimpan CRC-32 per file; unduhan rusak menyenyalkan diri saat ekstrak sehingga Anda dapat mencoba ulang daripada diam-diam menerima byte rusak.

Langkah demi langkah di Windows 10 dan 11

  1. Cari folder di File Explorer. Jika Anda mengkompres subset file daripada seluruh folder, buat folder kosong baru dan salin file ke dalamnya terlebih dahulu - ZIP tidak dapat "pilih-dan-arsipkan" file longgar secara natif tanpa mengelompokkannya.
  2. Klik kanan folder. Di Windows 11 menu shell baru muncul; jika Anda tidak melihat opsi ZIP segera, klik "Tampilkan opsi lainnya" di bagian bawah untuk mengungkapkan menu klasik.
  3. Pilih "Kirim ke" → "Folder terkompres (zip)". Windows membuat <nama-folder>.zip di direktori yang sama.
  4. Ganti nama ZIP jika diinginkan (default mewarisi nama folder).
  5. Lampirkan ZIP ke email Anda. Seret-drop ke jendela komposisi atau gunakan tombol klip / "Lampirkan file".

Windows 11 menambahkan dukungan native 7z dan tar pada 2023; untuk lampiran ZIP secara khusus, tetap dengan .zip karena macOS dan setiap panel pratinjau webmail mendekodenya tanpa unduhan. Jika penerima mengkonfirmasi 7-Zip atau yang setara terinstal, beralih ke .7z dengan LZMA2 dapat mencukur 10-20% lagi dari ukuran dengan biaya langkah kompres yang lebih lambat.

Langkah demi langkah di macOS 13+ (Ventura, Sonoma, Sequoia)

  1. Pilih folder di Finder.
  2. Klik kanan (atau Control-klik) → "Kompres '<nama-folder>'". macOS membuat <nama-folder>.zip di direktori yang sama.
  3. Ganti nama arsip jika perlu. macOS mempertahankan metadata file dan resource fork di dalam folder __MACOSX, yang beberapa penerima Windows anggap kekacauan; lihat FAQ tentang menghapusnya di bawah.
  4. Lampirkan ke email via seret-drop standar atau aksi Lampirkan File.

macOS juga mendukung pengarsipan berbasis Terminal dengan zip -r archive.zip folder/ (BSD zip) atau ditto -c -k --sequesterRsrc folder archive.zip (perintah ditto Apple). Ditto adalah yang Finder panggil di bawah selubung dan lebih disukai untuk folder apa pun yang berisi resource fork khusus macOS.

Batas lampiran email yang memblokir folder ZIPed pada 2026

Batas khusus penyedia mendorong sebagian besar kesalahan "ZIP folder saya tidak akan mengirim". Pemblokir hampir selalu adalah langit-langit penerima, bukan batas pengiriman Anda - ketahui keduanya sebelum Anda mengkompres, karena ZIP folder 30 MB yang Gmail Anda kirim baik-baik saja masih memantul terhadap penerima Outlook 25 MB. Folder sangat terpengaruh karena arsip direktori cenderung membundel puluhan item sekaligus; satu foto besar di dalam folder cukup untuk mendorong seluruh ZIP melebihi batas penerima.

  • Gmail: kirim 25 MB / terima 50 MB. Lampiran lebih besar otomatis dialihkan melalui Google Drive.
  • Outlook (Microsoft 365): pengiriman default 20 MB; terima 25 MB untuk sebagian besar paket. Tenant perusahaan dapat dikonfigurasi hingga 150 MB tetapi penerima di luar tenant sering masih melihat 25 MB.
  • Outlook.com (gratis): batas pengiriman 10 MB sebelum pengalihan OneDrive.
  • iCloud Mail: 20 MB; Mail Drop Apple muncul untuk lampiran lebih besar (server iCloud menampung file selama 30 hari).
  • Yahoo Mail: 25 MB.
  • SMTP korporat / kustom: sangat bervariasi (10-150 MB). Tanyakan admin Anda atau lakukan pengiriman tes dengan file ~15 MB.

Ketika ZIP melebihi batas penerima, pesan bounce biasanya mengatakan "pesan terlalu besar" atau "554 5.7.1." Jika Anda melihat kesalahan itu, jangan kirim ulang dengan ZIP yang sama - beralih ke strategi tautan transfer file.

Kapan menambahkan kata sandi ke ZIP

Tambahkan kata sandi ketika folder berisi salah satu dari: dokumen hukum, kontrak yang ditandatangani, data HR, catatan keuangan, kredensial login, atau apa pun yang tunduk pada perjanjian kerahasiaan. ZIP yang dilindungi kata sandi mengenkripsi konten file dengan AES-256 (cipher "tradisional" ZIP 2.0 rusak dan tidak boleh digunakan). Penerima membutuhkan kata sandi melalui saluran terpisah - jangan letakkan kata sandi di email yang sama dengan lampiran.

Jalur tercepat dalam browser adalah Zip File yang membuat ZIP yang dilindungi kata sandi tanpa instalasi perangkat lunak. Jika Anda hanya perlu membuka ZIP yang sudah dilindungi kata sandi yang Anda kontrol, alat yang cocok adalah Hapus Kata Sandi ZIP. Untuk matriks keputusan singkat tentang siapa yang harus tahu apa, lihat panduan terkait tentang jenis kata sandi PDF: pemilik vs pengguna - taksonomi yang sama berlaku untuk izin akses vs edit ZIP.

Kapan melewatkan ZIP dan menggunakan layanan transfer file

ZIP adalah jawaban yang tepat untuk folder di bawah langit-langit lampiran penerima. Di atas itu, alur kerja berbasis lampiran menjadi rapuh - penerima memantul, filter spam menandai, kuota kotak masuk perusahaan terisi. Tiga sinyal bahwa layanan transfer file (Google Drive, OneDrive, Dropbox, WeTransfer, atau hosting Anda sendiri) mengalahkan lampiran:

  • ZIP lebih besar dari 20 MB menuju akun email konsumen apa pun.
  • Penerima pada email korporat yang membatasi (hukum, keuangan, kesehatan sering memiliki filter ukuran dan konten yang agresif).
  • Folder berat media (video, foto mentah) di mana kompresi tidak akan mengurangi ukuran secara berarti - hosting lebih efisien daripada melampirkan.

Jika Anda perlu penerima dapat melihat konten tanpa mengunduh terlebih dahulu (tinjauan hanya-baca), tautan yang dapat dibagikan ke folder cloud mengalahkan lampiran ZIP pada setiap sumbu: dapat dipratinjau, selalu terkini, dan dapat dicabut secara individual.

Mengurangi ukuran ZIP sebelum mengirim

Jika folder Anda ZIP ke tepat di atas batas ukuran, empat pengurangan pragmatis sebelum beralih ke transfer file:

  • Kompres gambar terlebih dahulu. JPG pada kualitas 85 biasanya tidak dapat dibedakan dari sumber dan 30-50% lebih kecil. Jalankan gambar longgar melalui Compress Image sebelum ZIP.
  • Konversi foto HEIC dari iPhone. File HEIC, setelah berada di dalam ZIP, tidak dapat dipratinjau pada banyak penerima Windows. Konversi via HEIC to JPG terlebih dahulu - JPG lebih besar per-file tetapi terbuka di mana-mana tanpa instalasi codec. Lihat HEIC vs JPG vs WebP untuk keputusan format lengkap.
  • Hapus metadata macOS. Di macOS, arsip Finder menyertakan kebisingan __MACOSX dan .DS_Store. Perintah ditto menghapus ini; sebagai alternatif, kompres pada mesin Linux / Windows yang tidak pernah memproduksinya.
  • Pisahkan menjadi beberapa ZIP. Banyak klien email menerima 2-3 lampiran dengan lebih anggun daripada satu ZIP besar. Kelompokkan menurut topik (dokumen.zip + foto.zip) daripada batas byte sewenang-wenang.

Pertanyaan Yang Sering Diajukan

Mengapa ZIP saya masih melebihi batas setelah kompresi?

Karena folder sumber didominasi oleh konten yang sudah dikompres. JPG, PNG, MP3, MP4, DOCX, dan XLSX semuanya pra-dikompres secara internal; algoritma DEFLATE ZIP memiliki sedikit entropi tambahan untuk dihapus. Untuk folder di mana 80%+ byte berasal dari format ini, harapkan hanya beberapa persen pengurangan. Dalam kasus tersebut, beralih ke layanan transfer file daripada mencoba memeras ZIP di bawah batas.

Bisakah saya melindungi ZIP folder dengan kata sandi di Windows tanpa perangkat lunak tambahan?

Perintah shell Windows "Kirim ke → Folder terkompres" tidak mendukung lindungi-kata-sandi secara natif. Anda memiliki tiga opsi: (a) gunakan alat berbasis browser Zip File kami untuk membuat ZIP yang dilindungi kata sandi, (b) instal 7-Zip dan pilih "Tambah ke arsip..." dengan kata sandi, atau (c) gunakan PowerShell dengan cmdlet Compress-Archive plus langkah enkripsi pihak ketiga. Alat browser adalah opsi gesekan terendah karena tidak ada instalasi yang diperlukan.

Bagaimana jika penerima mengatakan "ZIP rusak" atau "ekstrak gagal"?

Tiga penyebab umum: (1) pemindai virus penyedia email memodifikasi arsip dalam perjalanan (tidak biasa tetapi terjadi dengan gateway antivirus yang lebih lama); (2) ZIP dibuat dengan Windows Explorer dan berisi nama file non-ASCII yang ekstraktor penerima tidak dapat decode; (3) unduhan dipotong (kurang mungkin pada HTTP modern tetapi masih mungkin pada jaringan seluler tidak stabil). Perbaikan: re-kompres folder pada OS yang berbeda (jika dibuat macOS, coba lagi di Windows atau gunakan ditto dengan --sequesterRsrc), atau unggah ulang via layanan transfer file yang memverifikasi checksum sisi server sebelum mengekspos tautan unduhan.

Ringkasan

Mengkompres folder untuk email adalah operasi shell native 10 detik di Windows dan macOS. Keputusan nyata adalah apakah ZIP yang dihasilkan muat di bawah langit-langit lampiran penerima - jika ya, kirim dengan menu shell klik kanan; jika tidak, pilih antara membagi menjadi ZIP yang lebih kecil dan beralih ke tautan transfer file. Tambahkan kata sandi kapan pun folder berisi apa pun yang diatur atau rahasia, dan ingat bahwa HEIC dan media yang sudah dikompres tidak akan menyusut banyak - konversi atau host yang terpisah.

Siap untuk membuat ZIP Anda? Mulai dengan Zip File untuk kompresi berbasis browser dengan perlindungan kata sandi opsional. Untuk arah sebaliknya, alat Hapus Kata Sandi ZIP membuka arsip yang sudah Anda miliki izin untuk membuka.
