Last updated: Nov 13, 2024

Ensuring your PDF files meet industry standards is essential, especially when dealing with print, legal, or archival formats like PDF/A. This preflight PDF online tool helps identify errors and ensures your PDF files comply with formats like PDF/A-1B and PDF/A-1A. With a simple online interface, you can preflight your PDFs quickly and verify that they meet the necessary criteria, all without installing software.


Pick a target page size from the dropdown - the included presets cover the standard print, photo-book, and trade-book sizes - and Preflight will check the document against that frame before you export. The full set of PDF utilities lives on the PDF tools hub.

Preflight PDF is the validator step among the PDF tools on this site: it audits an existing document against a target dimension and format profile, then reports compliance, rather than authoring a new file. If you instead need to create a fresh PDF from photos in a specific size, open Images to PDF; the same dimension catalog applies, but Images to PDF lays out new pages from images while Preflight inspects an existing one. The validator is read-only by design - it reports the non-compliance issues but does not auto-fix them; the actual fix happens in the source application (InDesign font embedding, Word font policy, image-resolution upscale) before you re-export and re-preflight. Two PDF/A profiles are supported here: PDF/A-1B (the print-shop default, embeds every visual element) and PDF/A-1A (PDF/A-1B plus tagged-PDF accessibility for screen-reader-mandated documents). PDF/A-2, PDF/X, and PDF/UA are out of scope for this validator. Knowing which profile your destination requires before you upload saves the round-trip of running the check, learning the profile was wrong, and uploading again - the fix on the source side is the same, but the report line items differ by profile.

The dimension dropdown above is shared with Images to PDF for a reason: both pages need to express the same set of canonical print, photo-book, and trade-book sizes, and a single curated catalog keeps the two pages internally consistent. The difference is direction of fit. Preflight expects an existing PDF as input and reports whether its trim size already matches the chosen dimension (a pass/fail report, no file is written). Images to PDF accepts JPG and PNG uploads and writes a fresh PDF at the chosen dimension (a new file, no validation). Pick this page when the document already exists; pick the other page when the document does not yet exist.

Where the validator role specifically diverges from the creator role. Preflight reads four signals out of the uploaded PDF that Images to PDF never touches: the embedded font dictionary (PDF/A-1B requires every font subsetted into the file), the tag tree (PDF/A-1A requires a structured tag tree with reading order, alt text on figures, and table-cell scoping), the colour-profile metadata (ICC tags on every device-colour image), and the trim-box geometry against the dropdown's selected dimension. Each of those four is a pass/fail line item in the report; none of them produces a file as output. Images to PDF, by contrast, writes those same four properties on a brand-new file from defaults the converter picks - so the natural workflow for a print-shop-bound document is Compose or Images-to-PDF first, then Preflight as the last step before sending the file to the printer, not the other way round.
