A worked example helps anchor the 100 KB target: a 200 MB folder of plain-text logs or CSV exports typically packs down to 30-50 MB inside a single .zip, a real two-thirds cut driven by DEFLATE removing repeated tokens. The same 200 MB stored as JPG photos packs to roughly 198-205 MB, because the JPG codec already squeezed out the redundancy the ZIP wrapper would otherwise compress. The mix of inputs sets the ceiling, not the level dial.

To compress a ZIP under 100 KB: pick inputs that pack well (plain-text, CSV, source code), set the compression level slider to a value that fits the runtime budget, and split the delivery instead when the inputs cannot fit. Already-compressed JPG, MP4, and PDF inputs ignore the slider.
