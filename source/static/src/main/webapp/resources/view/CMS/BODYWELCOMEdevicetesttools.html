If a test shows no signal, walk this five-row checklist before assuming the hardware is broken

Most camera, microphone, keyboard, or LCD failures on this hub resolve in one of five places: the browser's site-level permission for that sensor (look for the camera / microphone icon in the address bar), an HTTPS origin (Web APIs refuse to start on plain HTTP), the operating system's privacy panel for the device, another app already holding the sensor open, or — on phones — the iOS / Android sensor-permission prompt that only fires after the first user tap. Work the checklist top-to-bottom; the test re-runs the moment the permission flips.

Four short tests, one for each piece of meeting-day hardware

The hub bundles the four tests that actually break a video call or recording: Camera Test confirms the lens is reachable and the orientation is correct, Microphone Test plots the live input level so you can hear whether the gain is set, Keyboard Test highlights every key as you press it so a stuck or dead key is obvious within seconds, and LCD Test cycles full-screen solid colours so backlight bleed and stuck pixels appear against a uniform field. Run each one in isolation before booking a conference room or sending a warranty return.
