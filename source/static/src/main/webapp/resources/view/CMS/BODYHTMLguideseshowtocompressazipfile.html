Como comprimir un archivo zip

Necesitas un .zip mas pequeno para adjuntar a un correo o subir a un formulario? La receta en tres pasos abajo encamina el archivo por un creador de ZIP en navegador y escribe un archivo empacado en menos de un minuto. Nada se instala, no se crea cuenta, y el archivo se queda en la pestana del navegador hasta que haces clic en descargar.

Pregunta del lector"como comprimir un archivo zip" - la receta paso a paso

Los tres pasos

  1. Abre el creador de ZIP en navegador en /zip-file.html. Arrastra la carpeta (o los archivos sueltos) al espacio de subida.
  2. Elige el objetivo de compresion. Un .zip regular es el predeterminado; si el destinatario acepta archivos protegidos con contrasena, usa zip con contrasena en su lugar. La herramienta empaqueta el objetivo elegido inline.
  3. Descarga el .zip cuando la barra de progreso termine. El archivo es el que subes, adjuntas o compartes.

Cuando la compresion ZIP realmente ahorra bytes

La respuesta honesta de entrada: el empaquetado ZIP encoge entradas comprimibles (texto plano, codigo fuente, CSV, JSON, imagenes sin comprimir) de forma notable; no encoge medios ya comprimidos (JPG, MP4, MP3, PDF) por mas que unos pocos por ciento, y el overhead del empaquetado ZIP incluso puede hacer que el archivo sea ligeramente mas grande que la suma de las entradas en ese caso. Si el objetivo es un archivo mas pequeno de archivos de medios, la ganancia en bytes viene de re-codificar los medios primero - la guia relacionada Comprimir ZIP - cuando ahorra espacio recorre la regla de decision. Si el objetivo es un tamano-objetivo especifico (bajo 25 MB para correo, bajo 100 KB para un formulario), la guia hermana Como comprimir un archivo ZIP a un tamano especifico cubre el trade-off calidad-vs-tamano.

Preguntas frecuentes

El archivo se sube a un servidor?

El pipeline de subida en /zip-file.html corre server-side - la herramienta en navegador envia los archivos al servicio de procesamiento por HTTPS, el archivo se escribe alli, y el enlace de descarga se sirve de vuelta a la pagina. Los archivos se eliminan del servicio de procesamiento despues de una ventana corta de retencion. El contrato de subida compartido para toda herramienta server-side esta documentado en cada pagina de herramienta.

Cual es el tamano maximo de archivo que puedo comprimir?

El limite de tamano por subida se establece en el bootstrap de la pagina (el valor renderizado en la pagina de la herramienta al cargar). Para un archivo rutinario amigable con correo, el limite practico es el tope de adjunto del buzon receptor (Gmail 25 MB, Outlook y buzones corporativos frecuentemente 10-20 MB). Para un archivo mas grande, comparte via un enlace de drive en nube en lugar de un adjunto de correo.

Deberia usar ZIP o 7z o RAR?

ZIP es la opcion mas segura para compartir entre plataformas porque cada sistema operativo principal puede abrir un .zip sin instalar software extra. 7z comprime mas en algunas entradas pero el destinatario puede necesitar un extractor. RAR rara vez es la opcion correcta para compartir anonimamente porque no es abrible por defecto en la mayoria de los sistemas.
