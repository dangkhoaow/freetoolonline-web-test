Tiga niat nyata berada di bawah pencarian: (1) bundel beberapa file ke dalam satu .zip untuk dikirim atau disimpan - gunakan Zip File; (2) buka .zip yang Anda terima dan keluarkan file-nya - gunakan Unzip File; (3) perkecil satu file besar (foto, video) - itu kompresi gambar atau video, bukan ZIP. Mengapa "konverter file zip" adalah pencarian yang menyesatkan

"Konversi ke ZIP" terdengar seperti mengonversi JPG ke PNG atau HEIC ke JPG, tetapi ZIP bekerja pada lapisan yang berbeda. JPG, PNG, dan HEIC adalah format gambar - mereka menggambarkan bagaimana pixel dienkode di dalam satu file. ZIP adalah format arsip - membungkus sejumlah file (dalam format apa pun) ke dalam satu wadah .zip dan menerapkan kompresi tujuan umum pada byte. File asli mempertahankan format aslinya di dalam arsip. Ketika Anda mengekstrak, Anda mendapatkan JPG, PDF, dan DOCX yang sama yang Anda mulai.

Ini berarti "konverter file zip" benar-benar melakukan satu dari dua hal, tergantung arahnya. Masuk: mengambil file lepas Anda dan menghasilkan satu .zip - ini adalah "buat" atau "bundel", bukan "konversi". Keluar: membaca .zip dan menulis setiap file kembali sebagai dirinya sendiri - ini adalah "ekstrak" atau "unzip", juga bukan "konversi". Kata "konverter" melekat karena setiap operasi file lain di peramban disebut konverter, tetapi mekaniknya di sini berbeda.

Apa yang sebenarnya dikompres (dan tidak dikompres) ZIP

ZIP menerapkan DEFLATE - encoder entropi tujuan umum. Bagus dalam memeras redundansi dari teks, kode sumber, data mentah, dan gambar tanpa kompresi. Hampir tidak melakukan apa pun pada jenis file yang sudah memiliki kompresi internal sendiri. Pengurangan ukuran yang diharapkan:

  • Teks, kode, CSV, file log: 60-90% lebih kecil. Kompresi nyata.
  • Dokumen Office (DOCX, XLSX, PPTX): 1-5% lebih kecil. Ini sudah berupa file ZIP secara internal.
  • Foto JPG, PNG, WebP, HEIC: 0-2% lebih kecil. Codec gambar sudah mengompres pixelnya.
  • Audio MP3, AAC, FLAC, OGG: 0-1% lebih kecil. Sudah dioptimalkan entropi.
  • Video MP4, MOV, MKV, WebM: 0-1% lebih kecil. Format paling dikompres berat dalam penggunaan umum.

Untuk folder yang didominasi foto atau video, langkah ZIP sebagian besar tentang bundling (satu unduhan alih-alih lima puluh), bukan mengecilkan. Jika tujuannya benar-benar mengecilkan satu file media besar, ZIP adalah alat yang salah - jangkau kompresi gambar pada foto (Compress Image), konversi format pada HEIC iPhone (HEIC to JPG), atau tautan transfer file untuk video.

"Konversi ZIP ke PDF / JPG / sesuatu yang lain"

Frase ini biasanya berarti salah satu dari dua situasi spesifik. Pertama: Anda menerima .zip yang berisi satu PDF atau JPG, dan Anda ingin file itu keluar dari arsip. Alat yang tepat adalah Unzip File - membaca .zip dan menulis file kembali sebagai diri mereka sendiri. Tidak ada konversi yang terlibat; PDF di dalam .zip sudah merupakan PDF. Kedua: Anda memiliki beberapa PDF yang ingin Anda kemas bersama, dan Anda mengetik "konversi ZIP ke PDF" karena kesalahan. Alat yang tepat adalah Compose PDF - menggabungkan beberapa PDF menjadi satu dokumen. ZIP juga akan membundelnya, tetapi penerima harus melakukan unzip dulu.

Pertanyaan yang Sering Diajukan

Apakah ZIP format file yang harus saya "konversi" sebelum mengirim?

Hanya jika Anda memiliki beberapa file untuk satu penerima. ZIP adalah jawaban yang tepat ketika alternatifnya adalah mengirim lima lampiran secara terpisah - satu .zip lebih mudah di kedua sisi. Untuk satu file, ZIP tidak perlu dan sering tidak mengecilkannya. Kirim filenya sendiri, atau kompres langsung dengan alat per-format yang tepat (Compress Image untuk foto).

Mengapa .zip saya berakhir hampir seukuran aslinya?

Karena yang asli sudah dikompres. JPG, MP3, MP4, dan dokumen Office modern semuanya membawa kompresi mereka sendiri secara internal; DEFLATE di dalam ZIP tidak dapat memeras byte yang sama untuk kedua kalinya. Jika bundel masih terlalu besar untuk dikirim, jawabannya bukan kompresor yang lebih kuat - itu kompresi per-format pada item terbesar, atau tautan transfer file untuk video.

Haruskah saya menggunakan 7z atau RAR alih-alih ZIP?

Gunakan ZIP kecuali Anda memiliki alasan. ZIP terbuka secara native di Windows, macOS, iOS, dan Android tanpa perangkat lunak tambahan; 7z dan RAR membutuhkan aplikasi terpisah di sisi penerima. Keuntungan rasio kompresi 7z kecil pada jenis file yang sebenarnya dikirim orang (media yang sudah dikompres). Tabel trade-off ada di online ZIP vs 7z vs RAR - mana yang dipilih.
