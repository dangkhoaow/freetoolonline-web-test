"File zip online" adalah dua tugas pembaca di bawah satu kueri pencarian: membangun ZIP baru dari folder file di browser, atau membuka .zip yang sudah ada yang Anda terima. Apa yang sebenarnya dilakukan pembuat

Pembuat di /zip-tools/zip-file.html menerima file tunggal, pilihan multi-file, atau seluruh folder (dengan subfolder dipertahankan) dan menulis satu arsip ZIP yang Anda unduh di browser. File diunggah melalui HTTPS ke AWS, ZIP dibangun di sisi server, dan file sumber otomatis dihapus dari layanan pemrosesan setelah jendela retensi singkat - tidak ada penyimpanan persisten masukan, tidak perlu akun atau masuk, dan tidak ada pengiriman email. Hasilnya adalah satu .zip yang dibuka dengan unzip bawaan di Windows, macOS, dan sebagian besar manajer file Linux.

Kapan "file zip online" adalah pilihan tepat

Mengelompokkan kiriman multi-file ke dalam satu .zip adalah alasan paling umum pembaca mencapai halaman ini: serah terima proyek (file sumber + aset + README) masuk dalam satu amplop sehingga penerima tidak harus mengunduh sembilan lampiran; seluruh folder kerja diunggah ke tautan berbagi sebagai satu item alih-alih puluhan. Jalur kompresi massal pembuat dibangun persis untuk ini - jatuhkan folder, beri nama arsip di modal Pengaturan sehingga penerima melihat topik saat tiba, dan unduh satu .zip. Pemotongan ukuran pada ZIP multi-file berasal dari masukan yang berat teks (log, CSV, JSON, kode sumber) - foto dalam JPG, video dalam MP4, audio dalam MP3, dan sebagian besar PDF hanya mendapat 0 hingga 5 persen dari pembungkus karena sudah dikompresi gaya DEFLATE saat ditulis.

Kata sandi opsional (Standar / AES-128 / AES-256)

Modal Pengaturan mengekspos bidang kata sandi opsional dengan tiga pilihan enkripsi: enkripsi Standar ZIP (diterima secara universal tetapi lama), AES-128 (modern, diterima oleh Windows Explorer dan Archive Utility macOS saat ini) dan AES-256 (tingkat tertinggi, plafon kompatibilitas yang sama dengan AES-128). Pilih Standar saat platform penerima tidak diketahui atau file bersensitivitas rendah; pilih AES-128 atau AES-256 saat kata sandi adalah penghalang privasi sebenarnya dan penerima menggunakan OS atau alat unzip terkini. Bagikan kata sandi melalui saluran terpisah (chat, SMS, panggilan telepon) - mengirim .zip dan kata sandi dalam email yang sama mengalahkan tujuan lapisan kata sandi.

Cara membuka .zip di sisi penerima

File .zip standar dibuka dengan unzip yang sudah dibangun ke dalam setiap OS modern - pengguna Windows mengklik kanan dan memilih Extract All, pengguna macOS mengklik dua kali dan arsip mengembang di sebelahnya, manajer file Linux mengekspos opsi Extract di submenu klik kanan. Di ponsel alurnya sama: iOS Safari mengunduh ke aplikasi Files di mana satu ketukan menampilkan konten; Android menyimpan .zip di Downloads di mana manajer file sistem (atau Files by Google) menawarkan Extract dari menu tekan lama. Tidak ada instalasi pihak ketiga yang dibutuhkan di kedua platform. Penerima yang lebih suka tidak menjalankan instalasi (laptop terkelola yang memblokir perangkat lunak pihak ketiga, misalnya) juga dapat membuka arsip di browser di /unzip-file.html.

Apa yang TIDAK dilakukan pembuat

Pembuat ZIP berbasis browser tidak membuka kunci arsip yang kata sandinya tidak diketahui - ia membangun arsip baru dan (secara opsional) mengenkripsinya dengan kata sandi yang Anda berikan, tetapi tidak memecahkan lapisan kata sandi pada arsip yang dibangun orang lain. Ia juga tidak menghasilkan arsip 7z atau RAR - kedua format ini mengompresi sedikit lebih padat daripada ZIP, tetapi memerlukan instalasi pihak ketiga di sebagian besar platform penerima, jadi pembuat tetap pada ZIP untuk jangkauan lintas platform. Untuk pemulihan kata sandi yang tidak diketahui, alat remove-zip-password hanya bekerja saat kata sandi diketahui di sisi pengirim; untuk tugas terkait di kategori lain (meratakan PDF, kompresi gambar, hash) hub alat ZIP menunjuk ke saudara yang tepat.
