Frequently Asked Questions

What does the JSON Compare tool do?

It reads two JSON documents - one in each editor - and reports the differences keyed by JSON path. Three categories appear in the diff list: keys added in B but missing in A, keys removed in B but present in A, and keys present in both but with different values. The comparison runs in your browser; the payloads never leave the page.

When should I reach for JSON Compare instead of JSON Formatter?

Reach for JSON Compare when the question is "what changed between A and B" - two API responses you want to A/B against each other, two config files that may have drifted, or a staging payload that is silently missing a field present in prod. Reach for the single-pane JSON Formatter or JSON Formatter Editor when the question is "is this one document valid and well-shaped".

What complementary tools work well alongside JSON Compare?

JSON Formatter and JSON Formatter Editor pretty-print and validate a single payload before you paste it into the comparator. JSON Formatter Viewer renders a payload as a collapsible tree if you only need to read, not diff. For free-form text diffs against non-JSON content, use Text Diff instead.
