Cara mengompres file zip

Butuh .zip yang lebih kecil untuk dilampirkan ke email atau diunggah ke formulir? Resep tiga langkah di bawah merute file melalui pembuat ZIP dalam browser dan menulis arsip yang dikemas dalam kurang dari satu menit. Tidak ada yang diinstal, tidak ada akun yang dibuat, dan arsip tetap di tab browser sampai Anda klik unduh.

Tiga langkah

  1. Buka pembuat ZIP dalam browser di /zip-file.html. Seret folder (atau file lepas) ke slot unggah.
  2. Pilih target kompresi. .zip biasa adalah default; jika sisi penerima menerima arsip yang dilindungi kata sandi, gunakan zip dengan kata sandi. Alat mengemas target yang dipilih inline.
  3. Unduh .zip ketika bilah kemajuan selesai. Arsip adalah yang Anda unggah, lampirkan, atau bagikan.

Kapan kompresi ZIP benar-benar menyimpan byte

Jawaban jujur di muka: pengemasan ZIP mengecilkan input yang dapat dikompresi (teks polos, kode sumber, CSV, JSON, gambar tidak terkompresi) secara nyata; ia tidak mengecilkan media yang sudah terkompresi (JPG, MP4, MP3, PDF) lebih dari beberapa persen, dan overhead pengemasan ZIP bahkan dapat membuat arsip sedikit lebih besar daripada jumlah input dalam kasus itu. Jika tujuannya adalah arsip yang lebih kecil dari file media, kemenangan byte berasal dari re-encoding media terlebih dahulu - panduan terkait Kompres ZIP - kapan menyimpan ruang berjalan melalui aturan keputusan. Jika tujuannya adalah ukuran target tertentu (di bawah 25 MB untuk email, di bawah 100 KB untuk formulir), panduan saudara Cara mengompres file ZIP ke ukuran tertentu mencakup trade-off kualitas-vs-ukuran.

Pertanyaan yang sering diajukan

Apakah file diunggah ke server?

Pipeline unggah di /zip-file.html berjalan server-side - alat dalam browser mengirim file ke layanan pemrosesan melalui HTTPS, arsip ditulis di sana, dan tautan unduhan dilayani kembali ke halaman. File dihapus dari layanan pemrosesan setelah jendela retensi singkat. Kontrak unggah bersama untuk setiap alat server-side didokumentasikan di setiap halaman alat.

Berapa ukuran file maksimum yang bisa saya kompres?

Batas ukuran per unggah ditetapkan dalam bootstrap halaman (nilai yang dirender ke halaman alat saat memuat). Untuk arsip rutin yang ramah email, batas praktis adalah batas lampiran kotak masuk penerima (Gmail 25 MB, Outlook dan kotak masuk korporat sering 10-20 MB). Untuk arsip yang lebih besar, bagikan melalui tautan drive cloud daripada lampiran email.

Haruskah saya menggunakan ZIP atau 7z atau RAR?

ZIP adalah pilihan paling aman untuk berbagi lintas-platform karena setiap sistem operasi utama dapat membuka .zip tanpa menginstal perangkat lunak tambahan. 7z mengompresi lebih ketat pada beberapa input tetapi penerima mungkin memerlukan ekstraktor. RAR jarang menjadi pilihan yang tepat untuk berbagi anonim karena tidak dapat dibuka secara default pada sebagian besar sistem.
