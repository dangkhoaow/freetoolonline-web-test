Was Flachen einer PDF tatsaechlich tut

Eine PDF (das Seitenbeschreibungsformat definiert durch ISO 32000-1 / ISO 32000-2) ist ein Baum geschichteter Objekte. Die sichtbare Seite ist ein Strom von Zeichen-Operatoren; darueber kann die Datei ein AcroForm-Woerterbuch tragen (ausfuellbare Formularfelder), ein Array von Anmerkungen pro Seite (Kommentare, Stempel, Haftnotizen, freier Text, Tintenmarkierungen, Redaktionsmarken), ein optionales Signatur-Woerterbuch (die kryptografische Signatur plus ihre sichtbare Erscheinung), und optionale Optional Content Groups (Ebenen, die ein- oder ausgeschaltet werden koennen). All das sitzt "ueber" der statischen Seite wie transparente Overlays. Eine Flachen-Operation laeuft durch diesen Baum und malt jedes Overlay als einfache Tinte in den darunterliegenden Seiteninhaltsstrom: Der Formularfeldwert wird zu statischem Text, der an der Position des Felds gezeichnet wird, die Anmerkung wird zu einer festen Form mit fester Farbe, die sichtbare Erscheinung der Signatur wird zu einem statischen Bild des Signatur-Glyphen, und die Ebenen-Umschalt-Struktur wird auf das kollabiert, was zum Flachen-Zeitpunkt sichtbar war. Nach dem Flachen zeigt das Oeffnen der PDF in jedem Reader dieselben Pixel - aber das AcroForm-Woerterbuch, Anmerkungs-Arrays und Optional Content Groups wurden entfernt, sodass nichts mehr zum Ausfuellen, Bearbeiten oder Umschalten uebrig ist.

Wann Sie tatsaechlich flachen muessen

Flachen loest eine konkrete Problemkategorie: Die PDF muss ueberall identisch rendern und nicht-interaktiv bleiben. Die vier haeufigen Situationen:

  • E-Signing-Dienste, die geschichtete PDFs ablehnen. Viele E-Sign-Plattformen weigern sich, eine finale kryptografische Signatur auf eine PDF anzuwenden, die noch lebende Formularfelder oder unsignierte Signatur-Platzhalter hat, weil sich die Formularwerte nach dem Signieren aendern koennen. Das Formular zuerst zu flachen verwandelt die Feldwerte in statischen Text, danach kann der E-Sign-Schritt das gesamte Dokument in einen kryptografischen Umschlag huellen.
  • Gerichts- / Audit- / Regulierungseinreichungen. Einreichungen spezifizieren oft ein statisches Archivformat - manchmal PDF/A, manchmal "geflachte PDF". Der Pruefer braucht die Gewissheit, dass die Datei, die er erhalten hat, genau das ist, was jeder andere Pruefer sehen wird; geschichtete Objekte und Formularfelder schaffen die Moeglichkeit einer anders gerenderten Seite auf einem anderen Reader.
  • Annotierte Entwuerfe als Final teilen. Wenn die redaktionelle Pruefung abgeschlossen ist und die markierte PDF als Final versendet werden soll, baeckt Flachen die Kommentare und Markierungen in die Seite, damit sie nicht von einem nachgelagerten Leser geloescht werden koennen, der auf den falschen Knopf klickt. Die Anmerkungen bleiben sichtbar als Tinte statt als entfernbare Overlays.
  • Eine signierte Seite sperren, damit die sichtbare Signatur nicht driftet. Eine sichtbare Signatur-Erscheinung, die nur als Anmerkung existiert, kann von einem Editor verschoben, versteckt oder entfernt werden. Flachen wandelt die Erscheinung in statischen Seiteninhalt um; das kryptografische Signatur-Woerterbuch ist eine separate Angelegenheit, aber der sichtbare Glyph hoert auf, verschiebbar zu sein.

Wann NICHT zu flachen

Flachen ist einwegig. Die ursprungliche geschichtete Struktur ist nach dem Flach-Schritt weg; Sie koennen eine gebackene PDF nicht wieder in ein ausfuellbares Formular entflachen. Drei Faelle, in denen Flachen der falsche Zug ist:

  • Nachgelagerte Leser muessen das Formular noch ausfuellen. Wenn die PDF ein Steuerformular, ein Stellenangebot oder eine leere Vorlage ist, in die der naechste Leser tippen soll, hinterlaesst das Flachen vor dem Senden ihnen ein statisches Bild leerer Kaesten. Senden Sie die geschichtete Datei; lassen Sie den Empfaenger ausfuellen, dann bitten Sie sie, vor der Rueckgabe zu flachen, wenn der Workflow eine unveraenderliche Kopie braucht.
  • Ein zukuenftiger Editor erwartet, weiter zu bearbeiten. Geschichtete Anmerkungen und Kommentare sind, wie Pruefer kommunizieren. Das Flachen eines Entwurfs loescht den Dialog - der naechste Pruefer kann einen veralteten Kommentar nicht loeschen, weil er kein Kommentar mehr ist, sondern nur Tinte auf der Seite. Behalten Sie die geschichtete Version fuer den redaktionellen Loop und flachen Sie nur das Final.
  • Getaggte-PDF-Barrierefreiheit haengt von der logischen Struktur ab. Eine getaggte PDF traegt einen Strukturbaum (Absaetze, Ueberschriften, Listenelemente, Tabellenzellen), den Screenreader durchqueren. Einige Flachen-Implementierungen erhalten Tags; andere kollabieren sie. Wenn das Dokument von unterstuetzender Technologie konsumiert wird, ueberpruefen Sie, ob die Tags den Flach-Schritt ueberleben, oder flachen Sie ueberhaupt nicht.

Was Flachen NICHT ist

Drei Operationen werden oft mit Flachen verwechselt; jede ist eine andere Aktion mit einem anderen Ergebnis.

  • "Als PDF speichern" aus einem Druckdialog. Dies rendert die PDF durch einen virtuellen Drucker neu, was normalerweise Ebenen flacht - aber es kann auch Vektorinhalt neu rastern (sodass Text zu einem Bild von Text wird und nicht mehr auswaehlbar / durchsuchbar ist). Flachen erhaelt die zugrundeliegenden Zeichen-Operatoren, wo immer moeglich; Print-to-PDF moeglicherweise nicht.
  • Eine PDF komprimieren. Komprimierung schreibt die Seiteninhaltsstroeme neu, um weniger Bytes zu nehmen - sie entfernt NICHT das AcroForm-Woerterbuch, die Anmerkungs-Arrays oder Ebenen. Eine komprimierte PDF ist immer noch bearbeitbar; eine geflachte PDF ist es nicht. Siehe PDF Preflight Online - Was es tatsaechlich prueft fuer das Strukturpruefungs-Gegenstueck, das auch von Flachen verschieden ist.
  • Ein Passwort entfernen. Passwortentfernung aendert das Verschluesselungs- / Berechtigungswoerterbuch; sie beruehrt die geschichteten Objekte nicht. Eine PDF mit entferntem Passwort ist immer noch ausfuellbar / bearbeitbar.
