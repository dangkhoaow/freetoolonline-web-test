Cara menguji keyboard online, langkah demi langkah

Tinjauan terakhir 2026-05-02. Buka tes keyboard untuk menjalankan langkah-langkah di bawah - tanpa instalasi, tanpa akun, tanpa unggah.

Jawaban 60 detik. Buka tes keyboard di browser Anda, tekan setiap tombol di keyboard fisik Anda, dan amati setiap tombol menyala di tata letak di layar. Tombol yang tidak menyala saat ditekan tidak berfungsi. Tombol yang tetap menyala setelah dilepas macet. Tombol modifier (Shift, Ctrl, Alt, Cmd, Fn) dan tombol kunci (Caps, Num, Scroll) diperiksa dengan cara yang sama.

Mengapa menguji keyboard di browser

Tes browser lebih cepat daripada menginstal utilitas vendor, dan lebih jujur daripada mengetik ke dokumen - mengetik hanya mengungkap tombol yang kebetulan Anda tekan, sedangkan tes berbasis tata letak memaksa Anda berjalan setiap tombol secara berurutan. Browser melihat event tekan tombol yang sama yang diberikan sistem operasi ke aplikasi lainnya, jadi tombol yang gagal dalam tes juga akan gagal di editor, terminal, dan game Anda.

Langkah demi langkah

  1. Buka tes. Kunjungi /keyboard-test.html. Halaman memuat tata letak keyboard di layar. Tanpa login, tanpa permintaan izin.
  2. Jalani setiap tombol. Mulai dari Esc (kiri atas), tekan, amati menyala di tata letak. Pindah ke kanan melewati baris tombol fungsi, lalu turun melalui baris angka, baris QWERTY, dan akhiri di tombol spasi dan tombol panah. Setiap tekanan harus menerangi tombol yang sesuai di layar.
  3. Periksa tombol modifier dan kunci. Tekan Shift, Ctrl, Alt, Cmd (macOS) atau tombol Windows, dan tombol Fn jika keyboard Anda memilikinya. Lalu tekan Caps Lock, Num Lock, dan Scroll Lock. Tombol kunci adalah toggle - tekan sekali untuk mengaktifkan, tekan lagi untuk melepas.
  4. Tes input akord. Tahan Shift dan tombol huruf bersamaan - keduanya harus tetap menyala selama Anda tahan. Lepaskan; keduanya harus bersih. Ulangi dengan Ctrl + huruf, lalu Alt + huruf. Ini menangkap kegagalan tombol modifier yang hanya muncul dalam kombinasi.
  5. Tes perilaku n-key. Tahan W, A, S, D bersamaan (akord gaming standar). Keempat tombol harus menyala. Tambahkan Q sambil masih menahan yang lain. Pada keyboard 6-key-rollover Anda harus melihat kelima; pada keyboard 2-key-rollover, tekan kelima akan gagal.
  6. Cari tombol macet. Setelah melepas setiap tombol, tata letak di layar harus benar-benar bersih. Tombol yang tetap menyala setelah Anda lepas macet - paling sering kotoran di bawah keycap, kadang switch yang aus.

Apa arti setiap kegagalan

  • Tombol tidak menyala saat ditekan. Tombol tidak berfungsi - kotoran, switch aus, atau jejak putus. Coba udara bertekanan dulu; jika gagal, switch kemungkinan perlu diganti pada keyboard mekanik atau keyboard itu sendiri pada keyboard membran.
  • Tombol tetap menyala setelah dilepas. Tombol macet - biasanya kotoran antara keycap dan switch. Lepas keycap, tiup keluar kotoran, pasang kembali. Jika persisten, switch rusak.
  • Tombol modifier gagal hanya saat akord. Matriks atau firmware menjatuhkan modifier di bawah beban. Pada keyboard murah ini normal (2KRO). Untuk mengetik tidak masalah; untuk gaming atau musik bermasalah.
  • Beberapa tombol gagal bersamaan. Sering baris atau kolom matriks yang sama. Jika Q, A, dan Z semua gagal, kolom yang mereka bagi adalah tersangka. Jika Q, W, E, R, T semua gagal, baris adalah tersangka.
  • Keyboard nirkabel melewatkan tombol. Gangguan Bluetooth atau 2.4 GHz. Dekatkan receiver atau tes berkabel jika keyboard mendukungnya.

Yang tidak bisa diperiksa browser

Browser melihat ketukan tombol setelah sistem operasi memprosesnya. Beberapa hal tetap tak terlihat oleh tes:

  • Laju laporan USB. Polling pada 125 Hz, 500 Hz, atau 1000 Hz adalah pengaturan tingkat hardware; browser melihat tekanan saat tiba tetapi tidak mengukur interval pada resolusi sub-milidetik.
  • Pemetaan ulang tingkat firmware. Keyboard gaming dengan memori onboard memetakan ulang tombol sebelum mencapai OS. Browser melihat tombol yang dipetakan ulang, bukan yang fisik.
  • Kebisingan akustik tombol. Click dan clack bersifat fisik; browser melihat event tekan terlepas dari seberapa keras atau pelan switch-nya.
  • Kegagalan backlight atau RGB. Pencahayaan hanya hardware; browser tidak melihat bagaimana tombol Anda terlihat.
