Is your screen OLED rather than LCD? The same color cycle runs on OLED panels, but the failure modes shift: OLED pixels emit their own light, so the backlight-bleed check above cannot occur, and two OLED-specific defects (burn-in and image retention) become the checks worth watching for. Screen test on a hi-DPI or retina laptop

On a retina or hi-DPI laptop panel a single dot the eye reads as "one pixel" is usually four or six physical sub-pixels behind a `devicePixelRatio` of 2 or 3, so a stuck pixel can hide between the CSS pixels the browser paints. The on-page readout names the live viewport size, the screen resolution the operating system reports, and the native screen resolution (the resolution multiplied by `window.devicePixelRatio`) - confirm the third number matches the laptop's spec sheet before declaring a pixel dead, otherwise the suspect dot may just be a sub-pixel anti-aliasing artifact on a scaled display.

Want a deliberate sweep order rather than random clicks? The six color tiles map to different defect classes - run them in a sensible order so each tile earns its place in the test. Red and green make stuck pixels stand out against a uniform field (a stuck pixel emits one of the three sub-pixel primaries). Black surfaces backlight bleed and edge glow on LCD panels; on OLED the same tile reads as the brightest dark-uniformity check. White shows dust, hot pixels, and uniformity dips across the panel. Blue and yellow read color tint and any panel-shift the bezel might be applying. Press Reset (or Escape if true fullscreen is on) between tiles to return to the picker so the next color is a deliberate choice; the page does not record anything between presses.
