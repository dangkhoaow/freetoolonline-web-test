What the three resolution readouts mean. Under the color tiles the page lists three live numbers: your viewport, your screen resolution, and your native screen resolution (the resolution multiplied by window.devicePixelRatio). Confirm the native number matches your laptop or monitor spec sheet before flagging a suspect dot - on a hi-DPI panel a single CSS pixel hides several physical sub-pixels behind it, and a stuck sub-pixel can hide between them.

Why the "Test in full screen mode" box is ticked by default. Tick the full-screen checkbox before you start a sweep - the OS-level fullscreen mode hides browser chrome (address bar, tab strip, taskbar) so a stuck pixel sitting in the top or bottom row of the panel is not masked by UI furniture, and the test color reaches the actual edge of the display the panel manufacturer is responsible for. The box is preserved between sweeps, so once you tick it the next click on a color tile paints edge-to-edge again without re-ticking. On panels where the Fullscreen API is blocked by policy the page falls back to a window-fill paint that still pushes the chosen color across the visible canvas; the practical difference is only the centimetre of chrome at the top of the screen.

When to uncheck the "Test in full screen mode" box. The box is checked by default because a true fullscreen paint hides the browser chrome and lifts the test color edge-to-edge, which makes a stuck sub-pixel easier to catch at arm's length. Uncheck it in three situations: when you want the color tile and a second reference window (a spec sheet, a photo of the suspect dot) visible side-by-side without alt-tabbing; when you are on a kiosk or work laptop where the Fullscreen API is policy-blocked and the request silently fails; or when you want to repeat the same sweep quickly without pressing Escape between tiles. With the box unchecked, each tile click paints the page background instead of the whole display, and Reset returns you to the picker with one click.

Which tile catches which defect? The six color tiles are not interchangeable - each one surfaces a different failure mode. Use the black tile to scan for backlight bleed (uneven glow at the panel edges, only meaningful on LCD - on OLED the pixel itself emits light, so true black is true off and bleed cannot occur). Use the red, green, and blue tiles to catch stuck sub-pixels at the native resolution number the page lists on load; a stuck red sub-pixel is invisible on the red field but jumps out on the green and blue ones. White and yellow round out the sweep for uniform-tint defects.
