Going deeper on the signed-PDF workflow. Once the photos are assembled into a PDF, signing and sharing often follows - and if the recipient previously locked the PDF with an owner password that was since forgotten, the signed version has to start from the unlocked copy. The how-to-sign-pdf-after-removing-a-password guide walks through the remove-then-sign flow end-to-end, including when a fresh scan beats trying to break the existing password.

Bundling photos as one PDF vs zipping them. Combining photos into a single PDF is faster for some workflows than zipping the same images - the recipient opens one file in any PDF viewer instead of unzipping a folder first. Pick PDF for presentations, signed handoffs, slide decks, and long-term archives where every recipient already has a PDF reader. Pick ZIP (built at the ZIP creator) for raw file delivery where the recipient will work on the originals (re-edit, re-export, swap one image, rename files) - the JPG/PNG sources stay intact rather than getting rasterised into PDF pages.
