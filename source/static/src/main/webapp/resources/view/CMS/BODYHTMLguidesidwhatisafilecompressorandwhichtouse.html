Apa Itu Kompresor File dan Mana yang Harus Digunakan

Terakhir ditinjau 2026-04-27. Semua alat di bawah berjalan di browser pada perangkat Anda sendiri - tidak ada file yang meninggalkan komputer atau ponsel Anda.

Jawaban 30 detik. "Kompresor file" bukan satu alat tunggal. Ini adalah kategori dari enam operasi berbeda, dan pilihan yang tepat tergantung jenis file yang Anda miliki. Sebuah folder berisi file campuran: jadikan ZIP. Foto lepas: kompres gambarnya. Foto HEIC iPhone yang tidak bisa dibuka penerima Windows: konversi HEIC ke JPG. Beberapa gambar untuk satu file yang bisa dibagikan: gabungkan ke satu PDF. Beberapa PDF ke orang yang sama: satukan menjadi satu. Satu file sangat besar (video 4K, foto RAW, sumber desain): pakai link transfer file daripada lampiran. Tidak ada kompresor universal karena matematika kompresinya berbeda untuk setiap jenis file.

Mengapa "kompresor file" adalah pencarian yang terbagi lima

Frasa "kompresor file" duduk di paling atas funnel. Dua pertiga orang yang mengetiknya di Google belum memutuskan jenis file apa yang perlu dikompres. Ada yang mencoba mengirim PowerPoint 30 MB lewat email. Ada yang mencoba menyesuaikan folder kuitansi di bawah batas upload 5 MB. Ada yang mencoba mengirim 200 foto iPhone ke desktop Windows. Ada yang mencoba membuat MP4 drone 4 GB cukup kecil untuk dibagikan. Jawaban naif - "pakai kompresor file gratis" - membawa kelima pengguna itu ke alat yang sama, dan empat di antaranya mendapat hasil lebih buruk dibanding kalau mereka memilih alat khusus yang tepat.

Pohon keputusannya sederhana setelah Anda tahu. Bagian sulitnya adalah tidak ada mesin pencari yang dapat membedakan secara andal kelima maksud dari frasa "kompresor file" saja. Enam jenis kompresi, enam matematika berbeda

Orang bilang "kompres" untuk salah satu dari ini. Tiap jenis pakai algoritma yang fundamental berbeda dan menghasilkan pengurangan ukuran yang berbeda untuk input yang sama.

  • Kompresi container tanpa kehilangan (ZIP, 7z, RAR). Membundel banyak file menjadi satu arsip dan mengompres aliran byte dengan DEFLATE / LZMA / LZMA2. Reversibel - file yang diekstrak identik bit-per-bit dengan asli. Sangat baik pada teks dan kode sumber (50-80% pengurangan). Hampir tidak berpengaruh pada media yang sudah dikompres (JPG / MP4 / MP3 / PDF). Pilihan tepat ketika Anda perlu mempertahankan file tepat.
  • Kompresi gambar dengan kehilangan (kualitas JPEG, WebP). Membuang informasi visual yang paling kecil kemungkinan dilihat oleh mata manusia dan mengenkode ulang hasilnya. Menurunkan JPG 5 MB jadi 1 MB pada kualitas 80 tanpa perbedaan terlihat untuk kebanyakan foto. Tidak reversibel - setelah dikompres pada kualitas 60 Anda tidak bisa memulihkan aslinya. Pilihan tepat ketika file adalah satu gambar dan penerima tidak peduli ketepatan pixel-perfect.
  • Konversi format gambar (HEIC ke JPG, PNG ke JPG, TIFF ke JPG). Mengubah format enkoding. Perubahan ukuran adalah efek samping dari pindah dari format kurang efisien ke yang lebih efisien. PNG ke JPG memotong foto 12 MB jadi 1 MB; HEIC ke JPG mempertahankan ukuran mirip tapi membuka semua perangkat Windows / Android. Pilihan tepat ketika file dalam format yang salah untuk penerima.
  • Kompresi video dengan kehilangan (mengenkode ulang pada bitrate lebih rendah). Mengenkode ulang aliran video pada target bit-per-detik yang lebih rendah. Memotong MP4 4K 1 GB jadi 200 MB pada 1080p H.264 dengan kualitas yang dapat diterima. Alat browser menangani klip pendek; video panjang melalui software khusus seperti FFmpeg atau HandBrake.
  • Kompresi audio dengan kehilangan (MP3 / AAC / OGG). Ide yang sama dengan video tapi untuk suara. WAV 50 MB jadi MP3 5 MB pada 192 kbps. ZIP browser tidak membantu; langkah yang benar adalah konversi format.
  • Hosting link transfer. Secara teknis bukan kompresi - file tetap ukuran sama, tapi penerima mengunduh dari layanan alih-alih menerima sebagai lampiran. Ini satu-satunya jawaban praktis untuk file di atas ~25 MB, batas atas sebagian besar sistem email.

Apa yang sebenarnya dilakukan ZIP (dan tidak dilakukan)

ZIP adalah operasi yang pertama terlintas di benak karena sudah jadi kata kerja universal "kompres ini" di Windows sejak 1997 dan macOS sejak 2003. Sangat baik pada satu pekerjaan khusus dan lemah pada kebanyakan lainnya.

Apa yang ZIP kuasai: membundel banyak file, mempertahankan struktur folder, menerapkan password ke bundel, mengompres konten berbasis teks (kode sumber, CSV, dokumen biasa) 50-80%. Outputnya adalah satu file .zip yang bisa Anda lampirkan ke email atau unggah ke portal yang tidak mengizinkan unggah folder. Alat ZIP File menangani ini di browser tanpa upload apa pun.

Apa yang lemah dari ZIP: mengompres media yang sudah dikompres. JPG, PNG, MP3, MP4, HEIC, PDF semua sudah dikompres secara internal; algoritma DEFLATE tidak bisa memeras lebih lanjut. Mengompres ZIP folder berisi 100 foto iPhone biasanya menghemat 1-3% total ukuran, bukan 50%. Jika folder Anda didominasi file media, ZIP adalah alat yang salah. Cara membuat file zip lebih kecil membahas apa yang dilakukan ketika ZIP kurang menghasilkan.

Kapan mengkonversi mengalahkan mengompres

Untuk file tunggal, konversi format sering mengalahkan kompresi dengan margin lebar karena file sumber dalam format salah untuk kasus penggunaan:

  • HEIC ke JPG. HEIC iPhone 3 MB hanya bisa dibuka di Windows dengan codec berbayar; salinan JPG 1,5-3 MB tapi terbuka di mana saja. Konversi tentang kompatibilitas, bukan ukuran. Jalankan melalui HEIC to JPG. Untuk konteks lebih lanjut tentang HEIC lihat HEIC vs JPG vs WebP - kapan menggunakan tiap format.
  • PNG ke JPG (untuk foto). Foto PNG 12 MB menjadi JPG 1,2 MB pada kualitas 85. PNG adalah format tanpa kehilangan yang ditujukan untuk screenshot dan grafik; menggunakannya untuk foto memboroskan 90% ukuran file. Simpan PNG untuk transparansi, line art, atau screenshot pixel-perfect.
  • TIFF / BMP ke JPG. Keduanya 5-10x lebih besar dari JPG untuk konten visible yang sama. Konversi sebelum mengirim; penerima di ponsel sering tidak bisa membuka TIFF sama sekali.
  • WAV ke MP3. WAV 50 MB menjadi MP3 5 MB pada 192 kbps. ZIP browser tidak membantu; langkah yang benar adalah konversi format di software audio khusus.

Privasi: mengapa "hanya-browser" penting

Sebagian besar hasil "kompresor file gratis" mengunggah file Anda ke server, menjalankan kompresi di cloud mereka, dan mengirimi Anda link unduh. Itu baik untuk foto kucing. Itu tidak baik untuk formulir HR, rekam medis, kontrak, kode sumber, atau apa pun yang tunduk pada GDPR / HIPAA. Lima alat yang tercantum dalam panduan ini semuanya berjalan di perangkat Anda sendiri menggunakan API native browser (alat ZIP menggunakan API JavaScript Streams + Compression Streams; alat gambar menggunakan API Canvas + FileReader; konversi HEIC menggunakan libheif yang dikompilasi ke WASM). Tidak ada yang meninggalkan komputer atau ponsel Anda.

Untuk lebih lanjut tentang kategori ini lihat alat online gratis yang berfungsi tanpa mengunggah file.
